## 1. 악성 소프트웨어 개요

### 핵심 정의

- **악성 프로그램**: 의도적으로 컴퓨터의 보안 속성을 침해할 목적으로 작성된 프로그램
- 개발자 ≠ 사용자 → 개발자가 사용자가 원하지 않는 기능을 몰래 포함

### 3대 악성코드 비교

| 구분 | 바이러스 | 웜 | 트로이목마 |
|---|---|---|---|
| 침해 속성 | **무결성** | **가용성** | **기밀성** |
| 자기 복제 | 있음 | **매우 강함** | 없음 |
| 숙주 필요 | 필요 (파일/부트섹터) | 불필요 (독립 실행) | 불필요 |
| 전파 방법 | 사용자가 감염 파일 옮김 | 네트워크로 스스로 전파 | 사용자가 직접 실행 |
| 주요 피해 | 파일·시스템 손상 | 네트워크 성능 저하 | PC 성능 저하, 좀비PC |

### 비유로 기억하기

- **바이러스** = 감기 바이러스처럼 숙주(파일)에 붙어야 퍼짐
- **웜** = 혼자서 네트워크를 타고 기어다니는 벌레. 아무도 안 옮겨도 스스로 퍼짐
- **트로이목마** = 유용한 프로그램인 척 위장. 복제 없이 몰래 나쁜 짓

### 분류 기준

#### 독립형 vs 기생형

| 유형 | 설명 | 예시 |
|---|---|---|
| 기생형 | 다른 프로그램 없이 독립 존재 불가 | 바이러스, 논리폭탄, 백도어 |
| 독립형 | 자체적으로 구동 가능 | 웜, 좀비 프로그램 |

#### Payload(목적)별 분류

| 목적 | 악성코드 종류 |
|---|---|
| 시스템 파괴 | 논리폭탄, Stuxnet, 랜섬웨어 |
| 공격 에이전트 | 좀비, 봇(bot) |
| 정보 유출 | 키로거, 피싱, 스파이웨어 |
| 잠입(스텔스) | 백도어, 루트킷 |

## 2. 바이러스 (Virus)

### 세대별 분류 — 암기법: **원→암→은→갑→매**

| 세대 | 이름 | 핵심 특징 |
|---|---|---|
| 1세대 | 원시형 (Primitive) | 구조 단순, 고정 크기, 분석 쉬움 |
| 2세대 | 암호화 (Encryption) | 코드 일부 또는 대부분 암호화 |
| 3세대 | 은폐형 (Stealth) | 감염 후 파일 크기 안 변한 척, 백신 속임 |
| 4세대 | 갑옷형 (Armour) | 백신 개발자 공격, 분석·개발 지연시킴 |
| 5세대 | 매크로 (Macro) | MS Office 등 매크로 기능 악용, **현재 가장 많음** |

### 일시적 vs 상주형

- **일시적 바이러스**: 호스트 프로그램 실행될 때만 동작, 종료 시 끝
- **상주형 바이러스**: 메모리에 상주, 독립적으로 활성/비활성

### 다형성 바이러스 (Polymorphic Virus)

- 자신의 모습을 스스로 바꿔 바이러스 스캐너를 속임
- 스캐너가 새로운 바이러스로 인식 → 탐지 어려움

### 매크로 바이러스 특징

- **플랫폼 무관** (Windows/Mac 모두 감염 가능)
- 주로 **문서를 감염**시킴 (실행 파일 아님)
- 전파 경로: 주로 **전자메일**
- Adobe PDF도 피해 대상 (JavaScript 포함 가능)

## 3. 기타 악성코드 유형 전체 정리

### 숨는 놈 (잠입/은폐)

#### 백도어 (Backdoor / 트랩 도어)

- 개발자가 몰래 뚫어놓은 **뒷문**
- 정상 인증 과정 완전히 우회
- 원격에서 아무 때나 접속 가능
- 탐지 매우 어려움 (정상 트래픽처럼 보임)
- 대표 사례: **Back Orifice** (1998, Cult of the Dead Cow 제작)

#### 루트킷 (Rootkit)

- OS 깊숙이 숨어서 **자기 존재를 숨기는** 악성코드
- 백신 프로그램에도 안 보임
- 다른 악성코드를 숨겨주는 역할도 함
- 탐지하려면 OS 바깥(부팅 전)에서 검사해야 함

> **백도어 vs 루트킷**
> - 백도어 = 들어오는 **문** (침입 경로)
> - 루트킷 = **숨는 기술** (은폐 도구)
> - 보통 백도어로 침입 → 루트킷으로 숨는 방식으로 같이 사용됨

### 조종당하는 놈

#### 좀비 (Zombie)

- 해커 명령을 기다리는 **원격 조종 PC**
- 평소엔 정상 PC처럼 작동 → 주인은 감염 사실 모름
- 명령 받으면 DDoS 공격, 스팸 발송 등에 동원
- 여러 좀비PC를 묶은 것 = **봇넷(Botnet)**

### 퍼붓는 놈

#### 스패머 프로그램 (Spammer)

- **대량 스팸 메일 자동 발송** 도구
- 피싱 메일, 광고 메일 대량 살포
- 좀비PC를 이용해서 발송하는 경우 많음

#### 플러더 (Flooders)

- **대량 패킷으로 서버/네트워크를 마비**시키는 도구
- DoS/DDoS 공격에 사용
- 가용성(Availability) 침해
- 비유: 편의점 계산대 앞에 사람 1000명이 몰려서 아무도 계산 못하게 만드는 것

### 납치하는 놈

#### 브라우저 하이재커 (Browser Hijacker)

- 브라우저를 **납치**하는 악성코드
- 홈페이지 → 광고 사이트로 강제 변경
- 검색엔진 → 가짜 검색엔진으로 변경
- 광고 클릭 수익이 목적

### 웹에서 실행되는 놈

#### 모바일 코드 (Mobile Code)

웹에서 자동으로 다운받아 실행되는 코드

| 종류 | 방어 시점 | 방어 방식 | 비유 |
|---|---|---|---|
| **ActiveX** | **실행 전** | 서명 검증으로 차단 | 입장 전 신분증 확인 |
| **Java Applet** | **실행 중** | 샌드박스 안에 격리 | 입장은 시키되 격리실에 가둠 |

### 침투/실행 도구

#### Exploit

- 취약점을 공격해 시스템 제어권 획득

#### Downloaders

- 감염 후 **추가 악성코드 자동 다운로드** (선발대 역할)

#### Auto-rooter

- 취약점 자동 탐색 + 루트킷 자동 설치까지 자동화

#### Kit (공격킷)

- 악성코드 자동 생성 툴
- 비전문가도 악성코드 제작 가능하게 함

### 정보 수집/유출

#### 키로거 (Keylogger)

- 타이핑 내용 몰래 기록
- 비밀번호, 카드번호 등 탈취

#### 스파이웨어 (Spyware)

- 사용자 행동 감시 후 외부로 전송
- 번들(무료 프로그램에 끼워서) 설치되는 경우 많음

#### 애드웨어 (Adware)

- 강제 광고 노출 목적
- 스파이웨어 기능 내장된 경우 많음 (광고 타겟팅 위해 행동 감시)

| | 스파이웨어 | 애드웨어 |
|---|---|---|
| 목적 | 정보 훔침 | 광고 강제 노출 |
| 수익구조 | 훔친 정보 판매 | 광고 클릭 수익 |

#### 크라임웨어 (Crimeware)

- 금융정보 탈취 목적
- 금전적 이득이 최종 목표

### 기타

#### 다이얼러 (Dialer)

- 몰래 유료 전화 연결
- 전화 요금 폭탄 유발

#### 조크 (Joke)

- 실제로 실행되는 **장난 프로그램** (무섭게 생긴 화면 등)

#### Hoax

- 프로그램이 아님. **가짜 바이러스 경고 메시지** ("당신 PC 감염됨!" 같은 허위 이메일)

> **조크 vs Hoax**
> - 조크 = 실제로 실행되는 장난 프로그램
> - Hoax = 프로그램도 없이 그냥 허위 경고 문자/메일

## 4. 안티바이러스 방법

### 탐지 3단계

1. **탐지(Detection)**: 바이러스 있는지 판단하고 위치 파악
2. **식별(Identification)**: 어떤 바이러스인지 특정
3. **제거(Removal)**: 모든 감염 시스템에서 제거

### 필터링 방법 비교

| 방법 | 탐지 방식 | 장점 | 단점 |
|---|---|---|---|
| **Signature Scanning** | 알려진 바이러스 시그니처(패턴) 검색 | 정확도 높음 | 신종/변종 대응 불가 |
| **Behavioral Scanning** | 이상한 행동 패턴 감지 | 신종·Zero-day 대응 가능 | 오탐 가능성 있음 |

### 웜 방어 — 네트워크 모니터링

- **Inbound 모니터링**: 외부 → 내 시스템으로 들어오는 트래픽 감시 (침입 방지)
- **Outbound 모니터링**: 내 시스템 → 외부로 나가는 트래픽 감시 (이미 감염됐는지 확인)

> 웜은 들어오는 것도 문제지만, 감염 후 다른 시스템으로 퍼뜨리는 것도 막아야 하므로 **양방향 모니터링** 필요

## 5. HTTP 트랜잭션

### 트랜잭션이란?

**요청(Request) + 응답(Response)** 한 쌍 = 트랜잭션 1번

### HTTP 버전별 연결 방식

| 버전 | 연결 방식 | 특징 |
|---|---|---|
| **HTTP 1.0** | 비영속적 연결 | 요청마다 새로 연결 → 느림 |
| **HTTP 1.1** | 영속적 연결 | 한 번 연결 후 계속 사용 → 빠름 |

> **비유**
> - HTTP 1.0: 편의점에서 물건 하나 살 때마다 매번 새로 줄 서기
> - HTTP 1.1: 한 번 줄 서서 여러 개 한꺼번에 사기

### 특수 문자

| 기호 | 이름 | 의미 |
|---|---|---|
| SP | Space | 띄어쓰기 (0x20) |
| CR | Carriage Return | 커서를 줄 맨 앞으로 (0x0D) |
| LF | Line Feed | 커서를 한 줄 아래로 (0x0A) |
| **CR+LF** | 줄바꿈 | HTTP 공식 줄바꿈 표준 |

## 6. HTTP 요청 메시지

### 구조

```
[요청 라인]  GET /index.html HTTP/1.1 CR+LF
[헤더]       Host: www.naver.com CR+LF
[헤더]       Accept: text/html CR+LF
[헤더]       Accept-Language: ko CR+LF
[헤더]       Accept-Encoding: gzip CR+LF
[헤더]       Connection: keep-alive CR+LF
[빈 줄]      CR+LF              ← 헤더 끝 신호
[본문]       (GET은 보통 없음)
```

### 요청 라인 구조

```
GET        /index.html      HTTP/1.1
 ↑              ↑               ↑
메서드(SP)  요청 URL(SP)    HTTP 버전
```

### 주요 헤더

| 헤더 | 의미 |
|---|---|
| `Host` | 요청하는 서버 도메인 |
| `Accept` | 받을 수 있는 콘텐츠 형식 (예: text/html) |
| `Accept-Language` | 원하는 언어 (예: ko) |
| `Accept-Encoding` | 압축 방식 (예: gzip) |
| `Connection` | 연결 유지 여부 (keep-alive / close) |

> **Accept 헤더란?**
> 브라우저가 서버한테 "나 이런 형식 받을 수 있어" 라고 미리 알려주는 것
> 음식점에서 "저 채식주의자라 고기 빼주세요" 미리 말하는 것과 같음

### HTTP 메서드

| 메서드 | 의미 |
|---|---|
| GET | 데이터 조회 |
| POST | 데이터 전송/생성 |
| PUT | 데이터 수정 |
| DELETE | 데이터 삭제 |

## 7. HTTP 응답 메시지

### 구조

```
[상태 라인]  HTTP/1.1 200 OK CR+LF
[헤더]       Content-Type: text/html; charset=utf-8 CR+LF
[헤더]       Content-Length: 1024 CR+LF
[헤더]       Date: Thu, 26 Mar 2026 09:00:00 GMT CR+LF
[헤더]       Server: nginx CR+LF
[헤더]       Connection: keep-alive CR+LF
[헤더]       Set-Cookie: session=abc123 CR+LF
[빈 줄]      CR+LF              ← 헤더 끝 신호
[본문]       <html><body>안녕하세요</body></html>
```

### 상태 라인 구조

```
HTTP/1.1    200    OK
   ↑          ↑     ↑
HTTP 버전  상태코드  상태 설명
```

### 상태 코드 5가지 분류

| 분류 | 의미 | 주요 코드 |
|---|---|---|
| **1xx** | 정보 | 100 Continue |
| **2xx** | 성공 | 200 OK, 201 Created |
| **3xx** | 리다이렉트 | 301 Moved Permanently, 302 Found |
| **4xx** | 클라이언트 오류 | 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found |
| **5xx** | 서버 오류 | 500 Internal Server Error, 503 Service Unavailable |

> **암기 팁**: 4xx = 내(클라이언트) 잘못 / 5xx = 서버 잘못

### 주요 응답 헤더

| 헤더 | 의미 |
|---|---|
| `Content-Type` | 본문 형식 (예: text/html) |
| `Content-Length` | 본문 크기 (바이트) |
| `Date` | 응답 시각 |
| `Server` | 서버 소프트웨어 정보 |
| `Set-Cookie` | 클라이언트에 쿠키 설정 |
