# 암호학 개요

## 1. 암호학(Cryptology)

암호학은 **정보를 안전하게 보호하기 위한 학문**

구성

```
암호기술 (Cryptography)
암호해독 (Cryptanalysis)
```

- **암호기술** : 데이터를 암호화하는 기술
- **암호해독** : 암호를 풀려고 하는 기술

---

# 2. 암호에서 사용하는 등장 인물

암호학에서는 이해를 쉽게 하기 위해 **가상의 인물**을 사용한다.

| 이름 | 의미 |
|---|---|
| Alice | 송신자 |
| Bob | 수신자 |
| Eve | 도청자 (소극적 공격자) |
| Mallory | 악의적인 공격자 (적극적 공격) |
| Trent | 신뢰기관 |
| Victor | 검증자 |

핵심

```
Alice → Bob (통신)
Eve → 도청
Mallory → 공격
```

Alice와 Bob은 **1979년 Ron Rivest**가 처음 사용

---

# 3. 송신자, 수신자, 도청자

도청자(Eavesdropper)는 반드시 **사람만 의미하지 않는다**

예시

- 도청 장비
- 해킹 프로그램
- 패킷 스니퍼

---

# 4. 암호화와 복호화

암호화는 **평문을 암호문으로 변환하는 것**

복호화는 **암호문을 평문으로 되돌리는 것**

암호의 목적

```
기밀성 (Confidentiality)
```

---

## 암호화 / 복호화 수식

암호화

```
C = E_k(P)
```

복호화

```
P = D_k(C)
```

의미

| 기호 | 의미 |
|---|---|
| P | 평문 (Plaintext) |
| C | 암호문 (Ciphertext) |
| E | 암호화 알고리즘 |
| D | 복호화 알고리즘 |
| k | 키 |

---

# 5. 키 스페이스 (Key Space)

키가 가질 수 있는 **모든 경우의 수**

키 스페이스가 크면

```
무차별 대입 공격 (Brute Force Attack)
```

이 어려워진다.

핵심

```
키 공간이 클수록 안전하다
```

---

# 6. 암호 알고리즘과 키의 분리

암호 시스템에서 **변경 가능한 부분은 키**

```
알고리즘 → 공개
키 → 비밀
```

이 원칙을 사용한다.

---

# 7. 대칭키와 비대칭키

## 대칭키 암호

하나의 키 사용

```
암호화 키 = 복호화 키
```

문제

```
키 배송 문제
```

---

## 비대칭키 암호

두 개의 키 사용

```
공개키 (Public Key)
개인키 (Private Key)
```

특징

```
수학적 원리 사용
```

---

## 키 종류 정리

| 키 | 의미 |
|---|---|
| Secret Key | 비밀키 |
| Shared Key | 공유키 |
| Session Key | 세션키 |
| Public Key | 공개키 |
| Private Key | 개인키 |

# 암호와 보안 상식

## 1. 알고리즘을 숨기는 보안은 위험

```
Security by Obscurity
```

숨겨서 보안을 유지하려는 방식

→ **좋은 보안 방식이 아님**

---

## 2. 약한 암호는 사용하지 않는 것보다 위험

이유

```
잘못된 안전감 제공
```

---

## 3. 모든 암호는 언젠가 해독될 수 있다

그래서 중요한 것

```
시간 vs 보호 가치
```

암호는 **공격자가 풀기 어렵게 만드는 것**

---

## 4. 암호는 보안의 일부일 뿐

보안에서 가장 약한 부분

```
사람
```

예시

```
사회공학 공격
```

---

# 암호 기법 분류

## 1. 치환 암호 (Substitution Cipher)

문자를 **다른 문자로 대체**

```
값을 바꿈
```

특징

```
평문 문자 ≠ 암호문 문자
```

예시

```
A → D
B → F
```

---

## 2. 전치 암호 (Transposition Cipher)

문자의 **위치만 변경**

```
자리를 바꿈
```

특징

```
평문 문자 = 암호문 문자
```

예시

```
HELLO → LHOEL
```

---

# 블록 암호와 스트림 암호

## 블록 암호 (Block Cipher)

데이터를 **블록 단위로 암호화**

특징

- 일정한 크기 단위 처리
- Round 구조 사용
- 높은 확산

예시

```
DES
AES
```

---

## 스트림 암호 (Stream Cipher)

데이터를 **순차적으로 암호화**

특징

- XOR 연산 사용
- 내부 상태 존재
- 실시간 통신에 적합

예시

```
RC4
```

---

## 블록 암호 vs 스트림 암호

| 구분 | 특징 |
|---|---|
| 블록 암호 | 한 번에 처리 |
| 스트림 암호 | 순차 처리 |

---

# 위치에 따른 암호화

## 링크 암호화 (Link Encryption)

각 네트워크 구간에서 암호화

특징

- 라우터에서 복호화 후 다시 암호화
- 홉(hop) 단위

---

## 종단간 암호화 (End-to-End Encryption)

송신자와 수신자 사이만 암호화

특징

```
중간 장비에서 복호화 없음
```

계층

```
Application Layer (L7)
```

---

## 링크 암호화 vs 종단간 암호화

| 구분 | 특징 |
|---|---|
| 링크 암호화 | 네트워크 구간마다 암호화 |
| 종단간 암호화 | 송신자와 수신자만 암호화 |

---

# 암호 시스템 구현 방식

## 하드웨어 암호 시스템

예시

```
AES 암호 칩
```

특징

- 빠름
- 보안성 높음

---

## 소프트웨어 암호 시스템

프로그램으로 암호화 수행

특징

- 유연함
- 상대적으로 느림
