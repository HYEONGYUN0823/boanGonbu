# 🖥️ 보안 운영체제 & TCB & TPM

---

## 📌 먼저 읽기! 핵심 개념 관계도

```
참조모니터 (추상적 개념)
    ↓ 구현
보안커널 (H/W + F/W + S/W)
    ↓ 구성
TCB (신뢰 컴퓨팅 기반)
    ↓ 평가
CC (국제 공통 평가 기준)
```

> 💡 **비유 : 건물 보안 시스템**
> 참조모니터 = 보안 규칙 (출입 통제 정책)
> 보안커널   = 실제 경비원 + 자물쇠 + CCTV (정책을 실제로 구현)
> TCB        = 건물 전체 보안 시스템 (보안커널 포함한 모든 것)

---

## 1. 보안 운영체제 개요

```
기존 운영체제 내에 보안커널을 추가로 이식한 운영체제
목적 : 해킹으로부터 시스템 보호

대표 예시
SELinux = MAC이 적용된 보안 운영체제
Trusted Solaris = MAC 기반
```

---

## 2. 보호 대상 및 방법

### 보호 대상

```
1. 메모리
2. 보조기억장치의 파일/데이터
3. 메모리상에서 실행 중인 프로그램
4. 파일들의 디렉터리
5. 하드웨어 장치
```

### 보호 방법 (구현 복잡도 순)

| 방법 | 설명 | 복잡도 |
|---|---|---|
| 물리적 분리 | 별도의 장비만 사용 | 낮음 |
| 시간적 분리 | 동일 시간에 하나씩만 실행 | ↑ |
| 논리적 분리 | 논리적인 구역을 지정 | ↑ |
| 암호적 분리 | 암호화하는 방법 | 높음 |

---

## 3. 파일시스템 보호

### 파일 보호 기법 ⭐⭐⭐

| 기법 | 설명 |
|---|---|
| **이름 명명 (Naming)** | 다른 사용자의 파일명을 알거나 추측하기 어렵다는 가정에 근거 |
| **패스워드** | 파일마다 서로 다른 패스워드를 설정 → 알아야만 접근 가능 |
| **암호화** | 파일 내용을 암호화 → 인가된 사용자(키 보유자)만 내용 파악 가능 |
| **ACL** | 접근제어 목록으로 권한 관리 |

> ⚠️ **파일 압축은 파일 보호 기법이 아님!**

## 4. 보안 운영체제의 보안 기능 (보안 요구사항)

```
1. 사용자 식별 및 인증
2. DAC (임의적 접근통제) - 신분기반 정책
3. MAC (강제적 접근통제) - 규칙기반 정책
4. 객체 재사용 보호
5. 완전한 조정
6. 신뢰 경로
7. 감사 및 감사 기록 축소
```

### 객체 재사용 (Object Reuse) 보호

> 💡 **비유 : 중고 노트**
> 이전에 쓴 내용이 지워지지 않은 노트를 다른 사람이 쓰면
> 이전 내용이 노출될 수 있음!

```
할당되는 기억공간에 이전 데이터가 남아있을 수 있음
→ 비밀 데이터 노출 위험
→ 디지털 포렌식과 관련
```

### 완전한 조정

```
임의적/강제적 접근통제가 효과적이려면
→ 모든 접근을 빠짐없이 통제해야 함
```

### 신뢰 경로 (Trusted Path)

```
패스워드 설정, 접근 허용 변경 등 보안 관련 작업 시
안전한 통신 경로(신뢰 경로)를 제공해야 함
TCB와 관련됨
```

### 감사 및 감사 기록 축소

```
로그(감사 기록)는 양이 매우 많음
→ 감사 기록 축소 기능 필요 (기술적 보안)
```

## 5. 보안커널 (Security Kernel)

> 💡 **비유 : 건물 경비원**
> 참조모니터(보안 규칙)를 실제로 구현해서 집행하는 존재
> 모든 출입을 중재하고 감시

```
구성 : TCB 내의 H/W + F/W + S/W
역할 : 참조모니터 개념을 구현하고 집행
       주체와 객체 사이의 모든 접근과 기능을 중재
```

### 보안커널 3가지 요구사항

| 요구사항 | 설명 |
|---|---|
| **격리성 (Isolation)** | 참조모니터 프로세스를 위한 분리 제공, 변조 방지 |
| **완전성 (Completeness)** | 모든 접근 시도에 실행, 우회 불가능, 오류 없이 구현 |
| **검증가능성** | 충분히 시험되고 검증 가능해야 함 |

---

## 6. 참조모니터 & 보안커널 & TCB ⭐⭐⭐⭐⭐

### 참조모니터 (Reference Monitor)

```
= 주체의 객체에 대한 모든 접근통제를 담당하는 추상머신 ⭐⭐⭐
= 개념적인 존재 (실제 구현체가 아님!)

역할 : 객체에 대한 주체의 모든 접근을 중재
       SKDB(보안커널 데이터베이스)를 참조해서 접근허가 결정

3가지 조건
✅ 부정조작이 없어야 함 → 격리성
✅ 항상 호출되어야 함 (우회 불가) → 완전성
✅ 분석/테스트로 검증 가능해야 함 → 검증가능성
```

### 세 개념의 관계

```
참조모니터 (추상적 개념)
       ↓ 실제로 구현한 것
보안커널 (H/W + F/W + S/W)
       ↓ 보안커널로 구성된 것
TCB (신뢰 컴퓨팅 기반)
       ↓ 평가 기준
CC (Common Criteria, 국제 공통 평가 기준)
```

### TCB (신뢰 컴퓨팅 기반)

> 💡 **비유 : 건물 전체 보안 시스템**
> 경비원(보안커널) + CCTV + 자물쇠 + 출입카드 시스템...
> 이 모든 보안 메커니즘의 총체 = TCB

```
= 컴퓨터 시스템 내 모든 보호 메커니즘의 총체
= 보안 정책을 적용할 책임을 가짐

구성 : H/W + F/W + S/W
```

### 보안 경계 (Security Perimeter)

```
신뢰(TCB)와 비신뢰 구성요소의 경계

신뢰(TCB) 영역         | 보안 경계 | 비신뢰 영역
H/W 권한               |          | 사용자 응용
(입출력, 클럭,         |          | 유틸리티
 레지스터, 메모리)     |          | 사용자 프로세스

⚠️ 보안 경계는 물리적 실체가 아닌 개념적(가상의) 경계선!
```

## 7. 신뢰 플랫폼 모듈 (TPM) ⭐⭐⭐

> 💡 **비유 : 금고 안의 금고**
> 컴퓨터 안에 물리적으로 박힌 아주 작은 보안 전용 칩
> 해킹해도 쉽게 변조할 수 없음 (훼손 방지!)

```
표준화 : TCG(Trusted Computing Group) 산업체 컨소시엄
구현   : 주로 하드웨어 칩 (소프트웨어로도 구현 가능)
핵심   : 훼손 방지 (Tamper Protected = Tamper Proof)
```

### TPM의 신뢰 관련 연산 ⭐⭐⭐⭐⭐

```
(a) 암호화 키의 생성과 저장
(b) 패스워드의 저장
(c) 무결성 검증을 위한 측정값 저장 (플랫폼 + 프로그램 대상)
(d) 디지털 인증서 관련 신뢰 연산 제공
```

### TPM 기본 서비스 ⭐⭐⭐⭐⭐

| 서비스 | 설명 |
|---|---|
| **인증된 부트 서비스** | 운영체제를 단계적으로 부팅 → OS 각 부분이 **승인된 버전임을 보장** |
| **인증 서비스** | TPM의 **사설키**로 설정 정보에 **서명** → 디지털 인증 생성 |
| **암호화 서비스** | 특정 기계가 특정 설정일 때만 복호화 가능하도록 **암호화 지원** |

### TPM 구성요소 ⭐⭐⭐

| 구성요소 | 설명 |
|---|---|
| **HMAC 엔진** | 다양한 인증 프로토콜에서 사용 |
| **전원 탐지** | TPM 전원 상태와 플랫폼 전원 상태 관리 |
| **Opt-in** | 사용자 의도대로 TPM 활성화/비활성화 (사전동의) |
| **비휘발성 메모리** | TPM 신원, **상태 인자 저장** (전원 꺼져도 유지) |
| **휘발성 메모리** | **임시 인자 저장** (현재 TPM 상태, 암호화키, 세션 정보) |

## 📌 최종 핵심 암기 요약

```
✅ 참조모니터 → 보안커널 → TCB → CC 순서로 구체화

✅ 참조모니터
   = 추상머신 (개념!) ⭐⭐⭐
   = 모든 접근 중재
   = SKDB 참조해서 접근허가 결정
   3조건 : 격리성 / 완전성 / 검증가능성

✅ 보안커널
   = 참조모니터를 실제로 구현한 것 (H/W + F/W + S/W)
   = 3요구사항 : 격리성 / 완전성 / 검증가능성

✅ TCB
   = 컴퓨터 시스템 내 모든 보호 메커니즘의 총체
   = 보안 정책 적용 책임
   보안 경계 = 신뢰/비신뢰 구성요소의 가상 경계선

✅ 파일 보호 기법
   이름 명명 / 패스워드 / 암호화 / ACL
   ❌ 파일 압축은 보호 기법 아님!

✅ 보호 방법 복잡도 순서
   물리적 → 시간적 → 논리적 → 암호적 (복잡도 증가)

✅ TPM
   = 하드웨어 칩 (훼손 방지!)
   4가지 연산 : 키생성/저장, 패스워드 저장, 무결성 측정값 저장, 인증서 연산
   3가지 서비스 : 인증된 부트 / 인증 / 암호화
   비휘발성 메모리 = 상태 인자 저장 (영구)
   휘발성 메모리   = 임시 인자 저장 (일시)
```
