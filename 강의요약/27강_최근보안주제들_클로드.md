## 1. 랜섬웨어

### 📌 정의
> **Ransom(몸값) + Ware(소프트웨어)**  
> 피해자의 파일을 암호화한 뒤, 복호화 키를 대가로 금전을 요구하는 악성코드

```
감염
 ↓
PC의 모든 파일이 암호화됨 (열 수 없어짐)
 ↓
"비트코인 X개 보내면 복호화 키 줄게" 메시지 출력
 ↓
돈 안 내면 파일 영구 삭제 or 유출 협박
```

---

### 📌 특징

| 특징 | 설명 |
|------|------|
| **강력한 암호화** | AES, RSA 등 강력한 알고리즘 사용 → 키 없이 복호화 불가 |
| **Tor 네트워크 이용** | 익명 네트워크로 해커 추적 극도로 어려움 |
| **암호화폐 요구** | 비트코인, 모네로 등 → 계좌 추적 불가 |
| **즉각적·가시적 피해** | 감염 즉시 피해자가 인지함 |

**Tor 네트워크란?**
```
일반 인터넷:  접속자 IP 추적 가능 → 해커 신원 노출
Tor 네트워크: 여러 노드를 거쳐 IP 익명화 → 추적 거의 불가능
```

---

### 📌 일반 악성코드 vs 랜섬웨어 차이점

| 구분 | 일반 악성코드 | 랜섬웨어 |
|------|------------|---------|
| **목적** | 정보 탈취, 시스템 파괴 | 금전 갈취 |
| **활동 방식** | 백그라운드에서 조용히 활동 | 즉각적·가시적 피해 |
| **피해자 인지** | 모르는 경우 많음 | 화면에 메시지 띄워 즉시 알림 |
| **복구 가능성** | 백신으로 제거 가능 | 복호화 키 없으면 복구 불가 |

---

### 📌 주요 감염 경로

#### ① 신뢰할 수 없는 사이트 — 드라이브 바이 다운로드 (Drive-by Download)
> 악성 사이트에 **접속만 해도** 자동으로 악성코드가 설치되는 기법

```
악성 사이트 접속
 ↓
브라우저/플러그인 취약점 자동 악용
 ↓
사용자 모르게 랜섬웨어 자동 다운로드·실행
```
- 클릭 한 번 안 해도 감염 가능
- 최신 OS·브라우저 업데이트로 예방

#### ② 스팸메일 및 스피어 피싱
```
"택배 배송 확인하세요.exe" 첨부파일
"카드 명세서 확인.doc" (매크로에 악성코드 삽입)
```

#### ③ 파일 공유 사이트
- 크랙 파일, 불법 소프트웨어에 랜섬웨어 숨겨서 배포

#### ④ SNS (사회관계망 서비스)
- 단축 URL로 악성 링크 유포
- DM으로 악성 첨부파일 전송

---

### 📌 랜섬웨어 공격 및 대응 절차

```
[공격 흐름]
감염 → 파일 암호화 → 몸값 요구 → 협상/지불 or 포기

[대응 절차]
① 즉시 네트워크 차단 (다른 PC로 전파 방지)
② 감염된 PC 격리
③ 백신/포렌식으로 감염 경로 파악
④ 백업 데이터로 복구 시도
⑤ 복호화 툴 확인 (No More Ransom 사이트)
⑥ 기관 신고 (KISA 등)
```
> ⚠️ **몸값 지불 비권장** — 지불해도 키를 안 주는 경우 다수 존재

---

### 📌 랜섬웨어 피해 예방 5대 수칙 (KISA)

| 수칙 | 내용 |
|------|------|
| ① **백업** | 중요 파일 정기 백업 (외장하드, 클라우드) |
| ② **OS·SW 업데이트** | 취약점 패치로 드라이브 바이 다운로드 차단 |
| ③ **백신 설치·업데이트** | 최신 랜섬웨어 패턴 대응 |
| ④ **출처 불명 파일·링크 클릭 금지** | 스팸메일, 불법 파일 주의 |
| ⑤ **이메일 첨부파일 주의** | 업무 위장 피싱 주의 |

## 2. APT 공격

### 📌 기본 개념
> **Advanced Persistent Threat**  
> 특정 목표를 향해 오랫동안 조용히 파고드는 **맞춤형 고급 지속 위협**

| 키워드 | 의미 |
|--------|------|
| **Advanced (고급)** | 정교하고 복잡한 기법 사용, 맞춤형 악성코드 제작 |
| **Persistent (지속적)** | 몇 달~몇 년간 장기간 잠복 (낮고 느리고 지속적인 공격) |
| **Threat (위협)** | 구체적인 목표를 가진 실제 공격자 집단 |

**일반 해킹 vs APT 비교**
```
일반 해킹:  불특정 다수를 향해 총질 (광범위, 비정교)
APT 공격:   특정 한 명을 몇 달에 걸쳐 저격 (맞춤형, 정교)
```

---

### 📌 공격 단계

```
① 선행조사 → ② 침투 → ③ 검색 → ④ 수집 → ⑤ 유출
```

| 단계 | 내용 |
|------|------|
| **선행조사** | 목표 조직 구성원, 시스템 구조, 취약점 파악 |
| **침투** | 스피어피싱 등으로 내부 발판 최초 확보 |
| **검색** | 내부 네트워크 탐색, 권한 상승 |
| **수집** | 목표 데이터 찾아서 모으기 |
| **유출** | 외부로 데이터 빼돌리기 |

---

### 📌 주요 침투 기법

#### ① 스피어 피싱 (Spear Phishing)
> **"작살 낚시"** — 불특정 다수가 아닌 딱 한 명을 정조준

```
일반 피싱:    "당신의 계정이 위험합니다" (누구에게나 뿌림)
스피어 피싱:  "김형윤 대리님, 지난번 프로젝트 계약서 확인 부탁드립니다"
              (이름, 직책, 업무 내용까지 맞춤 제작)
```
- 개인 맞춤형이라 속아 넘어갈 확률이 훨씬 높음

#### ② 드라이브 바이 다운로드 (Drive-by Download)
> 목표 조직 구성원이 **자주 방문하는 사이트를 미리 해킹**, 접속 즉시 악성코드 자동 감염

```
공격자: "A회사 직원들이 자주 가는 사이트가 어디지?" → 사전 조사
 ↓
해당 사이트 침해 후 악성코드 심어놓기
 ↓
직원이 평소처럼 접속하는 순간 → 자동 감염
```

#### ③ 워터링 홀 (Watering Hole)
> **"물웅덩이 근처에서 매복하는 포식자"**  
> 표적 집단이 반드시 방문하는 사이트에 함정을 설치하는 표적 공격의 일종

```
사자가 물웅덩이 옆에서 기다림
= 해커가 목표 조직이 반드시 방문할 사이트에 악성코드 심어놓기
```
- 드라이브 바이 다운로드와 유사하나, **특정 집단을 노린다**는 점이 핵심

#### ④ USB 드롭 공격
> 악성코드가 삽입된 USB를 목표 조직의 이동 동선에 **방치**, 누군가 꽂는 순간 감염

```
공격자: 악성 USB를 회사 주차장/복도에 슬쩍 놓아둠
 ↓
호기심 많은 직원이 주워서 PC에 꽂음
 ↓
자동 실행 → 내부 네트워크 감염
```
> 💡 실제 사례: **스턱스넷(Stuxnet)** — 이란 핵시설을 USB로 공격한 역사적 사건

## 3. 사이버 킬 체인

### 📌 개념
> **록히드 마틴(Lockheed Martin)사**가 군사 개념을 사이버 보안에 적용한 공격 분석 프레임워크  
> **"침입 타격 순환 체계"**

**핵심 원리:**
> 7단계 중 **단 하나만 막아도** 공격 전체가 실패한다

```
정찰 → 무기화 → 전달 → 취약점악용 → 설치 → C&C → 목표실행
 ↑
여기서 막으면 뒤 단계 전부 무의미!
```

---

### 📌 7단계 모델 — **RWDEICA**

| # | 단계 | 영문 | 핵심 행동 | 방어 포인트 |
|---|------|------|---------|-----------|
| 1 | **정찰** | Reconnaissance | 목표 정보 수집 (SNS, 홈페이지, 이메일 등) | 공개 정보 최소화 |
| 2 | **무기화** | Weaponization | 취약점 + 악성코드 결합해서 무기 제작 | 취약점 패치 |
| 3 | **전달** | Delivery | 이메일·USB·웹으로 무기 유포 | 메일 필터링, 웹 차단 |
| 4 | **취약점 악용** | Exploitation | 취약점 실행, 코드 동작 | IDS/IPS 탐지 |
| 5 | **설치** | Installation | 백도어·RAT 설치로 거점 확보 | 백신, EDR |
| 6 | **명령 및 제어** | Command & Control | 공격자가 원격으로 조종 (C&C 서버) | C&C 서버 차단 |
| 7 | **목표 실행** | Actions on Objectives | 데이터 탈취·파괴·랜섬웨어 실행 | 데이터 암호화, 백업 |

> 💡 **암기 팁:** **R**econnaissance → **W**eaponization → **D**elivery → **E**xploitation → **I**nstallation → **C**ommand & Control → **A**ctions = **RWDEICA**

## 4. 공급망 공격 (Supply Chain Attack)

### 📌 개념
> 목표 조직을 직접 공격하기 어려우니, **그 조직이 신뢰하는 공급업체를 먼저 해킹**하는 방식

```
[직접 공격]
해커 ──공격──▶ 대기업 (보안 철저, 막힘 ❌)

[공급망 공격]
해커 ──공격──▶ 소프트웨어 공급업체 (보안 약함 ✅)
                    ↓
       공급업체가 대기업에 업데이트 배포
                    ↓
       대기업이 신뢰하고 설치 → 감염!
```

> 💡 **실제 사례: 솔라윈즈(SolarWinds, 2020)**  
> IT 관리 소프트웨어 회사 해킹 → 업데이트에 악성코드 삽입 → 미국 정부기관 포함 수천 곳 감염

- APT 공격 그룹들은 공급망 공격과 같이 **고도화된 방법**을 이용한다

---

### 📌 공급망 공격 유형

| 유형 | 설명 |
|------|------|
| **빌드/업데이트 인프라 변조** | 빌드 서버 해킹 → 정상 업데이트에 악성코드 삽입 |
| **인증서·개발계정 유출을 통한 변조** | 코드 서명 인증서 탈취 → 악성코드를 "정품"처럼 위장 |
| **하드웨어·펌웨어 변조** | 제조 단계에서 칩·펌웨어에 백도어 심기 |
| **악성코드 감염 제품 판매** | 처음부터 악성코드가 포함된 제품 유통 |

---

### 📌 대응 방안

#### S/W 공급업체의 대응 방안
- 개발자 PC 및 빌드/업데이트 서버의 **관리 강화**
- **SDLC 가이드라인**에 따른 S/W 업데이트 절차 준수
- **Secure SDLC** 가이드에 맞춰 보안성을 고려한 S/W 개발 및 유지보수 이행

#### 기업/기관의 대응 방안

**핵심: 가시성(Visibility) 확보**
```
어떤 S/W가 내 환경에 존재하는지 파악
 ↓
불필요하거나 출처 불명 S/W 즉시 제거
 ↓
EDR 솔루션으로 엔드포인트 이상 행위 실시간 탐지
```

| 대응 수단 | 설명 |
|---------|------|
| **가시성 확보** | 내부에 어떤 S/W가 있는지 파악하는 것이 우선 |
| **EDR 도입** | 백신보다 고급 — 행동 패턴 분석으로 미지의 위협도 탐지 |
| **소프트웨어 무결성 검증** | 업데이트 파일의 해시값·서명 검증 |

> 💡 **EDR이란?**  
> 백신: "알려진 악성코드 차단"  
> EDR: "행동 패턴 분석 → 알려지지 않은 위협도 탐지"

---

## 🔗 전체 흐름 연결

```
APT 공격자
 ↓
공급망 공격으로 소프트웨어에 백도어 삽입
 ↓
킬 체인 1단계(정찰) → 7단계(목표실행) 순차 진행
 ↓
최종 목적: 랜섬웨어 배포 or 기밀 탈취
 ↓
Tor 네트워크 + 암호화폐로 익명성 유지하며 몸값 요구
```
