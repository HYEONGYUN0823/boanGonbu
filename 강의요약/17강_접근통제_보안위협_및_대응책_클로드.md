# ⚠️ 접근통제 보안위협 및 대응책

---

## 1. 패스워드 크래커 ⭐⭐⭐

> 💡 **비유 : 자물쇠 따기**
> 사전공격 = 자주 쓰는 열쇠 모음집으로 하나씩 시도
> 무차별공격 = 가능한 모든 열쇠를 다 만들어서 시도
> 레인보우 테이블 = 미리 만들어둔 열쇠-자물쇠 대응표로 빠르게 찾기

### 사전 공격 (Dictionary Attack)

```
패스워드 사전 파일(단어 목록)을 이용해서 로그인 시도
소프트웨어 툴에 포함된 단어와 포착된 패스워드 해시값을 비교

✅ 개인 정보를 충분히 알고 있다면 매우 효율적
❌ 사전에 없는 복잡한 패스워드엔 무력
```

### 무차별 공격 (Brute-Force Attack)

```
= Exhaustive(전수) 공격
가능한 모든 문자 조합을 성공할 때까지 전부 시도

키 공간(Key Space = Key Domain)과 관련
→ 키 공간이 클수록 무차별 공격에 시간 오래 걸림

워다이얼링(Wardialing) : 소프트웨어로 수천 개 전화번호에 자동으로 전화
                         → 모뎀 번호를 찾아 방화벽 우회
```

### 크리덴셜 스터핑 (Credential Stuffing)

```
다른 곳에서 확보한 아이디/비밀번호를 임의의 다른 서버에 대입하는 공격
→ "어디선가 털린 내 비번이 다른 사이트 해킹에 사용됨"
→ 같은 비밀번호를 여러 사이트에 쓰면 위험한 이유!
```

### 하이브리드 공격 (Hybrid Attack)

```
사전 공격 + 무차별 공격을 결합한 가장 효과적인 방법

예) 사전 공격으로 "algisa"를 찾은 뒤
    무차별 공격으로 algisa1, algisa01, algisas1... 시도
```

### 레인보우 테이블 공격 (Rainbow Table Attack)

```
미리 계산해 둔 해시값 테이블로 패스워드를 역으로 찾는 공격

레인보우 테이블 구조
하나의 패스워드 → 변이 함수로 여러 패스워드 생성
→ 각 패스워드의 해시를 고리처럼 연결 → 체인(Chain)
→ 체인들의 집합 = 레인보우 테이블

✅ 대응 : 솔팅(Salting)으로 레인보우 테이블 무력화!
```

**윈도우 패스워드 해시 취약성**
```
LM 해시 → NTLM v1 → NTLM v2 (현재 권고)
LM(Lan Manager) = 윈도우에서 가장 취약한 인증 방식
```

## 2. 사회공학 (Social Engineering)

> 💡 **비유 : 사람의 심리를 이용한 사기**
> 기술적 취약점이 아니라 **사람의 심리**를 공격!
> "제일 약한 부분은 사람이다"

### 인간기반 사회공학 공격

| 공격 | 설명 | 대응 |
|---|---|---|
| **어깨 넘어 훔쳐보기** (Shoulder Surfing) | 뒤에서 화면/키보드 훔쳐보기 | 화면 보호 필름, 주의 |
| **쓰레기통 뒤지기** (Dumpster Diving) | 버린 문서에서 정보 수집 | 문서 파쇄 |
| **따라 들어가기** (Piggybacking/Tailgating) | 출입증 소지자 뒤에 붙어서 함께 입장 | **맨트랩(Mantrap)** 설치 |
| **협박 메일** (Blackmail) | 정보 폭로 협박으로 재물 갈취 | 신고 |
| **내부자 결탁** | 내부 직원이 공격자와 협력 | 직무분리, 직무순환 |

**대응책**
```
✅ 전화 통화 시 음성/내용이 이상하면 주의
✅ 임직원 보안인식 교육 주기적 실시
```

## 3. ICT 기반 사회공학 공격 ⭐⭐⭐

### 피싱 (Phishing)

```
= Private data(개인정보) + Fishing(낚시)
이메일/SMS에 가짜 링크를 첨부해서 가짜 사이트로 유도
→ 개인정보 입력하게 만들어 탈취

특징 : 사용자가 주의를 기울이면 탐지 가능
```

### 파밍 (Pharming)

```
정상 도메인 자체를 탈취해서 가짜 사이트로 리다이렉션(redirecting)
→ 정상 주소를 입력해도 가짜 사이트로 연결됨

특징 : 실제 도메인명이 그대로 사용됨 → 탐지 매우 어려움!
```

### 피싱 vs 파밍 비교

| 구분 | 피싱 | 파밍 |
|---|---|---|
| 주된 목적 | 가짜 사이트로 유도해 개인정보 탈취 | 가짜 사이트로 유도해 개인정보 탈취 |
| 공격 특징 | 이메일, SMS 등 첨부 링크를 통해 접속 유도 | 정상 도메인 입력만으로도 공격 가능 → **별도 유인매체 불필요** |
| 위험성 | 사용자가 주의를 기울이면 **공격 탐지 / 피해 방지 가능** | 실제 도메인명이 그대로 사용되므로 **공격 탐지 곤란** |

### 스미싱 (SMishing)

```
= SMS + Phishing
SMS로 악성 소프트웨어 설치를 유도
→ 설치된 악성 SW로 개인정보 탈취 / 소액결제 피해

대상 : 휴대폰 사용자
```

**ICT 기반 공격 대응책**
```
✅ 악성 소프트웨어 탐지 및 제거
✅ 접속 사이트 및 DNS 서버 진위 인증 확인
```

## 4. 은닉 채널 (Covert Channel)

> 💡 **비유 : 숨겨진 통신 경로**
> 정식으로 허가된 통신이 아닌, 기존 채널에 몰래 기생하는 통신
> = 스테가노그래피와 유사한 개념

```
정의 : 보안 메커니즘에 의해 통제되지 않는 정보 흐름
목적 : 허가되지 않은 방식으로 정보를 수신
결과 : 시스템 보안 정책 명백히 위반

BLP 모델의 대표적 약점이기도 함!
```

**대응책**
```
✅ 로그 분석
✅ HIDS(호스트 기반 IDS) 탐지 (VIP 지키는 청원경찰)
✅ 통신 대역폭에 대한 엄격한 제한
✅ 시스템 자원 분석
```

## 5. 방사 (Emanation)

> 💡 **비유 : 건물 밖에서 도청**
> 컴퓨터가 작동할 때 발생하는 전자기파를 건물 밖에서 가로채서
> 화면에 표시되는 내용을 알아내는 것

```
컴퓨터/장치에서 방출되는 전기적 신호를 가로채서
원래 데이터를 복원하는 공격 방법
```

**대응책 3가지**

| 방법 | 설명 |
|---|---|
| **템페스트 (TEMPEST)** | 차폐 물질로 방사 신호를 억제하는 기술 표준 (인증 필요) |
| **백색 잡음 (White Noise)** | 무작위 전기 신호를 의도적으로 방출 → 방사 신호를 방해 |
| **통제 구역 (Control Zone)** | 벽면에 특수 물질을 사용해서 전기 신호가 새어나가지 못하게 함 |

## 📌 최종 핵심 암기 요약

```
✅ 패스워드 공격 4종류
   사전 공격       : 단어 목록으로 비교
   무차별 공격     : 모든 조합 시도 (= Exhaustive 공격)
   레인보우 테이블 : 미리 계산된 해시 테이블 → 솔팅으로 방어!
   하이브리드      : 사전 + 무차별 결합 (가장 효과적)

✅ 크리덴셜 스터핑
   다른 곳에서 탈취한 ID/PW를 다른 사이트에 대입
   → 같은 비밀번호 여러 곳에 쓰면 위험!

✅ 사회공학 공격
   Piggybacking/Tailgating → 맨트랩으로 방어
   피싱 : 이메일/링크로 유도 → 주의하면 탐지 가능
   파밍 : 도메인 탈취 → 탐지 어려움!
   스미싱 : SMS로 악성 SW 설치 유도

✅ 은닉채널 대응 : HIDS + 대역폭 제한 + 로그 분석

✅ 방사 대응 3가지
   템페스트 (차폐) / 백색 잡음 (방해) / 통제 구역 (격리)
```
