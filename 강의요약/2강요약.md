# 기본 보안 용어 정리

## 1. 자산 (Asset, A)
보호해야 할 가치 있는 것

- 데이터
- 시스템
- 장비
- 정보

자산은 **자산 소유자(Data Owner)** 가 가치를 부여한 대상이다.

무형 자산도 포함된다.

예시
- 개인정보
- 기업 데이터
- 서버

---

## 2. 취약점 (Vulnerability, V)

보안이 약한 부분

예시

- 암호화 안된 데이터
- 약한 비밀번호
- 패치 안된 서버
- 접근 통제 미설정

쉽게 말하면

```
공격자가 이용할 수 있는 약점
```

---

## 3. 위협 (Threat, T)

자산에 피해를 줄 수 있는 **행동 또는 사건**

예시

- 해커 공격
- 자연재해
- 내부자 실수

---

### 위협 유형

| 공격 | 의미 | 영향 |
|---|---|---|
| 가로채기 | 정보 탈취 | 기밀성 |
| 가로막음 | 서비스 차단 | 가용성 |
| 변조 | 데이터 변경 | 무결성 |
| 위조 | 가짜 생성 | 무결성, 인증 |

---

### 위협의 종류

위협은 크게 2가지

```
자연적 위협
인간에 의한 위협
```

---

### 1. 자연적 위협

예시

- 화재
- 지진
- 홍수
- 태풍

---

### 2. 인간에 의한 위협

#### (1) 비의도적 위협
사람의 **실수 또는 태만**

예시

- 파일 삭제
- 잘못된 설정
- 패치 미적용

대책

```
보안 정책
교육
사전 점검
```

---

#### (2) 의도적 위협
의도적인 공격

예시

- 해킹
- 내부자 공격

대책

```
기술적 보안 대책
```

---

## 4. 위험 (Risk)

위험은 **피해가 발생할 가능성**

구성 요소

```
자산 × 위협 × 취약점
```

```
A × T × V
```

위험은 **가능성과 영향의 결합**

---

## 5. 노출 (Exposure)

공격에 노출되어 있는 상태

예시

- 서버가 인터넷에 그대로 공개
- 비밀번호가 노출
- 방화벽 없음

한 줄 정의

```
공격자가 취약점을 이용할 수 있는 상태
```

---

## 6. 대책 / 안전장치 (Control)

위험을 줄이기 위한 방어 방법

예시

- 강력한 패스워드 정책
- 방화벽
- 접근 통제
- 백업
- 암호화

한 줄 정의

```
위험을 줄이기 위한 조치
```

---

## 7. 사회공학 (Social Engineering)

사람을 속여서 정보를 얻는 공격

핵심

```
보안에서 가장 약한 부분은 사람
```

특징

- 인간의 신뢰 이용
- 기술이 아닌 심리 공격

예시

- 피싱
- 사칭 전화
- 택배 사기

---

# 집 비유로 이해하기

상황

```
내 집에 금고가 있음
```

---

## 자산 (Asset)

보호해야 할 것

예시

```
금고 속 현금
```

---

## 위협 (Threat)

피해를 줄 수 있는 존재

예시

```
도둑
```

---

## 취약점 (Vulnerability)

보안이 약한 부분

예시

- 창문 안 잠김
- CCTV 없음
- 비밀번호 1234

---

## 노출 (Exposure)

위협이 취약점을 이용할 수 있는 상태

예시

- 창문 열어둔 상태
- 비밀번호가 문 옆에 있음

즉

```
도둑이 바로 들어올 수 있는 상태
```

---

## 위험 (Risk)

실제로 피해가 발생할 가능성

예시

```
도둑이 금고를 털 가능성
```

---

## 대책 (Control)

위험을 줄이기 위한 조치

예시

- 창문 잠그기
- CCTV 설치
- 경비 시스템
- 강력한 비밀번호

---

## 사회공학

사람을 속여서 문을 열게 만드는 공격

예시

```
"택배 왔습니다"
```

내가 직접 문을 열어줌

---

# 시점별 통제

보안 통제는 **3가지 시점**이 있음

```
예방
탐지
교정
```

---

## 1. 예방 통제 (Preventive Control)

사고가 **발생하기 전에 막는 것**

예시

- 문 잠그기
- 경비원
- 강력한 비밀번호

---

## 2. 탐지 통제 (Detective Control)

공격 발생을 **발견**

예시

- CCTV
- 침입 탐지 시스템
- 접근 로그

---

## 3. 교정 통제 (Corrective Control)

사고 발생 후 **복구**

예시

- 백업 복구
- 보험
- 시스템 복구
- BCP / DRP

---

# 전체 흐름

```
자산 (금고)

↓

위협 (도둑)

↓

취약점 (문 안잠김)

↓

노출 (문 열림)

↓

위험 (털릴 가능성)

↓

대책 (잠금장치)
```

---

# 한 줄 핵심 정리

```
보안은 도둑이 못 들어오게 하거나,
들어와도 빨리 발견하고 피해를 줄이는 것이다.
```
