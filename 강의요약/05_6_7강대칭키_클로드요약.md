# 🔐 현대 대칭키 암호

---

## 📌 먼저 읽기! 자주 나오는 기호 설명

| 기호 | 의미 | 예시 |
|---|---|---|
| P | 평문 (Plaintext) = 원본 데이터 | P1 = 첫 번째 평문 블록 |
| C | 암호문 (Ciphertext) = 암호화된 데이터 | C1 = 첫 번째 암호문 블록 |
| j | 블록 번호 | Pj = j번째 평문 블록 |
| ⊕ | XOR = 비트 섞기 연산 | A ⊕ B |
| K | 키 (Key) | K1 = 첫 번째 라운드 키 |
| IV | 초기 벡터 = 암호화 시작할 때 쓰는 랜덤값 | - |

> 💡 **블록이 뭔데?**
> 데이터를 일정한 크기로 잘라낸 조각이야
> 예) 64bit 블록 암호라면 데이터를 64bit씩 잘라서 처리함
> 마치 긴 편지를 페이지 단위로 나눠서 암호화하는 것!

## 1. 블록 암호의 핵심 개념

### 혼돈과 확산 ⭐⭐⭐

블록 암호는 두 가지 핵심 개념으로 설계돼.

| 개념 | 목적 | 쉬운 설명 |
|---|---|---|
| 혼돈 (Confusion) | 암호문과 **키**의 관계를 숨김 | 키가 1비트만 바뀌어도 암호문이 완전히 달라짐 |
| 확산 (Diffusion) | 암호문과 **평문**의 관계를 숨김 | 평문의 패턴이 암호문 전체에 퍼져서 분석 불가능 |

> 💡 **비유로 이해하기**
> 대칭키 암호 = 스크램블 에그 만들기
> 달걀(평문)을 휘저어서(암호화) 원래 모양을 알아볼 수 없게 만드는 것!

---

## 2. P-Box (위치를 바꾸는 박스)

> 💡 **P-Box = 자리 바꾸기**
> 값은 그대로인데 위치만 섞는 것
> 예) ABCD → CDAB (값은 같고 순서만 바뀜)

| 종류 | 설명 | 역함수 | 비유 |
|---|---|---|---|
| 단순 P-Box | 비트 위치만 바꿈 | ✅ 있음 | 자리 바꿨다가 다시 돌아올 수 있음 |
| 축소 P-Box | 일부 비트가 **사라짐** | ❌ 없음 | 일부가 없어져서 원래대로 못 돌아옴 |
| 확장 P-Box | 비트 수가 **늘어남** | ❌ 없음 | 복사본이 생겨서 어떤 게 원본인지 모름 |

> ⚠️ 역함수가 없으면 복호화 불가능
> → 축소/확장 P-Box는 암호화 중간 과정에서만 사용

---

## 3. S-Box (값을 바꾸는 박스)

> 💡 **S-Box = 값 바꾸기**
> 치환 암호를 비트 단위로 구현한 것
> 예) 0101 → 1010 (자리가 아니라 값 자체가 바뀜)

**특징**
- 입력 비트 수 = 출력 비트 수
- 역함수가 있을 수도, 없을 수도 있음
- 목적 : **혼돈(Confusion)**

> 💡 **P-Box vs S-Box 차이**
> P-Box = 자리만 바꿈 → ABCD → CDAB
> S-Box = 값을 바꿈  → ABCD → XYZW

---

## 4. 블록 암호의 주요 연산

| 연산 | 하는 일 | 목적 | 비유 |
|---|---|---|---|
| Substitution (치환) | 값을 다른 값으로 바꿈 (S-Box) | 혼돈 | 단어를 다른 단어로 교체 |
| XOR (⊕) | 데이터와 키를 섞음 | 키를 암호문에 숨김 | 두 퍼즐을 겹쳐서 섞기 |
| Shift | 비트 위치를 이동 | 확산 | 줄 서있는 사람들 옆으로 한 칸씩 이동 |
| Swap | 왼쪽/오른쪽 블록 교환 | Feistel 구조에서 사용 | 왼손과 오른손에 든 것을 서로 교환 |

## 5. 합성 암호 ⭐⭐⭐

치환 + 전치 + 기타 연산을 **조합**한 복합 암호

> 💡 **Shannon이 만든 개념**
> - 정보이론 창시자
> - One-Time Pad 절대 안전성 증명
> - 혼돈/확산 개념 정의

## 6. 라운드 (Round)

> 💡 **라운드 = 같은 암호 과정을 반복하는 횟수**
> 마치 빨래를 한 번만 돌리는 것보다 여러 번 돌릴수록 더 깨끗해지는 것처럼
> 라운드가 많을수록 암호가 더 강해짐!
>
> 치환 → 섞기 → XOR = 1라운드

| 알고리즘 | 라운드 수 |
|---|---|
| DES | 16 |
| 3DES | 48 (DES 16라운드 × 3번) |

## 7. Feistel 구조 vs SPN 구조 ⭐⭐⭐⭐⭐

### Feistel 구조

> 💡 **비유 : 두 팀이 매 라운드마다 서로 영향을 주고받으며 자리를 바꾸는 것**

**동작 방식**
```
1. 데이터를 왼쪽(L) / 오른쪽(R) 으로 절반씩 나눔
2. 오른쪽이 왼쪽에 영향을 줌
3. 왼쪽/오른쪽 자리를 바꿈 (Swap)
4. 이걸 16번 반복 → 완전히 뒤섞임

[왼쪽 L] [오른쪽 R]
     ↓         ↓
     └────→ [섞기(F함수)]
                  ↓
[새 오른쪽 = L ⊕ F(R,K)] [새 왼쪽 = R]
     ↓                          ↓
           (이걸 16번 반복)
```

**핵심 특징**
- 복호화 구조 = 암호화 구조와 **동일**
- 키 순서만 **반대**로 사용
- 암호화 : K1 → K2 → ... → K16
- 복호화 : K16 → K15 → ... → K1

> ✅ **왜 좋냐?**
> 복호화할 때 새로운 구조를 만들 필요 없이
> 똑같은 구조에 키 순서만 뒤집으면 돼서 구현이 쉬움!

---

### SPN 구조

> 💡 **S-Box(값 바꿈) + P-Box(자리 바꿈) 를 반복하는 구조**
> Feistel처럼 절반으로 나누지 않고 전체를 한꺼번에 처리

```
전체 데이터
    ↓
[S-Box: 값 바꾸기]   ← 혼돈
    ↓
[P-Box: 자리 바꾸기] ← 확산
    ↓
[키 XOR]             ← 키 섞기
    ↓
(반복...)
```

**특징**
- 블록을 절반으로 나누지 않음
- 복호화 시 **역연산** 필요 (Feistel과 다름!)
- 역함수 있는 요소만 사용

---

### Feistel vs SPN 비교 ⭐⭐⭐⭐⭐

| 구분 | Feistel | SPN |
|---|---|---|
| 복호화 방식 | 암호화와 **동일** (키만 역순) | **역연산** 필요 |
| 블록 분할 | 절반으로 나눔 | 나누지 않음 |
| 대표 알고리즘 | DES, SEED, 3DES | AES, ARIA |

## 8. 대칭 블록 암호 구현 요소 ⭐⭐⭐⭐⭐

암호 강도를 결정하는 5가지 요소

```
1. 블록 크기    → 한 번에 처리하는 데이터 양 (클수록 좋음)
2. 키 길이      → 보통 128bit (길수록 안전)
3. 라운드 수    → 반복 횟수 (많을수록 안전)
4. 서브키 생성  → 원래 키에서 각 라운드용 키를 만드는 방식
                  예) DES : 56bit 원래 키 → 48bit 서브키 16개 생성
5. 라운드 함수  → 각 라운드에서 실제로 수행하는 암호 연산
```

## 9. 블록 암호 공격 방법 ⭐⭐⭐

| 공격 방법 | 쉬운 설명 | 공격 유형 |
|---|---|---|
| 차분 분석 | 평문을 살짝 바꿔보고 암호문이 어떻게 달라지는지 추적 | 선택 평문 공격 |
| 선형 분석 | 복잡한 암호를 단순한 수식(y=ax+b)으로 근사해서 키 추측 | 기지 평문 공격 |
| 전수 공격 | 가능한 키를 전부 다 시도 (키가 k비트면 2^k번 시도) | 무차별 공격 |
| 통계적 분석 | 문자 빈도 패턴으로 분석 (예: 영어에서 e가 가장 많이 나옴) | 통계 기반 |

> 💡 **DES는 선형분석으로 2^56 → 2^47 수준으로 줄었어**
> **AES는 차분/선형 분석 모두 대비해서 설계됨 → 안전!**

## 10. DES ⭐⭐⭐⭐⭐

> 💡 **DES = 1970년대 미국 표준 암호**
> 지금은 키가 너무 짧아서 안전하지 않음 → 사용 금지

### DES 핵심 숫자 (꼭 외워!)

| 항목 | 값 | 설명 |
|---|---|---|
| 블록 크기 | 64 bit | 한 번에 64bit씩 암호화 |
| 키 길이 | 56 bit | 실제 입력은 64bit, 8bit는 오류 검출용(패리티) |
| 라운드 수 | 16 | 암호화 과정을 16번 반복 |
| 서브키 수 | 16개 (각 48bit) | 각 라운드마다 다른 키 사용 |
| 구조 | Feistel | 암호화 = 복호화 구조 |

### DES 라운드 함수 구성

```
[32bit 입력]
    ↓
[확장 P-Box]  → 32bit를 48bit로 늘림
    ↓
[키 XOR]      → 48bit 서브키와 섞음
    ↓
[8개의 S-Box] → 48bit를 32bit로 줄이면서 값 변환
    ↓
[단순 P-Box]  → 자리 섞기
    ↓
[32bit 출력]
```

### DES 취약점

- 1998년에 **단 3일 만에** 전수 공격으로 해독됨
- 키가 56bit로 너무 짧음

> 💡 **쇄도효과 (Avalanche Effect)**
> 평문이나 키가 **1비트만 바뀌어도** 암호문이 절반 이상 달라지는 효과
> → 이 효과가 클수록 좋은 암호!
>
> 예) "안녕"  → 암호화 → "X7#@!K"
>     "안녕!" → 암호화 → "P2$KmQ" (완전히 다름!)

## 11. 3DES ⭐⭐⭐

> 💡 **DES가 뚫리니까 DES를 3번 반복해서 보안을 높인 것**
> 지금은 너무 느려서 거의 안 씀

**구조**
```
평문 → [암호화 K1] → [복호화 K2] → [암호화 K3] → 암호문
```

> ❓ **왜 중간에 복호화를 하냐?**
> K1 = K2 = K3 으로 같은 키를 쓰면 그냥 DES와 같아지게 하려고!
> (이전 DES 시스템과 호환 가능하게)

**속도 비교 (느린 순)**
```
3DES  <  DES  <  AES  <  RC4
(제일 느림)              (제일 빠름, 스트림 암호)
```

| 구분 | DES | 3DES (키 2개) | 3DES (키 3개) | AES |
|---|---|---|---|---|
| 블록 크기 | 64bit | 64bit | 64bit | 128bit |
| 키 길이 | 56bit | 112bit | 168bit | 128/192/256bit |

## 12. AES ⭐⭐⭐⭐⭐

> 💡 **현재 세계 표준 암호 알고리즘**
> 벨기에 암호학자(Joan Daemen, Vincent Rijmen)가 만든 Rijndael이 선정됨
> NIST가 15개 → 5개 → 최종 1개(Rijndael) 선정

### AES 핵심 숫자 (꼭 외워!)

| 키 길이 | 라운드 수 | 기억법 |
|---|---|---|
| 128 bit | 10 | 128 → 10 |
| 192 bit | 12 | 192 → 12 |
| 256 bit | 14 | 256 → 14 |

> 💡 **키가 길수록 라운드도 많아짐 = 더 안전함**

### AES 구조 (SPN)

```
[SubBytes]    → S-Box로 값 바꾸기 (혼돈)
     ↓
[ShiftRows]   → 행을 옆으로 밀기 (확산)
     ↓
[MixColumns]  → 열을 섞기 (확산)
     ↓
[AddRoundKey] → 라운드 키와 XOR (키 섞기)
     ↓
(10/12/14번 반복)

⚠️ 마지막 라운드에서는 MixColumns 없음!
```

### DES vs AES 비교 ⭐⭐⭐⭐⭐

| 항목 | DES | AES |
|---|---|---|
| 연도 | 1976 | 1999 |
| 블록 크기 | 64 bit | 128 bit |
| 키 길이 | 56 bit | 128 / 192 / 256 bit |
| 구조 | Feistel | SPN |
| 라운드 | 16 | 10 / 12 / 14 |
| 출처 | IBM, NSA | 벨기에 암호학자 |
| 현재 상태 | ❌ 사용 금지 | ✅ 현재 표준 |

## 15. 스트림 암호

> 💡 **블록 단위가 아니라 비트/바이트 단위로 순차 암호화**
> 실시간 통신에 적합 (무선, 모바일)
> 예) 라디오 방송처럼 데이터가 흘러가면서 실시간으로 암호화

**설계 조건**
```
1. 긴 주기       → 같은 패턴이 반복되면 안 됨
2. 진짜 난수처럼 → 0과 1의 개수가 비슷해야 함
3. 키 길이       → 최소 128bit 이상
```

| 종류 | 특징 | 비유 |
|---|---|---|
| 동기식 | 키 스트림과 암호문 **독립** → 정보 유출 적음 | 두 사람이 같은 악보를 보고 따로 연주 |
| 비동기식 | 키 스트림과 암호문 **종속** → 오류 전파 제한적 | 앞 사람 연주를 듣고 따라서 연주 |

### One-Time Pad ⭐⭐⭐

> 💡 **이론적으로 절대 해독 불가능한 암호**
> 키를 딱 한 번만 사용하고 버림
> 단점 : 키가 평문만큼 길어야 해서 실용적이지 않음

```
암호화 : 평문 ⊕ 키 = 암호문
복호화 : 암호문 ⊕ 키 = 평문
```

**절대적 안전성** (Shannon이 증명)

## 16. 블록 암호 운영 모드 ⭐⭐⭐

> 💡 **블록 암호를 실제로 어떻게 사용할지 정한 방식**
> NIST가 5가지 정의
> 같은 AES라도 어떤 모드를 쓰느냐에 따라 동작이 달라짐!

> 💡 **패딩(Padding)이란?**
> 블록 크기에 맞게 빈 공간을 채우는 것
> 예) 블록이 8칸인데 데이터가 5칸 → 나머지 3칸을 더미로 채움
> ECB, CBC만 패딩 필요!

> 💡 **IV(초기 벡터)란?**
> 암호화 시작할 때 사용하는 랜덤한 첫 번째 값
> 같은 평문이라도 IV가 다르면 암호문이 달라짐 → 보안 향상
> ECB는 IV 없음!

### 5가지 모드 한눈에 보기

| 모드 | 이름 | 패딩 필요 | IV 사용 | 오류 전파 | 특징 |
|---|---|---|---|---|---|
| ECB | 전자 부호표 | ✅ | ❌ | 없음 | 같은 평문 = 같은 암호문 (취약!) |
| CBC | 암호 블록 연쇄 | ✅ | ✅ | 2블록 | VPN, Kerberos에서 사용 |
| CFB | 암호 피드백 | ❌ | ✅ | 전파됨 | 스트림 암호처럼 동작 |
| OFB | 출력 피드백 | ❌ | ✅ | 없음 | 오류 전파 없음 |
| CTR | 카운터 | ❌ | ✅ | 없음 | 병렬처리 가능 (가장 빠름) |

---

### ECB 모드 (Electronic CodeBook)

> 💡 **각 블록을 독립적으로 암호화하는 가장 단순한 방식**

```
평문1 → [암호화] → 암호문1
평문2 → [암호화] → 암호문2
평문3 → [암호화] → 암호문3
(각각 따로따로, 연결 없음)
```

**단점** : 같은 평문이면 항상 같은 암호문!
예) "송금 1000원"이 여러 번 나오면 암호문도 똑같이 반복됨
→ 패턴 분석 가능 (트래픽 분석 취약)

---

### CBC 모드 (Cipher Block Chaining) ⭐⭐⭐

> 💡 **이전 암호문을 다음 블록에 섞어서 연결하는 방식**
> 앞 블록의 결과가 뒷 블록에 영향을 줌 → "체인처럼 연결"

```
      IV (랜덤 시작값)
       ↓
IV ⊕ 평문1(P1) → [암호화] → 암호문1(C1)
                                   ↓
               C1 ⊕ 평문2(P2) → [암호화] → 암호문2(C2)
                                                  ↓
                           C2 ⊕ 평문3(P3) → [암호화] → 암호문3(C3)
```

**오류 전파 설명**

예를 들어 3번째 블록(C3)에서 오류가 생겼다면?

```
C3에서 오류 발생
  ↓
P3 (3번째 평문) → 대부분의 비트 오류 (많이 망가짐)
P4 (4번째 평문) → 딱 1비트만 오류
P5 (5번째 평문) → 완전 정상! ✅
P6, P7 ...      → 모두 정상! ✅
```

**사용처** : IPSec(VPN), Kerberos v5 인증

---

### CFB 모드 (Cipher FeedBack) ⭐⭐⭐

> 💡 **블록 암호를 스트림 암호처럼 사용하는 방식**
> 이전 암호문을 암호화해서 키스트림을 만들고 평문과 XOR
> 패딩 없이 데이터를 즉시 전송 가능!

```
      IV
       ↓
    [암호화]
       ↓
키스트림1 ⊕ 평문1(P1) → 암호문1(C1)

       ↑ C1을 다시 입력으로 사용! (이전 암호문을 피드백)
       C1
       ↓
    [암호화]
       ↓
키스트림2 ⊕ 평문2(P2) → 암호문2(C2)

       C2
       ↓
    [암호화]
       ↓
키스트림3 ⊕ 평문3(P3) → 암호문3(C3)
```

> 핵심 : **이전 암호문(C)** 을 암호화해서 다음 키스트림을 만듦
> → 암호문이 다음 블록에 영향을 줌 → 오류 전파 발생!

**오류 전파 설명**

예를 들어 C2에서 오류가 생겼다면?

```
C2에서 오류 발생
  ↓
P2 (2번째 평문) → 같은 위치 1비트 오류
P3 (3번째 평문) → 약 50% 비트 오류 (C2가 키스트림 만들 때 사용됐으니까)
P4 (4번째 평문) → 정상! ✅ (C3는 오류 없으니까)
```

- 패딩 불필요
- **비동기식** 스트림 암호
- 복호화 시에도 **암호화 함수** 사용 (복호화 함수 아님!)

---

### OFB 모드 (Output FeedBack) ⭐⭐⭐

> 💡 **키스트림을 평문과 독립적으로 미리 생성해두는 방식**
> 오류가 퍼지지 않아서 위성통신 같은 곳에서 사용

```
      IV
       ↓
    [암호화]
       ↓
키스트림1 ────→ 키스트림1 ⊕ 평문1(P1) → 암호문1(C1)
       ↓
    ↑ 키스트림 자체를 다시 입력! (암호문이 아님!)
    [암호화]
       ↓
키스트림2 ────→ 키스트림2 ⊕ 평문2(P2) → 암호문2(C2)
       ↓
    [암호화]
       ↓
키스트림3 ────→ 키스트림3 ⊕ 평문3(P3) → 암호문3(C3)
```

> 핵심 : **키스트림끼리 연결** (평문/암호문과 무관!)
> → 암호문에 오류가 생겨도 키스트림은 영향 없음 → 오류 전파 없음!

**오류 전파 설명**

예를 들어 C2에서 오류가 생겼다면?

```
C2에서 오류 발생
  ↓
P2 (2번째 평문) → 같은 위치 1비트 오류
P3 (3번째 평문) → 완전 정상! ✅ (키스트림은 암호문과 무관하게 생성됐으니까)
P4, P5 ...      → 모두 정상! ✅
```

- **동기식** 스트림 암호
- 키스트림이 평문과 **무관**하게 생성됨

---

### CTR 모드 (CounTeR)

> 💡 **카운터 값을 암호화해서 키스트림을 만드는 방식**
> 각 블록마다 카운터를 1씩 증가시켜서 사용
> 모든 블록이 완전히 독립적 → 동시에 처리 가능!

```
카운터1        카운터2        카운터3
(nonce+1)     (nonce+2)     (nonce+3)
    ↓              ↓              ↓
 [암호화]       [암호화]       [암호화]
    ↓              ↓              ↓
키스트림1      키스트림2      키스트림3
    ↓              ↓              ↓
⊕ 평문1(P1)   ⊕ 평문2(P2)   ⊕ 평문3(P3)
    ↓              ↓              ↓
암호문1(C1)   암호문2(C2)   암호문3(C3)

(세 블록을 동시에 처리 가능! → 병렬처리)
```

> 핵심 : 각 블록이 **완전히 독립적** (카운터만 1씩 증가)
> → 순서 상관없이 동시에 처리 가능 → **병렬처리 가능!**

- **동기식** 스트림 암호
- **병렬 처리 가능** → 가장 빠름!

---

### CFB vs OFB 차이 한눈에 보기

```
CFB : 이전 [암호문(C)] → [암호화] → 키스트림 → ⊕ 평문
              ↑
         암호문을 피드백 → 오류 전파 있음!

OFB : 이전 [키스트림] → [암호화] → 키스트림 → ⊕ 평문
              ↑
         키스트림을 피드백 → 오류 전파 없음!
```

> 💡 **핵심 차이**
> CFB = 암호문을 다시 넣음 → 오류가 다음 블록에 영향
> OFB = 키스트림을 다시 넣음 → 암호문 오류가 다음 블록에 영향 없음

---

### 모드별 오류 전파 정리 ⭐⭐⭐⭐⭐

> 💡 **오류 전파 = 암호문 한 블록이 망가졌을 때 몇 개의 평문 블록이 영향을 받는가**

| 모드 | 오류 전파 범위 | 설명 |
|---|---|---|
| ECB | 해당 블록만 | 독립적이라서 다른 블록 영향 없음 |
| CBC | **Pj** 많이 망가짐 + **Pj+1** 1비트 오류 | 이전 암호문이 다음 블록 복호화에 사용됨 |
| CFB | **Pj** 1비트 오류 + **다음 블록들** 약 50% 오류 | 오류난 암호문이 키스트림 만들 때 사용됨 |
| OFB | 해당 블록만 | 키스트림이 독립적이라서 전파 없음 |
| CTR | 해당 블록만 | 카운터가 독립적이라서 전파 없음 |

## 📌 최종 핵심 암기 요약

```
✅ Feistel = 복호화 동일, 키만 역순 → DES, SEED
✅ SPN    = 복호화 역연산 필요     → AES, ARIA

✅ DES  : 블록 64bit  / 키 56bit            / 라운드 16
✅ AES  : 블록 128bit / 키 128·192·256bit   / 라운드 10·12·14
✅ ARIA : AES와 블록·키 동일               / 라운드 12·14·16 (AES+2)
✅ SEED : 블록 128bit / 키 128bit           / 라운드 16 / Feistel

✅ 패딩 필요      : ECB, CBC
✅ 병렬처리 가능  : CTR
✅ 오류 전파 없음 : OFB, CTR
✅ 속도 (느린 순) : 3DES < DES < AES < RC4
```
