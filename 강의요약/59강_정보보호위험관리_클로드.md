## 1. 위험관리 개요

### 📖 정의 ⭐⭐⭐

> **위험관리** = 조직 자산의 위험을 수용 가능한 수준으로 유지하기 위해  
> 위험을 분석하고, 비용 대비 효과적인 보호대책을 마련하는 **일련의 과정**

한마디로:

```
뭐가 위험한지 찾고 → 얼마나 위험한지 계산하고 → 적절히 막고 → 계속 점검하는 것
```

---

### 🔄 보안관리 활동 전체 흐름

```
┌─────────────────────────────┐
│  1. 보안정책 수립             │
│  "우리 회사 보안 규칙 정하기"  │
└──────────────┬──────────────┘
               ↓
┌─────────────────────────────┐
│  2. 위험관리 (위험분석)        │
│  뭐가 위험한지 분석            │
│  얼마나 위험한지 계산           │
└──────────────┬──────────────┘
               ↓
┌─────────────────────────────┐
│  3. 대응책 구현               │
│  방화벽 / 백신 / 접근권한 설정  │
└──────────────┬──────────────┘
               ↓
┌─────────────────────────────┐
│  4. 사후관리                  │
│  잘 막히는지 점검              │
│  새로운 위험 확인              │
└──────────────┬──────────────┘
               ↓
           ♻️ 반복
```

---

### 💡 위험관리가 필요한 이유

| 이유 | 설명 |
|------|------|
| **① 과잉/과소 투자 방지** | 작은 홈페이지에 10억 투자 → 과잉, 은행인데 보안 없음 → 과소. 필요한 곳에 적절히 투자 |
| **② 위험은 계속 변함** | 새 해킹기술 등장, 시스템/인원 변화 → 1년에 1~2회(상/하반기) 반복 점검 필요 |

> 💬 **핵심 포인트**: 위험분석 없이는 "어디에 얼마나 투자해야 하는지"를 알 수 없다

## 2. 위험의 구성요소

### 📐 위험 공식 ⭐⭐⭐

```
위험 = f(자산, 위협, 취약점)

위험 = 위협이 성공할 가능성 × 위협 성공 시의 손실 크기
```

---

### 🔗 구성요소 관계도

```
        [위협 (Threat)]
     해킹 / 랜섬웨어 / 화재
              │
              │ 공격 (취약점을 이용)
              ▼
        [취약점 (Vulnerability)]
     암호화 미흡 / 패치 미적용
              │
              │ 노출 (자산을 드러냄)
              ▼
         [자산 (Asset)]
      서버 / DB / 고객정보
              │
              │ 보유
              ▼
          [가치 (Value)]
       돈 / 중요도 / 사업 영향
              │
              ▼
      📈 위험(Risk) 증가

──────────────────────────────

[보안요구사항] ──충족──▶ [보안대책]
      │                      │
      └─────▶ [위험] ◀───감소─┘
```

| 요소 | 정의 | 예시 |
|------|------|------|
| **자산 (Asset)** | 보호 대상이 되는 유·무형의 모든 것 | 서버, DB, 고객정보, 회사 이미지 |
| **위협 (Threat)** | 자산에 손실을 줄 수 있는 잠재적 원인/행위자 | 해커, 랜섬웨어, 화재, 정전 |
| **취약점 (Vulnerability)** | 위협이 이용하는 자산의 약점 | 비밀번호 약함, 암호화 미적용 |
| **정보보호대책** | 위험을 줄이는 절차·방법·기술 | 방화벽, 백신, OTP, 접근통제 |

---

### 🔐 위협의 분류

```
위협
├── 환경적 요인
│   ├── 자연재해 (홍수, 지진)
│   └── 장비 고장
└── 인간에 의한 위협
    ├── 의도적 위협 (해킹, 내부자 공격)
    └── 우연한 위협 (실수, 부주의)
```

---

### 🛠️ 보호대책 분류

**유형별 분류**

| 구분 | 예시 |
|------|------|
| 기술적 보호대책 | 방화벽, 백신, 암호화 |
| 관리적 보호대책 | 보안정책, 교육, 감사 |
| 물리적 보호대책 | 출입통제, CCTV |

**적용 시점별 분류**

| 구분 | 설명 | 예시 |
|------|------|------|
| **예방 통제** | 위험 발생 전 사전 차단 | 물리적/논리적 접근통제 |
| **탐지 통제** | 위험 발생 감지 | 접근 위반 로그 |
| **교정 통제** | 탐지 후 피해 줄이기 | 데이터 백업, 재해복구센터 |

---

### 🧩 실제 예시로 이해하기

```
[상황]
고객 정보 DB가 있음 (자산)
암호화가 안 되어 있음 (취약점)
해커가 네트워크를 도청함 (위협)
    ↓
고객 개인정보 유출 발생 (위험 실현)
    ↓
기밀성 침해 → 회사 이미지 손상 + 법적 책임
```

## 3. 전체 위험 vs 잔여 위험

> 💡 100% 안전한 시스템은 존재하지 않는다. 보안대책을 적용한 후에도 항상 **잔여 위험**이 남는다.

### 📊 공식 ⭐⭐

```
전체 위험 (Total Risk)   = 위협 × 취약점 × 자산
잔여 위험 (Residual Risk) = 전체 위험 × 통제 격차 (Controls Gap)

전체 위험 - 보안대책 효과 = 잔여 위험
```

### 시각적 이해

```
위험 크기
100 ██████████████████████  ← 전체 위험 (보안대책 없음)
     ↓ 방화벽, 백신, OTP 적용
 20  ████                   ← 잔여 위험 (어쩔 수 없이 남는 부분)
```

## 4. 위험분석 방법론

### 📋 4가지 방법론 비교 ⭐⭐⭐

```
[기준선 접근법] ──초기 적용──▶ 고위험 영역 발견?
                                      │
                              ┌───────┴────────┐
                             예                아니오
                              ↓                  ↓
                    [상세 위험분석]      [계속 기준선 적용]
```

| 방법론 | 개요 | 적합 대상 | 장점 | 단점 |
|--------|------|-----------|------|------|
| **기준선 접근법** (Baseline) | 표준 체크리스트 적용 | 소규모 조직 | 빠르고 쉬움 | 조직 특성 반영 어려움, 과잉/부족 보호 가능성 |
| **비정형화 접근법** | 전문가 경험/직관 활용 | 경험 풍부한 전문가 있을 때 | 빠르고 비용 적음 | 주관적, 일관성 없음 |
| **상세 위험분석** | 자산·위협·취약점 전부 정밀 분석 | 사회기반시설 (철도, 금융) | 가장 정확 | 시간·비용 많이 듦 |
| **통합(복합) 접근법** | 고위험은 상세, 나머지는 기준선 | 대부분의 중·대형 조직 | 균형 잡힘 | 초기 위험평가 부정확 시 누락 가능 |

> 💬 통합 접근법 = 베이스라인 + 비정형 + 상세 위험분석을 **상황에 맞게** 조합

## 5. 상세 위험분석 절차

> 일반적으로 "위험분석을 수행한다" = 상세 위험분석을 수행한다는 의미

```
① 자산 분석
   └─ 자산 식별 → CIA(기밀성/무결성/가용성) 평가 → 자산 그룹핑
          ↓
② 위협 분석
   └─ 위협 유형 파악 → 발생 가능성 분석
          ↓
③ 취약점 분석
   └─ 자산별 취약점 식별 → 위험에 미치는 영향 분석
          ↓
④ 대응책 분석
   └─ 현재 보안대책 기능 수행 여부 파악
          ↓
⑤ 위험 평가
   └─ DoA(수용 가능 목표 위험수준) 설정 → 초과 위험에 우선순위 부여
          ↓
⑥ 잔류 위험 평가
   └─ 보안정책의 허용 위험수준 만족 여부 확인
```

### 자산 가치 산정 기준

| 구분 | 예시 |
|------|------|
| **정량적 기준** (돈으로 계산) | 자산 도입 비용, 복구 비용, 교체 비용 |
| **정성적 기준** (중요도로 판단) | 업무 기여도, 영향 범위, 복구 소요 시간 |

> 💡 `기밀성(C), 무결성(I)` → 주로 데이터에 의해 결정  
> 💡 `가용성(A)` → 주로 서비스에 의해 결정

## 6. 위험처리 절차 (AAMT)

> 📌 ISO/IEC 27005 기준 ⭐⭐⭐

```
           위험 발견
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼          ▼
  수용        감소        전가        회피
(Accept)  (Mitigate)  (Transfer)  (Avoid)
```

| 처리 방법 | 설명 | 실제 예시 |
|-----------|------|-----------|
| **위험 수용** (Acceptance) | 수용 가능 수준으로 판단하여 그냥 감수 | 피해 금액이 매우 작은 경우 |
| **위험 감소** (Mitigation) | 보안대책으로 위험 수준 낮추기 | 방화벽 설치, OTP 적용, 백업 구축 |
| **위험 전가** (Transfer) | 위험 책임을 제3자와 공유 (비용 동반) | 보험 가입, 외부 업체 계약 |
| **위험 회피** (Avoidance) | 위험이 있는 서비스/프로세스 자체를 포기 | 위험 기능 제거, 시스템 폐기 |

> ⚠️ **주의**: 위험 전가는 "책임을 제3자와 공유"하는 것 (상대방에게 전부 넘기는 게 아님)

### 위험 감소의 두 가지 방향

```
위험 감소
├── 사전(예방): 위험 발생 가능성 낮추기
│   └─ 방화벽 설치, 강한 인증수단 적용
└── 사후(교정): 위험 발생 시 피해 크기 낮추기
    └─ 데이터 백업, 재난 복구 계획, 중요 장치 이중화
```

## 7. 정량적 위험분석

> 모든 요소에 **금전적 가치와 숫자값**을 부여하여 객관적으로 분석

### 핵심 공식 ⭐⭐⭐⭐

| 용어 | 공식 | 설명 |
|------|------|------|
| **노출계수 (EF)** | - | 사고 발생 시 자산 손실 비율 (%) |
| **단일 예상 손실 (SLE)** | `자산가치 × 노출계수` | 사고 한 번 발생 시 예상 손실액 |
| **연간 발생 빈도 (ARO)** | - | 1년에 몇 번 발생하는가 (0.0 ~ 1 이상) |
| **연간 예상 손실 (ALE)** | `ARO × SLE` | 1년간 예상 총 손실액 |

### 계산 예시

```
[상황]
서버 자산가치    = 1,000만원
노출계수        = 50% (사고 시 절반 손실)
연간 발생 빈도  = 2회/년

[계산]
SLE = 1,000만원 × 0.5 = 500만원
ALE = 2 × 500만원 = 1,000만원/년
```

### 보안대책 도입 여부 판단

```
보안대책의 가치 = (도입 전 ALE) - (도입 후 ALE) - (연간 보안대책 비용)
```

**예시:**

```
도입 전 ALE = 12,000달러
도입 후 ALE =  3,000달러
연간 유지비용 =    650달러

보안대책 가치 = 12,000 - 3,000 - 650 = 8,350달러/년 → 도입 가치 있음 ✅
```

> 💡 **결론**: 보안대책 가치가 양(+)이면 도입, 음(-)이면 도입하지 않음

### ARO 값 이해

| ARO 값 | 의미 |
|--------|------|
| 0.1 | 10년에 1번 |
| 0.5 | 2년에 1번 |
| 1.0 | 1년에 1번 |
| 2.0 | 1년에 2번 |

## 8. 정성적 위험분석

> 전문가의 **판단, 직관, 경험**에 기반하여 위험을 주관적으로 분석

---

## 9. 정량적 vs 정성적 비교

| 구분 | 정량적 | 정성적 |
|------|--------|--------|
| **분석 방식** | 숫자·금액으로 계산 | 판단·경험으로 평가 |
| **결과 표현** | 연간 손실액(ALE) 등 수치 | 높음/중간/낮음 등 등급 |
| **객관성** | 높음 | 낮음 (주관적) |
| **비용·시간** | 많이 듦, 복잡 | 적게 듦, 빠름 |
| **자산 가치 평가** | 필요 | 불필요 |
| **자동화 도구 의존** | 높음 (신뢰도가 벤더에 의존) | 낮음 |

## 10. 분석 기법 총정리 ⭐⭐⭐⭐⭐

> 🚨 **시험 최단골 항목** — 정량적/정성적 기법 종류와 설명을 반드시 암기

### 정량적 위험분석 기법

| 기법 | 핵심 키워드 | 설명 |
|------|-------------|------|
| **과거 자료 분석법** | 과거 → 미래 예측 | 과거 사고 데이터로 발생 가능성 예측 |
| **수학공식 접근법** | 계산식, 빈도 추정 | 위협 발생 빈도를 수식으로 계량화 |
| **확률 분포법** | 최저/보통/최고 예측 | 확률적 편차로 위험 범위를 예측 |
| **점수법** | 가중치 | 위험 발생 요인에 가중치를 두어 점수 산정 |

### 정성적 위험분석 기법

| 기법 | 핵심 키워드 | 설명 |
|------|-------------|------|
| **델파이법** | 전문가 집단, 토론 | 전문가들이 토론으로 위협·취약점 분석 |
| **시나리오법** | 가정, 결과 추정 | 특정 조건 하에 발생 가능한 결과 시나리오 도출 |

| **퍼지 행렬법** | 정성적 언어 표현 | 애매한 언어 표현으로 기대 손실 평가 |

### 핵심 공식 모음

```
위험            = f(자산, 위협, 취약점)
전체 위험       = 위협 × 취약점 × 자산
잔여 위험       = 전체 위험 × 통제 격차
SLE             = 자산가치 × 노출계수(EF)
ALE             = ARO × SLE
보안대책 가치   = (도입 전 ALE) - (도입 후 ALE) - 연간 보안대책 비용
```
