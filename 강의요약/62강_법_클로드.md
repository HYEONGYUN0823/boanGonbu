## 1. 개인정보보호법 개요

| 항목 | 내용 |
|------|------|
| 제정일 | 2011년 3월 29일 |
| 시행일 | 2011년 9월 30일 (= **개인정보 보호의 날**) |
| 성격 | **일반법** (공공 + 민간 모두 적용) |
| 적용 범위 | 공공기관, 법인, 단체, 개인사업자 등 약 350만 사업자 |

**제1조(목적):** 개인정보의 처리 및 보호에 관한 사항을 정함으로써 개인의 자유와 권리를 보호하고, 나아가 개인의 존엄과 가치를 구현함을 목적으로 한다.

## 2. 용어 정의 (제2조)

> ⭐⭐⭐ **매우 중요** — 시험에 정의 문제로 자주 출제됨

### 개인정보

**살아있는** 개인에 관한 정보로서 다음 중 하나에 해당하는 것

| 유형 | 설명 | 예시 |
|------|------|------|
| 직접 식별 | 그 정보만으로 특정 개인을 알아볼 수 있음 | 이름, 주민번호, 얼굴 사진 |
| 간접 식별 | 다른 정보와 결합하면 알아볼 수 있음 | 직장명 + 부서 + 성별 |
| 가명정보 | 추가 정보 없이는 알아볼 수 없도록 처리한 것 | A001로 치환된 이름 |

> ⚠️ **사망자 정보는 개인정보 아님.** 실종신고자는 사망 간주이므로 마찬가지.

### 주요 용어 정리

| 용어 | 설명 |
|------|------|
| **가명처리** | 추가 정보 없이는 특정 개인을 알아볼 수 없도록 일부 삭제·대체하는 것 |
| **처리** | 수집·생성·연계·저장·이용·제공·파기 등 개인정보를 다루는 모든 행위 |
| **정보주체** | 처리되는 정보에 의해 알아볼 수 있는 사람 (= 개인정보의 주인) |
| **개인정보파일** | 개인정보를 검색할 수 있도록 체계적으로 배열한 집합물 (예: 고객 DB, 진료기록부) |
| **개인정보처리자** | 업무 목적으로 개인정보파일을 운용하는 공공기관·법인·단체·개인 |
| **고정형 영상정보처리기기** | 일정 장소에 고정 설치되어 지속·주기적으로 촬영하는 장치 (CCTV, 네트워크 카메라) |
| **이동형 영상정보처리기기** | 신체 착용·휴대하거나 이동 가능한 물체에 부착하는 장치 (몸캠, 디지털카메라, 블랙박스) |

> 💡 **차량 블랙박스**는 촬영 범위가 수시로 변경되어 **고정형이 아닌 이동형**으로 분류됨

## 3. 개인정보 보호 원칙 (제3조)

> ⭐⭐⭐ **매우 중요**

| 원칙 | 내용 |
|------|------|
| 목적 적합성 | 처리 목적에 필요한 범위에서 적합하게만 처리 |
| 정확성 | 정확성·완전성·최신성이 보장되도록 관리 |
| 안전성 | 권리 침해 가능성·위험 정도를 고려해 안전하게 관리 |
| 투명성 | 처리방침 등을 공개 |
| 최소화 | 익명 처리가 가능하면 익명, 불가능하면 가명으로 처리 |

## 4. OECD 프라이버시 8원칙

> 암기법: **수정이 목이 언정처의 책임이다**

| 번호 | 원칙 | 내용 |
|------|------|------|
| 1 | **수**집제한 | 필요 이상 수집 금지 |
| 2 | **정**보 정확성 | 틀린 정보는 바로잡아야 함 |
| 3 | 목적 **명**확화 | 수집 목적을 분명히 해야 함 |
| 4 | **이**용제한 | 목적 외 사용 금지 |
| 5 | **언**전성(안전성) 확보 | 안전하게 관리 |
| 6 | **정**보방침 공개 | 처리 방법 공개 |
| 7 | **처**리주체 참여 (정보주체 참여) | 본인이 자기 정보에 대해 확인·요구 가능 |
| 8 | **의**임(책임) | 문제 발생 시 처리자가 책임 |

## 5. 정보주체의 권리 (제4조)

> 암기법: **동열정피자**

우리나라는 **개인정보 자기결정권**을 인정함.
단, **절대적 권리가 아님** — 국가안보, 질서유지, 공공복리를 위해 법률로 제한 가능

| 권리 | 내용 |
|------|------|
| **동**의 결정권 | 동의 여부·범위를 선택하고 결정할 권리 |
| **열**람권 | 처리 여부 확인 및 열람(사본 발급 포함)·전송 요구 |
| **정**정·삭제·처리정지권 | 정정·삭제·파기 요구 |
| **피**해구제권 | 피해를 신속·공정하게 구제받을 권리 |
| **자**동화 결정 거부권 | AI 등 자동화된 결정을 거부하거나 설명 요구 가능 |

## 6. 다른 법률과의 관계 (제6조)

```
개인정보보호법 = 일반법 (기본법)
특별법 (신용정보법, 의료법 등) > 개인정보보호법
```

- 다른 법률에 특별한 규정이 있으면 그 법률이 우선 적용됨
- 다른 법률을 제·개정할 때는 개인정보보호법의 목적·원칙에 맞도록 해야 함

### 정보통신망법 vs 개인정보보호법 비교

| 구분 | 정보통신망법 | 개인정보보호법 |
|------|-------------|---------------|
| 규제기관 | 방통위, 과기부 | 개인정보보호위원회 |
| 적용 대상 | 통신·인터넷 사업자 | 거의 모든 공공·민간 조직 |
| 적용 범위 | 전자파일 중심, 임직원 정보 제외 | 전자파일 + 수기문서 + 임직원 정보 |

## 8. 개인정보보호위원회

| 항목 | 내용 |
|------|------|
| 소속 | **국무총리 소속** 중앙행정기관 |
| 위원 수 | **총 9명** (위원장 + 부위원장 포함) |
| 임기 | **3년** (연임 가능) |

### 구성 구조 (2+2+3+2 = 9명)

```
위원장·부위원장 (2명) : 국무총리 제청
위원장 제청 (2명)
여당 교섭단체 추천 (2명)
야당 교섭단체 추천 (3명)
```

### 회의 성립 조건

```
총 9명 → 5명 이상 출석(개의) → 출석위원 과반수 찬성(의결)
예: 5명 출석 시, 3명 이상 찬성이면 의결
```

### 주요 심의·의결 사항

- 기본계획 및 시행계획
- 정책·제도·법령 개선
- 국외 이전 중지 명령
- 과징금, 과태료 부과

## 11. 개인정보 수집·이용 (제15조)

> ⭐⭐⭐ **매우 중요**

**원칙: 정보주체의 동의를 받아 수집**

### 동의 없이 수집 가능한 경우

> 암기법: **법계생정공**

| 경우 | 예시 |
|------|------|
| **법**률상 의무 | 보험회사의 사고처리를 위한 정보 수집 |
| **계**약 이행 | 경품 배송을 위한 주소·연락처 수집 |
| **생**명·신체·재산의 급박한 이익 | 화재·수해 시 구조를 위한 정보 활용 |
| **정**당한 이익 (처리자) | 요금 징수·정산 등 |
| **공**공의 안전·안녕 | 코로나 감염병 대응 등 |

### 동의받을 때 알려야 할 사항

> 암기법: **목항기불**

1. **목**적 — 왜 모으는지
2. **항**목 — 뭘 모으는지
3. **기**간 — 얼마나 보관하는지
4. **불**이익 — 동의 거부 시 불이익

## 12. 개인정보 수집 제한 (제16조)

- **입증책임은 개인정보처리자**에게 있음 (처리자가 "꼭 필요했다"고 증명 못하면 패소)

## 13. 개인정보 제공 (제17조)

> ⭐⭐⭐ **매우 중요**

**제공 = 제3자에게 개인정보의 지배·관리권이 이전되는 것**

DB 접속권한 허용으로 열람·복사가 가능하게 하는 것도 **제공**에 해당

### 제공 시 알려야 할 사항

> 암기법: **받목항기불**

1. **받**는 자 — 누구에게 제공하는지
2. **목**적 — 이용 목적
3. **항**목 — 제공 항목
4. **기**간 — 보유·이용 기간
5. **불**이익 — 거부 시 불이익

---

## 14. 목적 외 이용·제공 제한 (제18조)

**원칙: 처음 정한 목적 밖으로 사용 금지**

```
상품 배송 목적으로 받은 주소 → 광고 문자 발송에 사용 ❌
AS용 고객정보 → 신상품 홍보 ❌ (별도 동의 필요)
```

### 목적 외 이용·제공 가능한 예외

| 경우 | 대상 |
|------|------|
| 별도의 동의 | 공공기관 + 민간 |
| 생명·신체·재산의 급박한 이익 | 공공기관 + 민간 |
| 수사, 재판, 형 집행 | **공공기관만** |
| 다른 법률에 근거 | **공공기관만** |
| 공공의 안전·안녕 (긴급) | 공공기관 + 민간 |

> ⚠️ 5~9호(수사, 재판, 형 집행 등)는 **공공기관에만 적용**됨

---

## 15. 제공받은 자의 이용·제공 제한 (제19조)

- **받은 목적 외 사용 금지**
- **다시 제3자에게 제공 금지**
- 예외: 별도 동의, 다른 법률 근거

> ⚠️ 위탁받아 처리하는 **수탁자**에게는 본 조항이 직접 적용되지 않음

## 16. 수집 출처 통지 (제20조)

**정보주체 이외(제3자, 공개된 출처)에서 수집한 경우**

- 정보주체 요구 시 **즉시(표준 지침: 3일 이내)** 통지해야 함

| 용어 비교 | |
|-----------|--|
| **고지** | 사전에 알려주는 것 |
| **통지** | 사후에 알려주는 것 |

### 요구 없이도 통지해야 하는 대량 처리자

| 기준 | |
|------|--|
| 민감정보·고유식별정보 | **5만명 이상** |
| 일반 개인정보 | **100만명 이상** |

### 통지 내용

1. 수집 출처
2. 처리 목적
3. 처리 정지 요구권 및 동의 철회권

## 17. 개인정보 파기 (제21조)

> 개인정보가 불필요하게 된 때: **지체없이(5일 이내, 표준지침)** 파기

### 파기 사유

- 보유기간 경과
- 처리 목적 달성
- 서비스 폐지, 사업 종료

### 파기 방법

| 유형 | 방법 |
|------|------|
| 전자파일 | 복구·재생 불가하도록 완전 삭제 (덮어쓰기, 소자 등) |
| 종이·인쇄물 | 마스킹, 구멍 뚫기 등 |

> ⚠️ 다른 법령에 의해 보존 의무가 있으면 **분리 저장·관리** 후 보존

## 18. 동의를 받는 방법 (제22조)

- 항목별로 **구분해서 각각 동의**를 받아야 함 (뭉뚱그려서 받으면 안 됨)
- 서면 동의 시 **알아보기 쉽게 표시**해야 함
- **선택적 동의 거부를 이유로 서비스 거절 금지**

- 법정대리인 동의 여부를 확인해야 함

## 25. 업무위탁 vs 제3자 제공 (제26조)

> ⭐⭐⭐ **매우 중요** — 반드시 구별해야 함

| 구분 | 업무위탁 | 제3자 제공 |
|------|---------|-----------|
| 목적 | 위탁자의 이익 | 제3자의 이익 |
| 예시 | 배송 위탁, 고객센터 위탁 | 광고회사에 정보 판매 |
| 동의 여부 | 원칙적으로 동의 불필요 | 원칙적으로 동의 필요 |
| 책임 | **위탁자 책임** | 제3자 책임 |

### 위탁 시 의무사항

- **문서(위수탁 계약서)** 필수
- 개인정보 처리방침에 **수탁자 공개**
- 위탁자는 수탁자를 **교육·감독** 해야 함
- 재위탁 시 **위탁자의 동의 필요**

### 수탁자 사고 시 책임

```
수탁자가 개인정보 유출 → 위탁자도 손해배상 책임 있음
(수탁자 = 위탁자 소속 직원으로 봄)
```

## 27. 가명정보 처리 특례 (제28조의2~5)

> ⭐⭐⭐ **매우 중요**

### 가명정보란?

개인을 직접 알아볼 수 없도록 이름·식별자 등을 변환한 정보

```
김형윤 → A001
```

### 동의 없이 처리 가능한 경우

> 암기법: **통과공**

| | 내용 |
|--|------|
| **통**계 | 통계 작성 |
| **과**학 | 과학적 연구 |
| **공**익 | 공익적 기록보존 |

### 핵심 금지사항

- **특정 개인을 재식별하려는 목적으로 사용 금지**
- 재식별 가능한 정보가 생성되면 **즉시 처리 중지 및 파기**

### 가명정보 결합

- 반드시 **전문기관**(보호위원회 또는 관계 중앙행정기관 지정)을 통해서만 결합 가능
- 외부 반출 시 전문기관 **승인 필요**

### 안전조치

- 추가정보(원본 연결표)는 **가명정보와 분리 보관**
- 처리 기록은 **3년 이상 보관**

## 32. 국내대리인 지정 (제31조의2)

## 34. ISMS-P 인증 (제32조의2, 제47조)

> ISMS(정보보호 관리체계) + PIMS(개인정보 관리체계) = **ISMS-P**

**쉽게 말해: "이 기관은 정보보안과 개인정보 보호를 체계적으로 잘 하고 있나?"를 인증하는 제도**

| 항목 | 내용 |
|------|------|
| 근거 | 정보통신망법 제47조 + 개인정보보호법 제32조의2 |
| 인증기관 | **KISA(한국인터넷진흥원)** |
| 유효기간 | **3년** |
| 사후관리 | **연 1회 이상** (서면심사 또는 현장심사) |

### 의무 인증 대상

| 구분 | 기준 |
|------|------|
| ISP | 서울·광역시 서비스 제공자 |
| IDC | 집적정보통신시설 사업자 |
| 매출액 | 정보통신서비스 부문 **100억원 이상** |
| 이용자 수 | 일일 평균 **100만명 이상** |
| 기타 | 연간 매출·세입 **1,500억원 이상**, 상급 종합병원, 재학생 **1만명 이상** 학교 |

### 인증 취소 사유

| 사유 | 구분 |
|------|------|
| 거짓·부정한 방법으로 인증받은 경우 | **강행규정** (반드시 취소) |
| 사후관리를 거부·방해한 경우 | 임의규정 (취소 가능) |
| 인증기준에 미달하게 된 경우 | 임의규정 (취소 가능) |
| 관련 법령을 중대하게 위반한 경우 | 임의규정 (취소 가능) |

## 35. 개인정보 영향평가 (제33조)

> ⭐⭐⭐ **매우 중요** — PIA (Privacy Impact Assessment)

**쉽게 말해: 개인정보 시스템 만들기 전에 "이거 위험하지 않나?"를 미리 분석하는 것**

### 의무 대상

- **공공기관** (민간은 자율)

### 평가 기준 (시행령 제35조)

| 조건 | |
|------|--|

| 변경 영향평가: **변경된 부분에 한정** |

### 평가 시 고려사항

1. 처리하는 개인정보의 수
2. 제3자 제공 여부
3. 정보주체 권리 침해 가능성·위험 정도
4. 민감정보·고유식별정보 처리 여부
5. 개인정보 보유기간

### 절차

사전준비 → 영향평가 수행 → 개선 이행

## 37. 정보주체 권리 보장 (제35~37조)

### 열람권 (제35조)

- 열람 요구 → **10일 이내** 조치
- 사본 발급 포함

### 정정·삭제권 (제36조)

- 잘못된 정보 정정·삭제 요구 가능
- 법령상 보존 의무 있으면 삭제 불가 (예: 전자상거래 기록 5년)
- **지체없이(10일 이내)** 조치

### 처리정지권 (제37조)

| 구분 | 동의철회 | 처리정지 |
|------|---------|---------|
| 범위 | 내가 동의한 것만 | 모든 개인정보 처리 |
| 넓이 | 좁음 | **더 넓음** |

### 처리정지 거절 가능한 경우

1. 법률상 의무
2. 다른 사람의 생명·신체 침해 우려
3. 공공기관 소관업무 불가
4. 계약 이행 불가 (해지 의사 미표명 시)

## 38. 손해배상 (제39조)

- 처리자가 법 위반으로 손해 발생 시 **정보주체가 배상 청구 가능**
- **입증책임은 개인정보처리자** ("우리는 고의·과실 없었다"를 처리자가 증명해야 함)
- 고의·중대한 과실 시 **최대 5배** 배상

### 법정손해배상 (제39조의2)

- 실제 손해액 입증 어려울 때 **최대 300만원** 인정 가능

## 39. 개인정보 분쟁조정위원회

> 소송 없이 분쟁을 해결하는 대안적 분쟁해결(ADR) 기관

| 항목 | 내용 |
|------|------|
| 위원 임기 | **2년** (1차 연임 가능) |
| 처리기간 | **60일 이내** |
| 조정안 수락 | 15일 내 거절 않으면 **수락 간주** |
| 조정 효력 | **재판상 화해와 동일** |

> cf. 개인정보보호위원회: **9명, 3년**

### 제척·기피·회피 구분

| 용어 | 의미 |
|------|------|
| **제척** | 법률상 강제로 빠지는 것 ("당신 빠지세요") |
| **기피** | 당사자가 특정 위원 배제를 요청하는 것 ("저 사람 빼주세요") |
| **회피** | 위원이 스스로 빠지는 것 ("제가 빠질게요") |

### 집단분쟁조정 (제49조)

- 대량 피해자가 발생한 경우 일괄 처리
- 공고 후 14일 이상 참가 신청 기간
- 공고 종료 다음 날부터 **60일 이내** 처리

## 40. 개인정보 안전성 확보 조치 기준

### 내부 관리계획 (제4조)

- 최고경영층 승인 후 임직원에게 공지
- **연 1회 이상** 보호책임자가 점검
- 유형1 (1만명 미만 소상공인 등): 일부 생략 가능
- 유형2 (공공기관, 대기업 등): 전체 적용

### 접근권한 관리 (제5조)

- **최소한의 범위**로 차등 부여
- 가명정보 담당자 ≠ 추가정보 담당자 (반드시 구분)
- 권한 변경 시 **지체없이** 말소·변경
- 접근권한 부여·변경·말소 기록 **최소 3년 보관**
- 계정 공유 금지
- 일정 횟수 이상 인증 실패 시 **접근 차단 (임계치 설정)**

### 암호화 의무 대상 (제7조)

| 무조건 암호화 저장 | 영향평가 or 위험도분석 후 결정 |
|-----------------|------------------------------|
| 비밀번호, 생체인식정보 (일방향) | 기타 고유식별정보 |
| 주민등록번호 | |

**암호화 필수 항목:** 주민등록번호, 여권번호, 운전면허번호, 외국인등록번호, 신용카드번호, 계좌번호, 생체인식정보

> ⚠️ 전화번호는 암호화 의무 대상 아님

### 접속기록 보관 (제8조)

| 구분 | 보관기간 |
|------|---------|
| 일반 | **1년 이상** |
| 5만명 이상 처리 / 고유식별·민감정보 처리 / 기간통신사업자 | **2년 이상** |

- **월 1회 이상** 점검

### 개인정보 파기 방법 (제13조)

| 유형 | 방법 |
|------|------|
| 전자파일 (전부) | 완전파괴, 소자장비, 초기화·덮어쓰기 |
| 종이·기타 (일부) | 마스킹, 구멍 뚫기 |

## 42. 정보통신망법 주요 조항

### 스마트폰 접근권한 동의 (제22조의2)

- **필수 권한**과 **선택 권한**을 구분하여 동의 받아야 함
- 선택 권한은 거부해도 서비스 이용 가능

### 주민등록번호 사용 제한 (제23조의2)

- 원칙: 수집 금지
- 대체수단 제공 의무: I-PIN, 휴대폰 인증 등

### 본인확인기관 (제23조의3, 제23조의4)

| 구분 | 내용 |
|------|------|
| 능동 (기관 스스로) | 휴지(30일 전 신고, 최대 6개월), 폐지(60일 전 신고) |
| 수동 (방통위 강제) | 정지·취소 명령 |

**취소 강행규정:** 거짓으로 지정받은 경우 → **반드시 취소**

### 게시물 권리 침해 (제44조의2)

- 사생활 침해·명예훼손 시 삭제 or 반박 게재 요청 가능
- **임시조치:** 판단 어려울 때 **30일 이내** 임시 차단

### 불법정보 유통 금지 (제44조의7)

- 음란물, 명예훼손, 불법 개인정보 거래, 총기·마약 관련 정보 등

### 정보보호 최고책임자 (CISO, 제45조의3)

- 일정 규모 이상 사업자는 CISO 지정 후 **과기부 장관에게 신고** (신고: 90일 이내)
- 다른 업무와 겸직 제한

**CISO 업무:**

1. 정보보호 계획 수립·시행·개선
2. 정기적 감사
3. 위험 식별·평가 및 대책 마련
4. 교육·모의훈련 계획 수립·시행

### 침해사고 대응 (제48조의2, 제48조의3)

- 정보통신망 침입 금지 (정당한 접근권한 없이 침입 금지)
- 침해사고 발생 시 **과기부 또는 KISA에 신고**
- KISA: 정보 수집·전파, 예보·경보, 긴급조치 수행

## 43. 정보통신기반보호법

> **국가 핵심 인프라(주요정보통신기반시설)를 보호하기 위한 법**

### 주요정보통신기반시설 예시

- 도로·철도·공항·항만 등 교통시설
- 전력·가스·석유 등 에너지·수자원
- 방송중계·국가지도통신망
- 원자력·국방과학 연구시설

### 수행 주체별 역할

| 주체 | 역할 |
|------|------|
| **정보통신기반보호위원회** | 보호정책 심의, 대책본부 운영 (국무조정실 소속, 25인 이내) |
| **관계중앙행정기관** | 소관 분야 보호계획 수립, 보호지침 제·개정, 기반시설 지정 |
| **관리기관** | 보호대책 수립·시행, 취약점 분석·평가, 침해사고 통지 |
| **과기부/국정원** | 보호대책 수립지침 작성·배포, 이행여부 확인, 기술지원 |
| **KISA/ETRI/ISAC** | 취약점 분석·평가, 침해사고 예방·복구 지원 |

### 기반시설 지정 기준 (제8조)

1. 국가사회적 중요성
2. 정보통신기반시설 의존도
3. 다른 시설과의 상호연계성
4. 침해사고 발생 시 피해 규모
5. 침해사고 발생 가능성·복구 용이성

### 취약점 분석·평가 시기 (시행령 제17조)

| 시기 | 기한 |
|------|------|
| 지정된 때 | **6개월 이내** (사유 있으면 9개월) |
| 이후 | **매년** (중대한 변화 발생 시 수시) |

### 침해행위 금지 (제12조)

1. 권한 없이 데이터 조작·파괴·유출
2. 바이러스·논리폭탄 투입
3. 대량 신호로 정보처리 오류 발생

### 대응 체계

| 단계 | 내용 |
|------|------|
| 통지 (제13조) | 교란·마비·파괴 인지 시 관계기관·수사기관·KISA에 즉시 통지 |
| 복구 (제14조) | 침해사고 발생 시 신속 복구 조치 |
| 대책본부 (제15조) | 광범위한 대형 사고 시 위원회 내에 한시 운영 |

### ISAC (정보공유·분석센터, 제16조)

- 취약점·침해요인 정보 제공
- 침해사고 발생 시 **실시간 경보·분석체계 운영**
