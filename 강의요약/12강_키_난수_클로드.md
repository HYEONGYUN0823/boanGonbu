# 🗝️ 키(Key) & 난수

---

## 1. 키(Key) 기본 개념

> 💡 **비유 : 금고 열쇠**
> 금고(암호화된 데이터)를 열려면 열쇠(키)가 필요해
> 열쇠가 많을수록(키 공간이 클수록) 도둑이 모든 열쇠를 다 시도하기 어려움!

```
키 공간(Key Space) = 가능한 키의 총 수
키가 n비트 → 키 공간 크기 = 2^n

키 공간이 클수록 → 무차별 공격에 강함!
```

> ⚠️ **핵심 : 키 = 평문과 같은 가치**
> 암호 해독의 목적 = 키를 찾거나 평문을 찾거나
> 키를 탈취하면 평문을 바로 얻을 수 있어서 키 보호가 매우 중요!

---

## 2. 다양한 키 종류 ⭐⭐⭐

### 세션키 vs 마스터키

> 💡 **비유 : 일회용 비밀번호 vs 고정 비밀번호**
> 세션키 = 카페에서 하루만 쓰는 와이파이 비밀번호 (통신할 때마다 새로 만듦)
> 마스터키 = 집 현관 비밀번호 (계속 반복해서 사용)

| 구분 | 세션키 | 마스터키 |
|---|---|---|
| 사용 횟수 | 한 번만 사용 후 폐기 | 반복 사용 |
| 보안성 | 높음 (탈취돼도 한 번만 피해) | 낮음 (탈취 시 계속 피해) |
| 다른 이름 | CEK | KEK |

### CEK vs KEK

```
CEK (Content Encryption Key)
= 실제 데이터(평문)를 암호화하는 키
= 세션키

KEK (Key Encryption Key)
= 키를 암호화하는 키
= 마스터키

비유 : 햄버거 레시피(CEK)를 금고에 넣고
       금고 열쇠(KEK)로 잠그는 것!
       금고 열쇠(KEK)만 잘 보관하면 레시피(CEK)도 안전
```

## 3. 패스워드 기반 암호 (PBE)

> 💡 **비유 : 패스워드로 금고 키를 만드는 것**
> "내 비밀번호 1234" 그대로 키로 쓰면 너무 약함
> → 패스워드 + 솔트 + 반복 연산 → 강한 암호화 키 생성!

### 솔트(Salt)의 역할 ⭐⭐⭐⭐⭐

> 💡 **비유 : 같은 햄버거에 소금(salt)을 다르게 뿌리기**
> 패스워드가 같아도 솔트가 다르면 완전히 다른 해시값이 나옴
> → 미리 계산해둔 공격 테이블이 무용지물!

```
솔트 = 패스워드에 추가하는 랜덤값

솔트가 없으면?
→ 사전 공격 : 자주 쓰는 비밀번호 목록으로 공격
→ 레인보우 테이블 공격 : 미리 계산해둔 해시값 테이블로 공격

솔트가 있으면?
→ 같은 비밀번호도 솔트마다 다른 해시값
→ 레인보우 테이블 사용 불가!
```

### PBKDF (Password-Based Key Derivation Function)

```
입력 : 비밀번호 + 솔트 + 반복횟수 + 생성할 키 길이
출력 : 암호화 키

반복 횟수가 많을수록 → 더 강한 키, 하지만 느려짐
```

### 암호키 저장 방법 ⭐⭐⭐

```
하드웨어 토큰
= 보안토큰 = HSM (Hardware Security Module)
= 스마트카드, USB 토큰 등

특징 : 저장된 정보가 위·변조되거나 외부로 노출되기 어려운 장치
```

## 4. 난수(Random Number)

### 난수의 용도

| 용도 | 설명 |
|---|---|
| 키 생성 | 대칭키, MAC 키 생성 |
| 키쌍 생성 | 공개키/비대칭키 생성 |
| IV 생성 | CBC, CFB, OFB 모드의 초기화 벡터 |
| 비표(nonce) 생성 | 재전송 공격 방지, CTR 모드 |
| 솔트 생성 | PBE 패스워드 암호화 |
| 일회용 패드 | 패딩에 사용되는 열 생성 |

---

## 5. 난수의 성질

> 💡 **비유 : 완벽한 주사위**
> 무작위성 = 1~6이 균등하게 나옴
> 예측 불가능성 = 이전 결과를 봐도 다음 숫자를 못 맞춤
> 재현 불가능성 = 같은 순서의 눈금이 다시 나오지 않음

| 성질 | 설명 |
|---|---|
| 무작위성 | 통계적 편중 없이 무작위로 분포 |
| 예측 불가능성 | 과거 수열로 다음 수를 예측 불가 |
| 재현 불가능성 | 같은 수열을 다시 만들 수 없음 (재현하려면 수열 자체를 보존해야 함) |

```
강한 난수일수록 세 가지 성질을 모두 만족
약한 난수 = 무작위성만 만족 (통계적으로 균등하지만 예측 가능할 수 있음)
```

## 6. 난수 생성기 ⭐⭐⭐

### RNG vs PRNG

| 구분 | RNG | PRNG |
|---|---|---|
| 전체 이름 | Random Number Generator | Pseudo Random Number Generator |
| 구현 방식 | **하드웨어** | **소프트웨어** |
| 진짜 난수? | ✅ 진짜 난수 | ❌ 의사(가짜) 난수 |
| 예시 | 방사선 측정, 열잡음 | 알고리즘으로 생성 |

> ⚠️ **소프트웨어만으로는 진정한 난수를 생성할 수 없음!**
> 그래서 소프트웨어 방식을 **의사(擬似)** 난수 생성기라고 부름

### 의사난수 생성기 구조

> 💡 **비유 : 레시피 + 재료 → 음식**
> 종자(seed) = 비밀 재료 (랜덤 비트열)
> 내부상태 = 현재 재료 상태 (메모리 값)
> 출력 = 의사난수열 (요리된 음식)

```
[종자(seed)] → [의사난수 생성기] → 의사난수열 출력
                     ↑
               내부상태(메모리 값) 관리

종자(seed) = 외부에서 주어지는 랜덤 비트열 (비밀!)
내부상태   = 의사난수 생성기가 관리하는 메모리 값
```

## 📌 최종 핵심 암기 요약

```
✅ 키 공간이 클수록 → 무차별 공격에 강함
✅ 키 = 평문과 같은 가치 → 키 보호 매우 중요!

✅ 세션키(CEK) = 한 번만 사용, 마스터키(KEK) = 반복 사용
✅ KEK = 키를 암호화하는 키

✅ 솔트(Salt)
   목적 : 사전 공격 + 레인보우 테이블 공격 방지
   원리 : 같은 비밀번호도 솔트가 다르면 다른 해시값

✅ HSM (Hardware Security Module)
   = 보안토큰 = 스마트카드, USB 토큰
   = 키를 안전하게 저장하는 하드웨어 장치

✅ 난수 3가지 성질
   무작위성 / 예측 불가능성 / 재현 불가능성

✅ RNG (하드웨어) = 진짜 난수
   PRNG (소프트웨어) = 의사난수 (진짜 난수 아님!)
   의사난수 생성기 = 종자(seed) + 내부상태(메모리 값)로 동작
```
