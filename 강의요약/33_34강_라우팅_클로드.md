## 1. 라우팅 기본 개념

> 💡 라우터 = **길찾기 기사**

- 패킷의 **목적지 IP(D-IP)** 를 보고
- 여러 경로 중 **최선의 경로** 를 선택해서 전달
- 동작하는 계층: **네트워크 계층 (L3)**

---

## 2. 정적 vs 동적 라우팅

| 구분 | 정적 라우팅 | 동적 라우팅 |
|------|------------|------------|
| 설정 방법 | 관리자가 직접 입력 | 라우터가 자동으로 구성 |
| 변화 대응 | ❌ 수동 수정 필요 | ✅ 자동으로 경로 재구성 |
| 적합한 환경 | 변화가 적은 소규모 NW | 변화가 잦은 대규모 NW |
| 보안 | ✅ 공격자가 경로 조작 불가 | ⚠️ 인증 설정 필요 |
| 오버헤드 | 낮음 | 높음 (라우팅 정보 교환 때문) |

> ✅ **정적 라우팅 명령어**
> ```
> Router(config)# ip route [목적지 네트워크] [마스크] [다음 라우터 or 인터페이스]
> ```

## 3. Routed Protocol vs Routing Protocol

| 구분 | 설명 | 예시 |
|------|------|------|
| **Routed Protocol** | 실제로 라우팅되는 **데이터 (손님)** | IP, IPX, AppleTalk |
| **Routing Protocol** | 경로를 찾는 **방법 (네비게이션)** | RIP, OSPF, BGP, EIGRP |

> 💬 비유: 라우티드 프로토콜은 **택배 물건**, 라우팅 프로토콜은 **배달 경로 앱**

---

## 4. AS / IGP / EGP

### 자율 시스템 (AS, Autonomous System)
- 라우터와 네트워크의 집합체
- 하나의 관리 정책 하에 운영되는 네트워크 그룹

### IGP vs EGP

| 구분 | 범위 | 프로토콜 |
|------|------|---------|
| **IGP** (Interior Gateway Protocol) | AS **내부** | RIP, OSPF, EIGRP, IGRP |
| **EGP** (Exterior Gateway Protocol) | AS **간** | BGP |

> 💬 비유: IGP = 회사 내부 통신, EGP = 회사 간 통신

---

## 5. 직접 전달 vs 간접 전달

```
직접 전달 : PC ─── 스위치(L2) ─── PC      (같은 네트워크)

간접 전달 : PC ─── 스위치(L2) ─── 라우터(L3) ─── 라우터(L3) ─── PC
                                                  (다른 네트워크)
```

## 6. 라우팅 알고리즘 3종

### ① 거리 벡터 (Distance Vector)

- **기준**: 홉 수 (정거장 몇 개?)
- **방법**: 이웃 라우터에게 주기적으로 자신의 테이블 전달
- **단점**: 느린 수렴, 라우팅 루프 발생 가능
- **대표 프로토콜**: RIP, IGRP

> 💬 비유: "3번만 갈아타면 도착해" 수준의 정보만 앎

### ② 링크 상태 (Link State)

- **기준**: 전체 네트워크 지도를 알고 있음
- **방법**: 변화 발생 시 전체에 **플러딩(flooding)** 으로 전달
- **알고리즘**: **Dijkstra** (최단 경로)
- **저장소**: LSDB (링크 상태 데이터베이스) - 모든 라우터가 동일한 복사본 보유
- **대표 프로토콜**: OSPF

> 💬 비유: "네이버 지도처럼 전체 도로 상황 다 앎"

### ③ 경로 벡터 (Path Vector)

- **기준**: 어떤 AS를 거치는지
- **대상**: AS 간 라우팅 (ISP끼리)
- **대표 프로토콜**: BGP

> 💬 비유: "KT → SK → LG 이 경로로 감"

### 요약 비교

| | 거리 벡터 | 링크 상태 | 경로 벡터 |
|--|-----------|-----------|-----------|
| 정보 범위 | 이웃만 | 전체 | AS 경로 |
| 수렴 속도 | 느림 | 빠름 | - |
| 정보 전달 | 주기적 | 변화 시만 | - |
| 대표 | RIP | OSPF | BGP |

## 7. RIP (Routing Information Protocol)

### 개요

- 가장 오래되고 널리 사용된 **IGP**
- 거리 벡터 알고리즘 사용
- 기준: **홉 수 (Hop Count)**
- **최대 홉: 15** → 16 = 도달 불가 (무한대)
- 30초마다 라우팅 정보 전송
- 전송 프로토콜: **UDP**

### 버전 비교

| 구분 | RIPv1 | RIPv2 |
|------|-------|-------|
| 서브넷 지원 | ❌ | ✅ (CIDR, VLSM) |
| 전송 방식 | 브로드캐스트 | 브로드캐스트 + **멀티캐스트** |
| 인증 | ❌ | ✅ |
| IPv6 | ❌ | ❌ (RIPng 사용) |

### 문제점

| 문제 | 설명 |
|------|------|
| 느린 수렴 | 30초 주기 → 모든 라우터가 같은 정보 가질 때까지 오래 걸림 |
| 라우팅 루프 | 잘못된 경로 정보가 계속 순환 |
| 무한 세기 | 라우터끼리 끝없이 잘못된 정보 주고받음 |
| 규모 제한 | 15홉 이상 불가 |

### 루프 방지 기법 ⭐

| 기법 | 설명 |
|------|------|
| **Triggered Update** | 변화 발생 시 즉시 알림 (30초 기다리지 않음) |
| **Hold Down** | 일정 시간 경로 갱신 무시 |
| **Split Horizon** | A→B로 보낸 정보는 다시 A에게 보내지 않음 |
| **Route Poisoning** | 죽은 경로를 홉 수 **16**으로 설정해 전파 → "여기 못 감" 즉시 알림 |

## 8. IGRP / EIGRP

### IGRP

- **Cisco 전용** 거리 벡터 프로토콜
- 홉 수 대신 **5가지 파라미터** 로 경로 계산 (대역폭, 지연, 신뢰도 등)
- 최대 4개의 경로 제공 (부하 분산)

| 비교 | RIP | IGRP |
|------|-----|------|
| 경로 수 | 1개 | 최대 4개 |
| 홉 제한 | 15 | 제한 없음 |
| 벤더 | 표준 | Cisco 전용 |
| 메트릭 | 홉 수 | 5가지 파라미터 |

### EIGRP

- IGRP의 발전형, **Hybrid** 프로토콜 (거리 벡터 + 링크 상태 특성)
- 빠른 수렴 속도
- 부분 갱신 지원 → 적은 대역폭 사용

---

## 9. OSPF (Open Shortest Path First)

### 개요

- **링크 상태** 라우팅 프로토콜
- **개방형 표준** (Cisco 전용 아님)
- 최단 경로 계산: **Dijkstra 알고리즘**
- 변화 발생 시에만 **플러딩** → 수렴 빠름
- 보안: **인증 지원**
- CIDR, 서브넷, 클래스 주소 모두 지원

### 플러딩 (Flooding)

> 💡 정보를 전체 네트워크에 뿌리는 방식

- 한 라우터 → 모든 라우터에게 전달
- 단톡방에 공지 올리는 것과 같음 📢
- OSPF는 **변화가 생겼을 때만** 플러딩 수행

### 계층 구조

```
[인터넷]
    │
[Backbone Area - Area 0] ← 본사
    ├── Area 1 ← 부서 A
    ├── Area 2 ← 부서 B
    └── Area 3 ← 부서 C
```

- **Area**: 라우터 그룹 (작은 AS처럼 동작)
- **Backbone (Area 0)**: 모든 Area를 연결하는 논리적 중심
- **ABR** (Area Border Router): Area 간 정보 전달
- 장점: 라우팅 트래픽 감소, 수렴 빠름

### 거리 벡터 vs 링크 상태 비교 ⭐

| | 거리 벡터 | 링크 상태 |
|--|-----------|-----------|
| 정보 수집 | 인접 이웃만 | 모든 라우터 |
| 정보 교환 | 주기적 | 변화 시만 |
| 수렴 시간 | 느림 | 빠름 |
| 전달 내용 | 전체 테이블 | 직접 연결 링크만 |

---

## 10. BGP (Border Gateway Protocol)

- 현재 인터넷에서 사용하는 **유일한 EGP**
- **경로 벡터** 알고리즘 기반
- 메트릭: **AS 번호** (홉 수 아님)
- 전송 프로토콜: **TCP** (신뢰성 보장)

```
AS1 ── AS2 ── AS3 ── AS4
(KT)  (SK)  (LG)  (목적지)
```

> 각 AS는 고유 번호를 가지며, BGP는 AS 경로를 기준으로 최적 경로 선택

## 11. 라우팅 테이블

> 💡 **라우팅 테이블 = 라우터의 길찾기 메모장**

### 구조

| 목적지 네트워크 | 넷마스크 | 다음 홉 (Next Hop) | 인터페이스 |
|---------------|---------|-------------------|-----------|
| 192.168.1.0 | /24 | 직접 연결 | eth0 |
| 10.0.0.0 | /8 | 192.168.1.1 | eth1 |
| 0.0.0.0 | /0 | 192.168.1.254 | eth2 |

### 동작 과정

1. 패킷의 **D-IP** 확인
2. 라우팅 테이블에서 비교
3. 가장 정확하게 맞는 경로 선택 → **Longest Prefix Match**

### Longest Prefix Match ⭐

> 💡 여러 경로가 매칭되면 **마스크 숫자가 가장 큰(범위가 좁은) 것** 선택

**예시: D-IP = 192.168.159.100**

| 번호 | 경로 | 포함 여부 | 마스크 | 선택? |
|------|------|---------|--------|-------|
| 1번 | 192.168.0.0/16 | ✅ | /16 | ❌ |
| **2번** | **192.168.159.0/24** | ✅ | **/24** | **✅** |
| 3번 | 0.0.0.0/0 | ✅ | /0 | ❌ |

> → **/24가 가장 정확하므로 2번 선택**

```
/8  → 넓음 (대략적)
/16 → 중간
/24 → 좁음 (정확)
/32 → 딱 하나의 호스트
```

### 기본 경로 (Default Route)

- `0.0.0.0/0` → "모르면 여기로 보내라"
- 라우팅 테이블에 없는 목적지는 모두 여기로

## 12. 라우터 모드 & 명령어

### 모드 구조

```
router>              ← 유저 모드 (일반 조회만)
    ↓ enable
router#              ← Privileged 모드 (관리자)
    ↓ configure terminal
router(config)#      ← 전역 설정 모드
    ↓ interface
router(config-if)#   ← 인터페이스 설정 모드
    ↓ CTRL+Z or exit
router#
```

### 핵심 명령어

```bash
# 호스트 이름 변경
hostname R1

# 인터페이스 IP 설정
interface g0/0
ip address 192.168.0.1 255.255.255.0
no shutdown

# 텔넷 설정 (vty 0~4 = 동시 5명 접속)
line vty 0 4
password 1234
login

# 설정 저장 (RAM → NVRAM)
copy running-config startup-config
```

### 메모리 개념

| 메모리 | 설정 파일 | 특징 |
|--------|---------|------|
| **RAM** | running-config | 현재 동작 중, 전원 꺼지면 사라짐 |
| **NVRAM** | startup-config | 비휘발성, 재부팅 후에도 유지 |

### 패스워드 설정

```bash
# 평문 저장 (위험)
enable password [패스워드]

# 암호화 저장 (MD5, 안전) ← 시험 포인트 ⭐
enable secret [패스워드]
```

> ⭐ `enable secret`이 설정되어 있으면 `enable password`는 무시됨

## 13. ACL (액세스 리스트)

> 💡 **ACL = 인터페이스(문) 앞에 세운 경비원**

### Standard ACL

- **출발지 IP(S-IP)만** 기준으로 차단/허용
- ACL 번호: **1 ~ 99**

```bash
access-list 5 permit host 192.168.159.131
access-list 5 deny any
```

### Extended ACL

- **출발지 IP + 목적지 IP + 포트 + 프로토콜** 기준
- ACL 번호: **100 ~ 199**

```bash
# 192.168.1.0/24에서 100.100.100.1:80 으로 가는 TCP 차단
access-list 150 deny tcp 192.168.1.0 0.0.0.255 host 100.100.100.1 eq 80
```

### ACL 번호 정리

| 번호 | 종류 |
|------|------|
| 1 ~ 99 | Standard ACL |
| 100 ~ 199 | Extended ACL |

### ACL 4가지 규칙 ⭐

| 규칙 | 내용 |
|------|------|
| **규칙 1** | 위에서부터 순서대로 검사 |
| **규칙 2** | 마지막에 `deny any` 자동 적용 (아무것도 안 걸리면 차단) |
| **규칙 3** | 새 줄은 항상 맨 뒤에 추가 (중간 삽입/삭제 불가) |
| **규칙 4** | 인터페이스에 ACL이 없으면 `permit any` (전부 허용) |

> ⚠️ 규칙 2 vs 4 혼동 주의!
> - 규칙 2: ACL은 있는데 해당 항목이 없음 → **deny**
> - 규칙 4: 인터페이스에 ACL 자체가 없음 → **permit**

### Wildcard 마스크

- 서브넷 마스크의 **반대** 개념
- `0` = 확인, `1` = 무시
- 예: `0.0.0.255` → 앞 3 옥텟 고정, 마지막 옥텟은 아무 값

## 14. 라우터를 이용한 네트워크 보안

### S-IP / D-IP

| 용어 | 의미 |
|------|------|
| **S-IP** (Source IP) | 출발지 IP |
| **D-IP** (Destination IP) | 목적지 IP |

> 라우터는 주로 **D-IP**를 보고 경로 결정, ACL 설정 시 **S-IP**도 활용

### 인터페이스 (Interface)

> 💡 **인터페이스 = 라우터의 문(포트)**

```
인터넷 ↔ [인터페이스] 라우터 [인터페이스] ↔ 내부망
```

### Ingress vs Egress 필터링 ⭐

| 구분 | 방향 | 검사 내용 |
|------|------|---------|
| **Ingress** | 외부 → 내부 (inbound) | S-IP, D-IP, 포트 |
| **Egress** | 내부 → 외부 (outbound) | S-IP (내부에서 나가는 패킷) |

#### Ingress 필터링 시 차단할 IP

- 사설 주소 (10.x, 172.16~31.x, 192.168.x)
- 루프백 주소 (`127.0.0.0/8`)
- 브로드캐스트 (`255.255.255.255`)
- 멀티캐스트 주소
- 연구용 주소 (E Class)

### Null Routing

> 특정 IP로 가는 패킷을 **가상의 쓰레기통(null 인터페이스)** 으로 버림

```bash
interface null 0
no ip unreachables
exit
ip route 211.1.1.1 255.255.255.255 null 0
```

> 💬 의미: "211.1.1.1로 가는 패킷은 그냥 버려라"

### Unicast RPF (Reverse Path Forwarding)

- 들어온 패킷의 **S-IP**가 실제로 그 인터페이스를 통해 들어왔는지 확인
- 라우팅 테이블에서 S-IP의 역방향 경로 검증
- 불일치 시 차단 → **IP 스푸핑 방어**

```
들어온 패킷 (S-IP: 10.10.10.10) → 라우팅 테이블 확인
→ "10.10.10.10은 eth0에서 와야 하는데, eth1에서 왔네?" → 차단
```

### 소스 라우팅

| 종류 | 설명 |
|------|------|
| **Loose** | 대략적인 경로만 지정 |
| **Strict** | 정확한 경로를 지정 |

> IPv4의 IP 옵션 필드를 사용 → 현재 보안상 이유로 비활성화 권장

### 라우팅 프로토콜 보안

| 방식 | 보안 수준 | 이유 |
|------|---------|------|
| 정적 라우팅 | ✅ 높음 | 공격자가 경로 조작 불가 |
| 동적 라우팅 | ⚠️ 인증 필요 | 라우팅 패스워드로 인증 설정 권장 |

## ✅ 최종 핵심 요약

| 항목 | 핵심 |
|------|------|
| 라우터 | D-IP 보고 최적 경로 선택 |
| 정적 | 수동, 보안 좋음 |
| 동적 | 자동, CPU 오버헤드 있음 |
| RIP | 홉 수 기준, 최대 15홉 |
| OSPF | Dijkstra, 빠른 수렴, 링크 상태 |
| BGP | AS 번호 기준, 인터넷 표준 EGP |
| Longest Prefix Match | 마스크 숫자 클수록 우선 |
| Standard ACL | S-IP만 (1~99) |
| Extended ACL | S-IP + D-IP + 포트 (100~199) |
| ACL 없으면 | permit any |
| ACL 있는데 해당 없으면 | deny any |
| Ingress | 외부→내부 필터링 |
| Null Routing | 쓰레기통 인터페이스로 버림 |
| RPF | S-IP 역방향 검증 |
