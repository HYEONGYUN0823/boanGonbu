# 네트워크 장비 정리

---

## 장비 계층 한눈에 보기

| 장비 | 계층 | 기준 |
|------|------|------|
| 랜카드(NIC) | L1 | 물리 연결 |
| 허브 | L1 | 없음 (전체 전송) |
| 리피터 | L1 | 신호 증폭 |
| 브리지 | L2 | MAC 주소 |
| 스위치 (L2~L7) | L2~L7 | MAC / IP / 포트 / 내용 |
| 라우터 | L3 | IP 주소 |
| 게이트웨이 | L7 | 프로토콜 변환 |

---

## 1. 랜카드 (NIC)

- PC와 네트워크를 연결하는 하드웨어
- **네트워크 드라이버(소프트웨어)** 가 있어야 동작
- 1계층 장비

---

## 2. 허브 (Hub)

- **1계층 장비** (물리 계층)
- 들어온 데이터를 **모든 포트에 그냥 전송** (무지성 브로드캐스트)
- 필터링 없음, 판단 없음, MAC 모름
- 충돌 많이 발생 → 비효율적

> 💡 허브 = 전체 카톡 단톡방

---

## 3. 리피터 (Repeater)

- **1계층 장비**
- 약해진 신호를 **증폭**해서 네트워크 거리 연장
- 현재는 허브/스위치에 기본 내장된 기능

---

## 4. 브리지 (Bridge)

- **2계층 장비** (데이터링크 계층)
- **MAC 주소**를 보고 판단해서 전송
- 네트워크 **충돌 도메인 분리** 목적
- Store-and-Forward 방식만 사용
- 보안성 향상 (네트워크 분산 구성)

> 💡 브리지 = 소형 스위치 느낌

---

## 5. 스위치 (Switch)

### 기본 개념

- **브리지의 진화형** (기본은 L2)
- 여러 포트(multiport) 보유
- MAC 주소를 기억하고 정확한 포트로만 전달

### 핵심 기능 5가지 (시험 필수)

| 기능 | 설명 |
|------|------|
| **Learning** | MAC 주소 학습 |
| **Flooding** | 모르면 전체 전송 |
| **Forwarding** | 알면 해당 포트로 전달 |
| **Filtering** | 필요 없는 포트는 차단 |
| **Aging** | 오래된 MAC 정보 삭제 |

---

### 스위치 종류 (OSI 계층 기준) ⭐ 매우 중요

| 종류 | 계층 | 기준 | 특징 |
|------|------|------|------|
| L2 스위치 | 2계층 | MAC 주소 | 일반적인 스위치 |
| L3 스위치 | 3계층 | IP 주소 | 라우팅 기능 포함 |
| L4 스위치 | 4계층 | 포트번호 | 서비스 종류별 분배 (SLB) |
| L7 스위치 | 7계층 | 내용(URL, 이메일 등) | 응용 세션 제어 |

#### L4 vs L7 차이 ⭐ 매우 중요

| 구분 | L4 스위치 | L7 스위치 |
|------|-----------|-----------|
| 기준 | TCP/UDP **포트번호** | **트래픽 내용** (URL, 이메일 등) |
| 예시 | HTTP(80), FTP(21) 구분 | 로그인 페이지 → A서버 등 |
| 수준 | 어디 서비스냐 | 내용이 뭐냐 |

---

### 스위칭 방식 (방식별 비교)

| 방식 | 동작 | 장점 | 단점 |
|------|------|------|------|
| **Store-and-Forward** | 전체 다 받고 → 검사 → 전송 | 오류 검출 우수 | 느림 |
| **Cut-Through** | 앞 48비트(MAC)만 보고 바로 전송 | 빠름 | 오류 검출 약함 |
| **Fragment-Free** | 앞 512비트 보고 전송 | 중간 수준 | 중간 수준 |

> 브리지 : Store-and-Forward 방식만 사용  
> 스위치 : 3가지 방식 모두 사용

---

## 6. 라우터 (Router) ⭐ 매우 중요

### 기본 개념

- **3계층 장비** (L1, L2, L3 모두 처리)
- **IP 주소** 기준으로 경로 결정
- 브로드캐스트 **차단** (다른 네트워크로 전달 안 함)
- 접근통제 목록(**ACL**) 기반 → DAC-ACL (객체 기반)

### 주요 기능

- 라우팅 (최적 경로 선택)
- 포워딩 (패킷 전달)
- 방화벽 기능
- VPN 기능
- NAT 기능 (사설IP ↔ 공인IP)

> 💡 패킷이 네트워크를 넘어갈 때 **링크 계층 주소(MAC)가 변경**됨

---

### 브리지 vs 라우터 비교 ⭐

| 구분 | 브리지 (L2) | 라우터 (L3) |
|------|-------------|-------------|
| 기준 | MAC 주소 | IP 주소 |
| 브로드캐스트 | **전달함** | **차단함** |
| 역할 | 같은 네트워크 내 분리 | 네트워크 자체를 나눔 |

---

## 7. 게이트웨이 (Gateway)

- **L7** (OSI 모든 계층 처리)
- 완전히 **다른 유형의 네트워크** 연결
- 프로토콜 및 데이터 형식 **변환** 수행
- 단점: 변환이 많아 **병목 현상** 발생 가능

> 💡 게이트웨이 = 언어 번역기

---

## 8. VLAN ⭐ 중요

### 기본 개념

- **물리적 선이 아닌 소프트웨어(논리적)** 로 네트워크를 나누는 기술
- 스위치를 이용해 **브로드캐스트 도메인** 분리
- 2계층 (데이터링크 계층) 기술

### 핵심 특징

- 같은 VLAN끼리만 통신 가능
- 다른 VLAN은 기본적으로 통신 불가 (보안)
- **VLAN 간 통신은 라우터 또는 L3 스위치** 필요
- 관리자가 그룹별로 다른 보안 정책 적용 가능

### VLAN 종류

| 종류 | 기준 | 설명 |
|------|------|------|
| Port 기반 | 스위치 포트 | 포트 단위로 VLAN 할당 (가장 많이 사용) |
| MAC 기반 | MAC 주소 | MAC으로 VLAN 소속 결정 |
| IP 기반 | IP 주소 | 네트워크 주소로 구분 |
| 프로토콜 기반 | 프로토콜 종류 | TCP/IP, IPX/SPX 등으로 구분 |

### VLAN 장점

- 브로드캐스트 도메인 분리 → 트래픽 감소
- 보안성 향상 (기밀성)
- 물리적 이동 없이 논리적으로 그룹 재구성 가능
- 비용 및 시간 절약

---

## 핵심 정리 ⭐⭐⭐

### 충돌 도메인 vs 브로드캐스트 도메인

| 구분 | 충돌 도메인 | 브로드캐스트 도메인 |
|------|-------------|---------------------|
| 의미 | 충돌이 발생할 수 있는 범위 | 브로드캐스트가 퍼지는 범위 |
| 나누는 장비 | **스위치, 브리지** | **라우터** |
| 허브 | 못 나눔 (전체 1개) | 못 나눔 |
| 스위치 | **나눔** (포트당 1개) | 못 나눔 |
| VLAN | - | **나눔** (스위치에서 가능) |

> 충돌 도메인 → **스위치**가 나눈다  
> 브로드캐스트 도메인 → **라우터** (또는 VLAN)가 나눈다

### 전체 흐름 요약

```
허브       → 그냥 다 뿌림 (L1)
브리지     → MAC 보고 충돌 도메인 나눔 (L2)
스위치     → 브리지 업그레이드, 포트별 충돌 도메인 분리 (L2~L7)
라우터     → IP 보고 브로드캐스트 도메인 나눔 (L3)
VLAN       → 스위치에서 브로드캐스트 도메인까지 나눔
게이트웨이 → 완전 다른 네트워크 연결, 변환 (L7)
```

### 비유로 이해하기

| 장비 | 비유 |
|------|------|
| 허브 | 단톡방 (전체한테 다 보냄) |
| 스위치 | 개인 DM (딱 맞는 사람한테 보냄) |
| 브리지 | 소형 스위치 |
| 라우터 | 건물 자체를 나눔 |
| L7 스위치 | 내용 읽고 맞는 서버로 보냄 |
| 게이트웨이 | 언어 번역기 |
