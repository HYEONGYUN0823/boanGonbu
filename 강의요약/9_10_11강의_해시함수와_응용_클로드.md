## 1. 일방향 해시함수 개요

### 기본 개념

> 💡 **비유 : 햄버거 → 재료 목록**
> 햄버거(메시지)를 보면 재료 목록(해시값)을 쉽게 만들 수 있어
> 하지만 재료 목록(해시값)만 보고 햄버거(메시지)를 똑같이 복원하는 건 불가능!
> 이게 바로 **일방향성**이야

```
입력 (임의 길이) → [해시함수] → 출력 (고정 길이)
"안녕하세요" (5글자)  →  해시  →  "a3f8c2..." (256bit 고정)
"안녕하세요!!!!" (8글자) →  해시  →  "b7d1e9..." (256bit 고정)
```

- 입력 : **D** (임의 길이), 출력 : **R** (고정 길이)
- h : D → R, |D| > |R| → **다대일 대응**
- 다대일이라서 **충돌은 반드시 존재**함 (수학적으로 피할 수 없음)

---

### 일방향 해시함수의 특징 ⭐⭐⭐⭐⭐

```
1. 임의 길이 메시지 → 고정 길이 해시값 출력
2. 해시값을 고속으로 계산 가능
3. 일방향성 : 해시값으로 원본 메시지를 역산 불가능
4. 메시지가 1비트라도 바뀌면 해시값이 완전히 달라짐 (무결성 확인 용도)
5. 충돌 내성 : 같은 해시값을 가진 다른 메시지를 찾기 어려워야 함
```

> 💡 **충돌(Collision)이란?**
> h(M1) = h(M2) 인데 M1 ≠ M2 인 경우
> 즉, 서로 다른 두 메시지가 같은 해시값을 가지는 것
> 충돌은 수학적으로 반드시 존재하지만, **찾기 어려워야** 안전한 해시함수!

---

### 해시함수 관련 용어 ⭐⭐⭐⭐⭐

| 용어 | 같은 말 |
|---|---|
| 일방향 해시함수 | 메시지 다이제스트 함수 = 메시지 요약함수 = 암호학적 해시함수 |
| 입력 메시지 | 프리이미지 (pre-image) |
| 출력 해시값 | 메시지 다이제스트 = 핑거프린트 |
| 무결성 | 완전성 = 보전성 |

## 2. 해시함수 보안 요구사항 ⭐⭐⭐⭐⭐

> 💡 **비유 : 지문 인식기**
> 좋은 지문 인식기는 3가지 조건을 만족해야 해
> 1. 지문(해시값)으로 원래 손가락(메시지)을 복원 못함 → 역상 저항성
> 2. 내 지문이랑 같은 지문을 가진 다른 손가락 못 만듦 → 제2역상 저항성
> 3. 지문이 같은 손가락 쌍 자체를 못 만듦 → 충돌 저항성

### 1) 역상 저항성 (프리이미지 저항성)

```
주어진 해시값 y = h(M)에서
원래 입력값 M을 역으로 찾는 것이 불가능해야 함

쉽게 : 해시값 보고 원본 못 찾음
```

### 2) 제2역상 저항성 (두 번째 역상 저항성, 약한 충돌 내성) ⭐⭐⭐

```
주어진 메시지 M이 있을 때
h(M) = h(M')을 만족하는 다른 메시지 M' (M ≠ M')을 찾는 것이 불가능해야 함

쉽게 : 원본 M과 해시값이 같은 다른 메시지 못 만듦
목적 : 메시지 위조 방지
```

> ⚠️ **공격 시나리오**
> 이브가 앨리스의 메시지 M과 h(M)을 가로챔
> h(M) = h(M')이 되는 위조 메시지 M'을 만들어서 밥에게 전송
> 밥은 해시값이 같으니 정상 메시지라고 착각!

### 3) 충돌 저항성 (충돌 회피성, 강한 충돌 내성)

```
h(x) = h(x')이 되는 임의의 두 입력값 x, x'을 찾는 것이 불가능해야 함

쉽게 : 해시값이 같은 메시지 쌍 자체를 못 만듦
→ 공격자가 어떤 정보도 없는 상태에서 공격
→ 약한 충돌 내성보다 공격 확률이 높음 → 강한 충돌 내성
```

### 세 가지 저항성 비교 ⭐⭐⭐⭐⭐

| 구분 | 목표 | 난이도 | 별칭 |
|---|---|---|---|
| 역상 저항성 | h(x) = y 에서 x 찾기 | 2^n | 프리이미지 저항성 |
| 제2역상 저항성 | h(x) = h(x') 인 x' 찾기 (x 주어짐) | 2^n | 약한 충돌 내성 |
| 충돌 저항성 | h(x) = h(x') 인 쌍 찾기 (자유) | **2^(n/2)** | 강한 충돌 내성 |

> 💡 **핵심 관계**
> 충돌 저항성 → 제2역상 저항성을 **보장함**
> (충돌 찾기도 못하면 당연히 특정 충돌도 못 찾음)
> 반대 방향은 보장 안 됨!

## 3. 전자서명에 이용되는 해시함수 특성 ⭐⭐⭐

| 특성 | 내용 | 목적 |
|---|---|---|
| 고속 계산 | 해시값을 빠르게 계산 | 성능 조건 |
| 약 일방향성 | h(M) = H에서 M을 역산 불가 | 안전성 |
| 강 일방향성 | h(M') = H인 M' ≠ M을 못 찾음 | 안전성 |
| 충돌 회피성 | h(M) = h(M')인 (M, M') 쌍 못 찾음 | **내부부정 방지** |

> 💡 **내부부정이란?**
> 앨리스가 M에 서명하고 나중에 "나는 M'에 서명했다"고 우기는 것
> 충돌 회피성이 있으면 M과 M'이 같은 해시값을 가질 수 없어서 이런 부정 방지!

## 4. 해시함수 종류

### 전용 해시함수 ⭐⭐⭐⭐⭐

> 💡 오늘날 사용되는 해시함수 대부분이 **MD4**를 기초로 설계됨

#### MD 시리즈 (Message Digest)

```
MD2 → MD4 → MD5 (진화 순서)

MD5 특징
- 메시지를 512bit 블록으로 나눔
- 128bit 다이제스트 출력
- 현재 128bit는 충돌 공격에 취약 → 사용 주의!
```

#### SHA 시리즈 (Secure Hash Algorithm) ⭐⭐⭐⭐⭐

```
설계 : NSA
배포 : NIST
용도 : DSS(서명전용)에 사용하기 위해 개발
```

| 구분 | SHA-1 | SHA-224 | SHA-256 | SHA-384 | SHA-512 |
|---|---|---|---|---|---|
| 계열 | SHA-1 | SHA-2 | SHA-2 | SHA-2 | SHA-2 |
| MD 길이 | **160** | 224 | 256 | 384 | **512** |
| 블록 길이 | 512 | 512 | 512 | **1024** | **1024** |
| 워드 길이 | 32 | 32 | 32 | **64** | **64** |
| 라운드 수 | 80 | 64 | 64 | 80 | 80 |
| 최대 메시지 | 2^64-1 | 2^64-1 | 2^64-1 | 2^128-1 | 2^128-1 |

> 💡 **외우는 팁**
> SHA-384, SHA-512 = 블록 1024 / 워드 64 → 나머지는 512 / 32
> SHA-2 = SHA-224, 256, 384, 512 묶음

#### SHA-3

```
SHA-1 대체를 위해 NIST 공모
2012년 KECCAK 알고리즘이 SHA-3으로 선정
AES와 같은 방식 (경쟁 방식으로 표준화)
AES 설계자 Joan Daemen이 참여!
```

#### 국산 해시함수

| 이름 | 설명 |
|---|---|
| **LSH** | 국가보안기술연구소, 2014년 개발, 경량 해시함수 (LSH-224/256/384/512) |
| **HAS-160** | KCDSA(국산 전자서명)에서 사용할 목적으로 개발 |

> ⚠️ **헷갈리지 말 것!**
> LEA = 국산 **블록 암호** (해시함수 아님!)

---

### 주요 해시 알고리즘 비교 ⭐⭐⭐

| 항목 | MD5 | SHA-1 | RIPEMD-160 |
|---|---|---|---|
| 다이제스트 길이 | **128bit** | **160bit** | 160bit |
| 처리 단위 | 512bit | 512bit | 512bit |
| 단계 수 | 64 (16번 × 4라운드) | 80 (20번 × 4라운드) | 160 (16번 × 5병행 라운드) |
| 최대 메시지 | **무한** | 2^64-1 bit | 2^64-1 bit |
| 엔디언 | Little | **Big** | Little |
| 출처 | 미국 | 미국 | **유럽** |
| 특이사항 | 충돌 취약 | - | **비트코인** 사용 |

## 5. 해시함수 공격 방법

### 생일 공격 (Birthday Attack) ⭐⭐⭐⭐⭐

> 💡 **비유 : 생일 파티**
> 23명만 모여도 그 중 두 명이 생일이 같을 확률이 50%가 넘어!
> 마찬가지로 해시값이 같은 두 메시지 쌍을 찾는 것도
> 생각보다 훨씬 적은 시도로 가능함!

```
목표 : 같은 해시값을 가진 2개의 메시지 쌍을 찾기
공격 대상 : 강한 충돌 내성 (충돌 저항성)
난이도 : 2^(n/2)  ← 전체 경우의 제곱근만큼 시도하면 됨!

예) SHA-512 (512bit) → 생일 공격 난이도 : 2^256
```

### 무차별 공격 (Brute Force) ⭐⭐⭐

```
목표 : 주어진 해시값과 같은 입력값 찾기
공격 대상 : 약한 충돌 내성 (제2역상 저항성)
난이도 : 2^n  ← 모든 경우를 다 시도해야 함

예) SHA-1 (160bit) → 무차별 공격 난이도 : 2^160
```

### 난이도 비교

| 공격 | 공격 목표 | 난이도 |
|---|---|---|
| 프리이미지 공격 | 역상 저항성 깨기 | 2^n |
| 제2프리이미지 공격 | 약한 충돌 내성 깨기 | 2^n |
| 생일 공격 | **강한 충돌 내성 깨기** | **2^(n/2)** |

> ✅ **핵심 : 충돌 공격(생일공격)이 프리이미지 공격보다 훨씬 쉬움!**

### 기타 해시함수 공격

| 공격 | 방법 |
|---|---|
| 일치블록 연쇄공격 | 미리 다양한 M'을 만들어두고, H(M)과 같은 해시값을 가진 것 골라 사용 |
| 중간자 연쇄공격 | 전체 해시값이 아니라 **중간 결과**에 대한 충돌쌍을 찾음 |
| 고정점 연쇄공격 | 특정 지점에서 임의의 블록을 중간에 삽입해도 해시값이 안 바뀌게 함 |
| 차분 연쇄공격 | 압축함수의 입출력 **차이**를 분석해서 충돌쌍 찾기 |

## 6. 일방향 해시함수로 해결할 수 없는 문제

```
✅ 해시함수로 가능한 것 : 변경/조작 탐지 (무결성)
❌ 해시함수로 불가능한 것 : 거짓행세 탐지 (인증)

→ 인증까지 하려면?
   MAC = 무결성 + 인증
   전자서명 = 무결성 + 인증 + 부인방지
```

## 7. SHA-512 상세 ⭐⭐⭐

```
출력 : 512bit 다이제스트
블록 : 1024bit 단위로 처리
최대 메시지 길이 : 2^128 bit 미만
```

**길이 필드와 패딩** ⭐⭐⭐
```
메시지 앞에 128bit 길이 필드 추가
→ 패딩 전 원래 메시지 길이를 비트수로 표현해서 저장
→ 공격자가 같은 해시값의 다른 메시지를 찾기 어렵게 하는 보안 요소
```

**SHA-512 안전성**
```
생일 공격 난이도 : 2^256
현재까지 취약점 보고 없음
하지만 SHA-1/2와 동일한 MD 구조 → 우려 있음
→ NIST가 2007년에 SHA-3 공모 시작
```

## 8. 메시지 인증 코드 (MAC) ⭐⭐⭐⭐⭐

> 💡 **비유 : 도장 + 봉인**
> 편지(메시지)에 도장(MAC)을 찍어서 보내는 것
> 도장을 찍으려면 도장(비밀키)이 필요하고
> 받는 사람도 같은 도장이 있어야 확인 가능
> 도장이 맞으면 → 내용이 안 바뀌었고(무결성) + 아는 사람이 보낸 것(인증)!

**정의**
```
입력 : 메시지 M + 공유 대칭키 K
출력 : 고정 길이 MAC 값

MAC = 무결성 + 인증 (변경 + 거짓행세 탐지 가능)
```

**특징**
```
✅ 전자서명보다 훨씬 빠름 (대칭키 사용하기 때문)
⚠️ 키 배송 문제 있음 (대칭키를 공유해야 함)
```

### MDC vs MAC

| 구분 | MDC | MAC |
|---|---|---|
| 키 | 없음 | 있음 (대칭키) |
| 제공 | 무결성만 | 무결성 + 인증 |
| 특징 | 누구나 계산 가능 | 키 가진 사람만 계산 가능 |

### MAC 수신 시 확인 가능한 것

```
MAC 값이 일치하면
✅ 메시지가 변경되지 않았다 (무결성)
✅ 메시지가 키를 아는 송신자로부터 왔다 (인증)
```

## 9. MAC 구현 사례

### HMAC ⭐⭐⭐⭐⭐

```
H = Hash (어떤 해시함수든 사용 가능)
MAC + Hash 조합

사용처 : TLS, IPSec(VPN), SET(전자상거래)
표준 : RFC 2104
```

### 축소 MAC

```
해시를 두 단계로 나눔
1단계 : 키 + 메시지 → 해시 → 중간 다이제스트
2단계 : 키 + 중간 다이제스트 → 해시 → 최종 다이제스트
```

### CBC-MAC / CMAC

```
CBC-MAC : 블록 암호의 CBC 모드와 유사한 방식으로 MAC 생성
         N개의 평문 블록 → 대칭키 N번 적용 → MAC 1개 생성

CMAC    : CBC-MAC과 같은 인증/무결성 제공
          수학적으로 더 안전
```

### CCM 모드 (Counter with CBC-MAC) ⭐⭐⭐ (무선보안)

> 💡 **비유 : 자물쇠(기밀성) + 도장(인증) 동시에!**
> CTR 모드(기밀성) + CBC-MAC(인증)을 합친 것

```
구성 요소 : AES 암호 + CTR 운영모드 + CBC-MAC 인증
제공 서비스 : 기밀성 + 인증(무결성)

cf) MAC만 있으면 인증만 → CCM은 기밀성도 추가!
    AES 이전 : WEP, WAP1에서 RC4(스트림 암호) 사용
```

### GCM 모드 → GMAC

```
CTR 모드 + 인증(GCM) 결합
암호문을 생성하면서 동시에 인증자(authenticator)도 생성
```

## 10. MAC에 대한 공격 - 재전송 공격

> 💡 **MAC의 한계**
> MAC은 메시지가 진짜인지 확인할 수 있지만
> **같은 메시지를 나중에 다시 보내는** 재전송 공격은 막지 못함!
>
> 비유 : 도장은 맞는데 이미 전에 써먹은 편지를 다시 보내는 것

### 재전송 공격 방어 방법 ⭐⭐⭐⭐⭐

| 방법 | 설명 | 단점 |
|---|---|---|
| 순서번호 | 메시지마다 1씩 증가하는 번호 포함 | 마지막 번호 기록 필요 |
| 타임스탬프 | 현재 시각을 메시지에 포함 | 송수신자 시계 동기화 필요 |
| 비표 (nonce) | 수신자가 일회용 랜덤값을 미리 전달 | 통신 데이터량 약간 증가 |
| 시도/응답 | 수신자가 도전값 보내고 송신자가 응답 | - |

## 11. MAC으로 해결할 수 없는 문제 ⭐⭐⭐

> 💡 **비유 : 도장이 2개인 상황**
> 앨리스와 밥이 같은 도장(비밀키)을 가지고 있으면
> 제3자 빅터에게 "이 도장은 밥이 찍었다"고 증명할 수가 없어!
> 앨리스도 같은 도장을 가지고 있으니까

### 제3자에 대한 증명

```
MAC에서는 앨리스와 밥 둘 다 같은 키를 가지고 있음
→ 제3자 빅터에게 "내가 아니라 상대방이 보냈다"고 증명 불가

해결책 : 전자서명 사용 → 제3자 증명 가능
```

### 부인방지

```
MAC에서는 앨리스와 밥 중 누가 보냈는지 판단 불가
→ 부인방지 불가능

해결책 : 전자서명 사용 → 부인방지 가능
```

### 기능 비교 정리 ⭐⭐⭐⭐⭐

| 기능 | 해시함수 | MAC | 전자서명 |
|---|---|---|---|
| 무결성 | ✅ | ✅ | ✅ |
| 인증 | ❌ | ✅ | ✅ |
| 부인방지 | ❌ | ❌ | ✅ |
| 제3자 증명 | ❌ | ❌ | ✅ |
| 속도 | 빠름 | 빠름 | 느림 |
| 키 | 없음 | 대칭키 | 비대칭키 |

## 📌 최종 핵심 암기 요약

```
✅ 해시함수 특징
   임의 길이 입력 → 고정 길이 출력
   일방향성 (역산 불가)
   1비트만 달라도 해시값 완전히 달라짐

✅ 3가지 저항성
   역상 저항성     : 해시값 → 입력 못 찾음        (2^n)
   제2역상 저항성  : M 주어졌을 때 같은 해시의 M' 못 찾음  (2^n)  ← 약한 충돌 내성
   충돌 저항성     : 같은 해시값 쌍 자체를 못 만듦  (2^n/2) ← 강한 충돌 내성

✅ 공격 방법
   생일 공격 → 강한 충돌 내성 공격 (2^n/2)
   무차별 공격 → 약한 충돌 내성 공격 (2^n)

✅ SHA 계열
   SHA-1  : 160bit / 블록 512 / 워드 32
   SHA-2  : SHA-224/256 (블록 512) / SHA-384/512 (블록 1024)
   SHA-3  : KECCAK, AES 설계자 참여

✅ 국산
   LSH    : 해시함수 (경량)
   HAS-160 : KCDSA용 해시함수
   LEA    : 블록 암호 (해시함수 아님!)

✅ MAC = 무결성 + 인증
   HMAC  = 해시함수 + MAC → TLS, IPSec, SET 사용
   CCM   = CTR(기밀성) + CBC-MAC(인증) → AES 기반

✅ 기능 비교
   해시함수 : 무결성만
   MAC      : 무결성 + 인증
   전자서명 : 무결성 + 인증 + 부인방지

✅ 재전송 공격 방어
   순서번호 / 타임스탬프 / 비표(nonce) / 시도응답
```
