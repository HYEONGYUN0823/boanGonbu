# 침해사고 대응 & 디지털 포렌식

> 정보보안기사 대비 강의 노트

---

## 📌 전체 흐름 한눈에 보기

```
해커가 침입
    ↓
침해사고 발생
    ↓
CERT / 대응팀 출동 (침해사고 대응)
    ↓
증거 수집 및 분석 (디지털 포렌식)
    ↓
보고서 작성 → 법적 대응
```

> **침해사고 대응** = 사고를 빨리 막고 피해를 줄이는 것  
> **디지털 포렌식** = 사고 이후 흔적을 찾아 법적 증거로 만드는 것

## 1. 해커(Hacker)

### 정의

- 원래 의미: **기술 개발에 정열을 쏟는 고급 기술자**
- 사이버 범죄자(Cracker, Attacker, Destroyer)와 엄격히 구분됨

### 수준별 분류 (Gilbert Alaverdian 5등급)

| 등급 | 명칭 | 설명 |
|------|------|------|
| 1 | **Elite** | 최고 수준의 전문 해커 |
| 2 | **Semi-Elite** | 상당한 기술 보유 |
| 3 | **Developed Kiddie** | 도구를 응용·제작 가능 |
| 4 | **Script Kiddie** | 남이 만든 도구만 가져다 씀, 기술 이해 부족 (⭐ 시험 단골) |
| 5 | **Lamer** | 거의 초보 수준 |

> 💡 **Script Kiddie 예시**: 캐나다의 15세 소년 *Mafiaboy*

### Hat 분류

| 분류 | 설명 |
|------|------|
| **White Hat** | 허가 하에 취약점을 찾아 알리는 윤리적 해커 (Ethical Hacker) |
| **Black Hat** | 악의적으로 시스템에 침입하는 해커 |
| **Gray Hat** | White ↔ Black 사이, 허가 없이 취약점을 찾기도 함 |

## 2. 해킹 기술의 변화

### 최근 트렌드 키워드

```
대중화  ·  은닉화  ·  분산화  ·  자동화
```

| 변화 | 설명 |
|------|------|
| **대중화** | 공격 도구가 보편화되어 낮은 실력으로도 시도 가능 |
| **은닉화** | 탐지를 피해 숨어서 공격 |
| **분산화** | 여러 시스템을 동시에 이용 (DDoS 등) |
| **자동화** | 프로그램이 자동으로 공격 수행 |

> **결과**: 공격에 필요한 지식 수준이 낮아지면서 침입자 수 급격히 증가

## 4. 침해사고 대응

### CERT (Computer Emergency Response Team)

- **정의**: 해킹·바이러스에 대응하는 보안기술 개발 및 서비스 기관
- **역할**
  - 공격 징후 탐지 및 분석
  - 사고 대응 방법 제시
  - 보안 기술 개발
  - 기관 간 협조

---

### ⭐ 사고대응 7단계 (매우중요)

```
① 사고 전 준비
        ↓
② 사고 탐지
        ↓
③ 초기 대응
        ↓
④ 대응 전략 체계화
        ↓
⑤ 사고 조사
        ↓
⑥ 보고서 작성
        ↓
⑦ 복구 및 해결
```

| 단계 | 명칭 | 주요 활동 |
|------|------|-----------|
| 1 | **사고 전 준비** | 대응 매뉴얼, 담당자, 도구 준비 |
| 2 | **사고 탐지** | 이상 트래픽, 수상한 로그인, 파일 변조 등 식별 |
| 3 | **초기 대응** | 초기 조사·기록, 대응팀 소집, 관련 부서 통지 |
| 4 | **대응 전략 체계화** | 수사기관 공조 여부 결정 |
| 5 | **사고 조사** | 피해 범위 파악, 확산 방지 방법 결정 |
| 6 | **보고서 작성** | 사고 내용 문서화 |
| 7 | **복구 및 해결** | 시스템 복구, 재발 방지 정책 수립 |

## 5. 디지털 포렌식

### 정의

> 다양한 디지털 장치에서 범죄와 관련된 자료를 발견·분석하여 법적 문제를 해결하는 작업

### 로카드 교환법칙 (Locard's Exchange Principle)

> 범죄자가 범죄 현장을 드나들 때 **반드시 흔적을 남긴다**  
> → 접촉한 두 물체 사이에는 반드시 교환이 일어남

| 물리 세계 | 디지털 세계 |
|-----------|-------------|
| 지문, 발자국, 머리카락 | 로그, 쿠키, 접속 기록, 삭제 파일 잔여 정보 |

---

### ⭐ 포렌식의 기본 원칙 (매우중요)

| 원칙 | 설명 |
|------|------|
| **정당성의 원칙** | 적법한 절차로 수집한 증거만 유효함. 위법하게 획득한 증거는 증거 능력 없음 |
| **재현의 원칙** | 동일한 환경에서 동일한 결과가 나와야 함 |
| **신속성의 원칙** | 휘발성 데이터는 빨리 사라지므로 신속하게 수집해야 함 (Live Response) |
| **연계 보관성의 원칙** | 증거의 수집 → 이송 → 분석 → 보관 → 제출 전 과정을 추적 가능해야 함 = **Chain of Custody** |
| **무결성의 원칙** | 증거가 중간에 변조되지 않았음을 해시값으로 입증 |

> 💡 **Live Response**: 종료된 시스템이 아닌 **살아있는 시스템**을 대상으로 포렌식 수행

---

### ⭐ 디지털 포렌식 절차

```
① 사전 준비
        ↓
② 증거 수집
        ↓
③ 보관 및 이송
        ↓
④ 조사 및 분석
        ↓
⑤ 보고서 작성
```

| 단계 | 설명 |
|------|------|
| **사전 준비** | 전문가 소집, 장비·소프트웨어 점검 |
| **증거 수집** | 디스크 이미징으로 무결성 보장하며 수집 (EnCase 도구 사용) |
| **보관 및 이송** | Evidence Bag에 보관, 이동 중 무결성 유지 |
| **조사 및 분석** | 디스크 브라우징, 파일 복구, 로그 분석 등 수행 |
| **보고서 작성** | 전 과정 문서화, 증거자료 타당성 확보 |

> ⚠️ **침해사고 대응 7단계**와 혼동 주의! 포렌식 절차는 **5단계**

---

### 연계 보관성 (Chain of Custody)

증거가 법정에 제출될 때까지:

```
누가 받았는지
    ↓
언제 받았는지
    ↓
어디에 보관했는지
    ↓
누구에게 넘겼는지
```

이 기록이 있어야 **"이 증거가 진짜다"** 라고 법정에서 인정됨

## 6. 디스크 복사 vs 디스크 이미징

| 구분 | 디스크 복사 | 디스크 이미징 |
|------|------------|--------------|
| 대상 | 파일 단위 | 모든 물리적 섹터 |
| 삭제 파일 복구 | ❌ 불가 | ✅ 가능 |
| 숨김/임시 파일 | ❌ 포함 안 됨 | ✅ 포함 |
| 포렌식 활용 | 부적합 | **적합** |

> 💡 포렌식에서는 반드시 **디스크 이미징** 방식을 사용해야 함  
> 💡 이미징 도구: **EnCase**, **FTK**

## 7. 디지털 증거 분석 기법

### MAC Time 분석 (타임라인 분석)

| 시간 | 설명 |
|------|------|
| **mtime** | 파일 생성 및 최근 수정 시간 |
| **atime** | 파일을 읽거나 실행한 시간 |
| **ctime** | 파일 속성이 변경된 시간 |

> 💡 Unix에서 `find` 명령으로 확인 가능

### 기타 분석 기법

| 기법 | 설명 |
|------|------|
| **시그니처 분석** | 파일 확장자를 변경해 숨긴 파일을 실제 형식으로 식별 |
| **해시 분석** | 파일 무결성 확인 (변조 여부 검출) |
| **로그 분석** | 접속·행동 기록 추적 |
| **프로세스 분석** | 실행 중이던 프로그램 확인 |

## 8. 휘발성 데이터

> 전원이 꺼지면 **사라질 수 있는** 데이터 → **가장 먼저 수집**

### 포함 항목

- RAM (메모리)
- 캐시 메모리
- 네트워크 연결 상태
- 실행 중인 프로세스 정보
- NIC 등 주변기기 온보드 메모리

### 수집 명령어

```bash
netstat     # 현재 네트워크 연결 상태 확인
arp         # ARP 캐시 (IP ↔ MAC 주소 대응) 확인
```

> 💡 **신속성의 원칙**이 여기서 적용됨

## 9. 데이터가 숨어 있는 곳

### 슬랙 공간 (Slack Space)

- 파일 크기와 저장 단위(클러스터) 사이에 생기는 **자투리 공간**
- 여기에 흔적 데이터가 남을 수 있음
- 종류: RAM 슬랙, 드라이브 슬랙, 파일 시스템 슬랙

### 삭제된 파일

- 파일을 지우면 **FAT / MFT(NTFS)** 등 파일 위치 정보만 삭제됨
- 실제 데이터는 덮어쓰기 전까지 디스크에 남아 있음 → **복구 가능**

### 인터넷 임시파일

| 종류 | 내용 |
|------|------|
| 임시 인터넷 파일 | Temporary Internet Files |
| 열어본 페이지 | History (방문 기록) |
| 임시 쿠키 파일 | Cookie |

## 10. 안티-포렌식 (Anti-Forensics)

> 디지털 데이터를 **조작·삭제·난독화**하여 포렌식 조사를 방해하는 행위

### 데이터 완전 삭제 방법

| 방법 | 설명 |
|------|------|
| **덮어쓰기 (Overwrite)** | 랜덤 이진값으로 공간을 여러 번 덮어씀 (NIST 권고: 7회 이상) |
| **소자 (Degaussing)** | 강한 자기장으로 저장 정보 제거 |
| **물리적 파괴** | 분쇄, 연마, 염산 처리 등으로 디스크 완전 파괴 |

### Overwrite가 필수인 상황 (재활용 전)

- PC를 수리 목적으로 외부 반출할 때
- 임대 PC 사용 기간 종료 후 반납할 때
- 고위 임원이 사용하던 PC를 재배분할 때

## 11. 역할 정리

### 1차 대응자 (First Responder)

> 현장을 처음 발견한 사람

**해야 할 것**: 현장 보호

**하면 안 되는 것**:
- 컴퓨터 전원 끄기 ❌
- 케이블 뽑기 ❌
- 마음대로 검색·조작하기 ❌

> 이유: 함부로 건드리면 **증거가 훼손**될 수 있음

### 최량 증거 원칙 (Best Evidence Rule)

> 복사본이 아닌 **원본**을 법정에 제출해야 함 (영미 증거법 원칙)

## 12. ⭐ 시험 핵심 암기 정리

### 포렌식 5대 원칙

```
정당성 - 적법한 절차로 수집
재현성 - 같은 조건 → 같은 결과
신속성 - 휘발성 데이터 빠르게 수집
연계보관성 - 증거 이동 경로 전체 추적 (Chain of Custody)
무결성 - 해시값으로 변조 여부 확인
```

### 포렌식 절차 5단계

```
사전 준비 → 증거 수집 → 보관·이송 → 조사·분석 → 보고서 작성
```

### 침해사고 대응 7단계

```
준비 → 탐지 → 초기 대응 → 전략 체계화 → 사고 조사 → 보고서 → 복구
```

### MAC Time

```
mtime = 수정 시간
atime = 접근 시간
ctime = 속성 변경 시간
```

### 해커 등급 순서

```
Elite > Semi-Elite > Developed Kiddie > Script Kiddie > Lamer
```
