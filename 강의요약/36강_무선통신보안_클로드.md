## 1. 무선 네트워크 기초

### 네트워크 유형 (범위 순서)

| 종류 | 기술 | 범위 |
|------|------|------|
| WPAN | 블루투스 | 개인 주변 (이어폰, 마우스 등) |
| WLAN | Wi-Fi (802.11) | 집, 카페, 학교 |
| WMAN | WiMAX 등 | 대도시, 대학 캠퍼스 |

### 무선 네트워크가 위험한 이유

| 요인 | 설명 |
|------|------|
| 채널 | 전파를 공기 중에 뿌리므로 누구나 수신 가능 → 도청 위험 |
| 이동성 | 기기 분실 시 정보 유출 위험 |
| 자원 부족 | 기기 성능이 낮아 강한 암호 알고리즘 적용이 어려움 |
| 접근 용이성 | 물리적 선 연결 없이 접속 가능 |

### 재밍 (Jamming)

> 강한 전파를 방사하여 특정 주파수 사용을 방해하는 공격

- 쉽게 말하면 **와이파이 방해 공격**
- 상대방이 소리를 질러서 대화를 방해하는 것과 동일한 개념

### 무선랜의 물리적 특성

| 특성 | 설명 |
|------|------|
| 감쇠 | 멀어질수록 신호 약해짐 |
| 간섭 | 다른 송신자의 신호와 섞임 |
| 다중경로 전달 | 벽에 반사되어 하나의 송신자로부터 여러 신호를 받을 수 있음 |

## 2. 무선랜 접근 제어

### 숨겨진 터미널 문제 ⭐

```
A ───────── B ─────────── C
  (A는 C 모름)    (C는 A 모름)
  
→ A와 C가 동시에 B에 전송 → 충돌 발생
```

> A와 C는 서로 존재를 모르기 때문에 동시에 전송해버림

### CSMA/CD vs CSMA/CA ⭐⭐

**CSMA/CD (유선 방식)**
- 충돌 발생 후 감지하여 재전송

**무선에서 CSMA/CD가 안 되는 이유 3가지**

1. 송신과 수신을 동시에 수행해야 충돌 감지가 가능한데, 무선 기기는 양방향 동시 동작이 어려움
2. 숨겨진 터미널 문제로 충돌이 발생해도 감지 불가
3. 거리로 인한 신호 감쇠로 다른 쪽의 충돌을 감지 못함

**→ CSMA/CA (무선 방식) 사용**
- 충돌을 **미리 회피**하는 방식
- 전송이 끝나도 바로 보내지 않고, **랜덤 시간 대기** 후 전송

> 📌 핵심: **CD = 충돌 감지 / CA = 충돌 회피**

## 3. 무선랜 표준 (IEEE 802.11)

| 표준 | 주파수 | 최대 속도 | 특징 |
|------|--------|-----------|------|
| 802.11b | 2.4GHz | 11Mbps | 최초 대중화, WEP 사용 → 보안 취약 |
| 802.11a | 5GHz | 54Mbps | b와 호환 안 됨 |
| 802.11g | 2.4GHz | 54Mbps | b와 호환됨 |
| **802.11i** ⭐ | - | - | **보안 강화 표준, WPA/WPA2 포함** |
| 802.11n | 2.4/5GHz | 600Mbps | MIMO (다중 안테나) |

### 802.11i 핵심 ⭐⭐

```
WPA (WPA1)  → TKIP 암호화 방식 사용
WPA2        → CCMP 암호화 방식 사용
```

| 인증 방식 | 설명 |
|-----------|------|
| WPA/WPA2-개인 (PSK) | 사전 공유 비밀키 사용, 인증 서버 없음 |
| WPA/WPA2-기업 | 802.1x/EAP + RADIUS 인증 서버 사용 |

## 4. 블루투스

### 구조

- **피코넷** : 기본 단위, 최대 8개 기기 (주국 1개 + 종국 7개)
- **스캐터넷** : 피코넷 여러 개를 연결한 구조 (한 피코넷의 종국이 다른 피코넷의 주국이 될 수 있음)

### 블루투스 공격 유형 ⭐

| 공격명 | 설명 |
|--------|------|
| 블루프린팅 (BluePrinting) | 블루투스 장치 탐색 |
| 블루스나핑 (BlueSnafing) | 장비의 파일 무단 접근 |
| 블루버깅 (BlueBugging) | 취약한 연결 악용 → 기기 임의 조작 |
| 블루재킹 (BlueJacking) | 익명으로 스팸 메시지 전송 |

## 5. 무선랜 보안 취약점

### ① 물리적 취약점

> 장비 자체에 직접 접근하는 공격

| 유형 | 설명 |
|------|------|
| 도난 및 파손 | 노출된 AP 도난/파손 → 저장 데이터 유출 |
| 설정 초기화 | 리셋 버튼으로 보안 설정 초기화 |
| 전원 차단 | 전원 케이블 분리 → 서비스 중단 |
| LAN 차단 | 내부 네트워크 케이블 분리 |

### ② 기술적 취약점

| 유형 | 설명 |
|------|------|
| 도청 | 암호화 미설정 시 전송 데이터 수집 가능 |
| DoS (재밍) | 방해전파로 와이파이 마비 |
| 불법 AP (Rogue AP) | 가짜 AP 설치 → 사용자 데이터 수집 |

### ③ 관리적 취약점

- 무선 장비 관리 미흡 (AP, 무선 랜카드 등)
- 전파 범위 관리 미흡 (외부로 신호 누출)

## 6. 무선 인증 기술

### SSID ⭐⭐

- **SSID** : 와이파이 이름 (최대 32바이트)
- 기본적으로 Broadcast로 공개됨
- 인증 없으면 SSID만 알아도 접속 가능

### 폐쇄 시스템 운영 ⭐⭐

> SSID를 숨겨서 모르는 사람이 접속하지 못하도록 설정

```
일반 방식 : AP가 SSID를 계속 브로드캐스트 → 누구나 목록에서 확인 가능
폐쇄 시스템 : SSID 숨김 + SSID=NULL 요청 차단 → 모르면 접속 불가
```

- `NetStumbler` 같은 툴로 스캔 방지 효과도 있음
- 단, 완전한 보안은 아님 (전문 툴로 탐지 가능)

### MAC 주소 인증

- 허용할 기기의 MAC 주소를 AP에 사전 등록
- 등록된 MAC만 접속 허용

| 단점 |
|------|
| 대규모 환경에서 관리 비용 증가 |
| MAC 주소는 복제(스푸핑) 가능 → 보안 우회 가능 |
| 기기 변경 시 재등록 필요 |

## 7. 무선 암호화 방식

### WEP ⭐⭐⭐

> **Wired Equivalent Privacy** - 현재 사용 금지 수준의 구식 암호화

**키 구조**

```
64비트  = 공유키 40비트  + IV 24비트
128비트 = 공유키 104비트 + IV 24비트
```

- 암호화 방식 : **RC4 (스트림 암호)**
- 무결성 검사 : **CRC-32 → ICV** (암호화 아님, 단순 오류 체크)

### WEP 패킷 암호화 흐름

```
1. 데이터 준비
2. CRC-32로 ICV(무결성 검사값) 생성
3. 공유키 + IV로 RC4 키스트림 생성
4. (데이터 + ICV)를 RC4로 암호화
5. IV + 암호문을 함께 전송
```

### WEP 복호화 흐름

```
1. IV 수신
2. 공유키 + IV로 RC4 복호화
3. 데이터 + ICV 추출
4. ICV 검사로 무결성 확인
```

### WEP 인증 절차

```
클라이언트                    AP
    │── 인증 요청 (SSID 포함) ──▶│
    │◀── 랜덤 문자열 전송 (평문) ─│
    │── RC4로 암호화하여 전송 ───▶│
    │◀── 성공 / 실패 통보 ────────│
```

### WEP 취약점 ⭐⭐⭐

| 취약점 | 설명 |
|--------|------|
| IV 재사용 | 24비트 IV는 짧아서 반복 사용 → 키 추측 가능 |
| 고정키 사용 | 키가 변경되지 않음 → 한번 유출되면 끝 |
| 단방향 인증 | AP가 사용자만 인증 (사용자는 AP 검증 안 함) → 가짜 AP에 속을 수 있음 |

### TKIP (WPA1)

> WEP의 임시 패치 버전 (SW 업그레이드로 적용)

| WEP 문제 | TKIP 개선 방법 |
|----------|---------------|
| IV 24비트 | **IV 48비트**로 확장 |
| CRC-32 무결성 | **MIC** (Message Integrity Code) 도입 |
| 재전송 공격 취약 | IV 순차 증가 규칙으로 보호 |

**TKIP 취약점** : 여전히 **RC4 사용** → 근본적인 취약점 잔존

### CCMP (WPA2) ⭐⭐⭐

> 처음부터 보안을 고려해 설계된 방식 (기존 하드웨어 무시)

| 항목 | 내용 |
|------|------|
| 암호화 알고리즘 | **AES (블록 암호)** |
| 키 길이 | **128비트** |
| IV 길이 | **48비트** |
| 운용 모드 | **CCM = CTR 모드 + CBC-MAC** |
| 특징 | 기밀성 + 무결성 동시 제공 |
| PN | 패킷 재전송(재연) 공격 방지 |

### 암호화 방식 비교 ⭐⭐⭐

| 구분 | Static WEP | Dynamic WEP | WPA (v1) | WPA2 (v2) |
|------|-----------|-------------|----------|-----------|
| 암호화 알고리즘 | RC4 | RC4 | RC4 (TKIP) | **AES (CCMP)** |
| IV 길이 | 24비트 | 24비트 | 48비트 | 48비트 |
| 암호 비트 | 64/128 | 128 | 128 | 128 |
| 보안 수준 | 하 | 중 | 상 | **최상** |

## 8. WPA / WPA2

### WPA vs WPA2 핵심 비교

| 항목 | WPA | WPA2 |
|------|-----|------|
| 암호화 | TKIP (RC4 기반) | **CCMP (AES 기반)** |
| 별칭 | - | RSN (Robust Security Network) |
| 특징 | 기존 하드웨어 지원 (SW 업그레이드) | 새 하드웨어 필요 |

> ⚠️ **WPA ≠ WAP** — WPA는 와이파이 보안, WAP는 옛날 모바일 인터넷 기술

### WPA-PSK (개인용) ⭐

- 인증 서버 없이 **사전 공유 키(PSK)** 로 인증
- 소규모 망 (가정, 소형 사무실) 에 사용
- **4-way Handshake** 로 키 생성 및 인증

**4-way Handshake 흐름**

```
AP            클라이언트
│── ① 난수(ANonce) 전송 ──────────▶│
│◀─ ② 난수(SNonce) + MIC 전송 ─────│
│── ③ GTK + MIC 전송 ─────────────▶│
│◀─ ④ 확인 응답 ────────────────────│

→ 이 과정에서 양쪽이 PTK(세션 키) 생성
```

> 비밀번호를 직접 주고받지 않고 키를 계산해서 인증함

**WPA-PSK 취약점** : 짧은 비밀번호 설정 시 **사전 공격(Dictionary Attack)** 으로 크랙 가능

## 9. 802.1x / EAP 인증

### 802.1x 개요

> 포트 기반 네트워크 접근 제어 표준 (기업망에서 사용)

**구성 요소 3가지**

| 역할 | 설명 |
|------|------|
| Supplicant (서플리컨트) | 접속을 요청하는 사용자/기기 |
| Authenticator (인증자) | AP 또는 스위치 |
| Authentication Server (인증 서버) | RADIUS 서버 |

**흐름**

```
사용자(Supplicant) → AP(Authenticator) → RADIUS 서버
                                         ↓
                                  인증 결과 반환
                                         ↓
                              AP가 포트 개방/차단
```

### EAP (Extensible Authentication Protocol)

> 다양한 인증 방식을 지원하는 인증 프레임워크

**캡슐화란?**

```
EAP 인증 데이터
     ↓
PPP / 802.3 / 802.11 등의 프로토콜 안에 담아서 전송

→ EAP = 내용물 / PPP, Wi-Fi = 택배 박스
```

**EAP 구조**

```
┌─────────────────────────┐
│  MD5 | TLS | TTLS | PEAP │  ← 인증 방식 종류
├─────────────────────────┤
│           EAP            │  ← 인증 프레임워크
├─────────────────────────┤
│  PPP | 802.3 | 802.11   │  ← 전송 수단
└─────────────────────────┘
```

### RADIUS 인증 흐름

```
클라이언트               AP               RADIUS 서버
    │── ① 접속 요청 ──────▶│
    │                      │── ② Challenge 전송 ──▶│
    │◀─ ② Challenge ───────│
    │── ③ 비밀번호 해시 ───▶│── ③ 해시 전달 ────────▶│
    │                      │                        │ ④ 검증
    │                      │◀─ ⑤ 암호화 키 전송 ─────│
    │◀─ ⑥ 암호화 키 전달 ──│
    │════ ⑦ 암호화 통신 ════│
```

### WEP/WPA-PSK vs 802.1x/EAP 차이점

| 항목 | WEP / WPA-PSK | 802.1x / EAP |
|------|--------------|--------------|
| 인증 서버 | 없음 | RADIUS 서버 있음 |
| 암호화 키 | 고정 또는 공유 | **세션마다 다른 키** |
| 중앙 관리 | 불가 | 가능 |
| 보안 수준 | 낮음 | 높음 |

## 10. WAP (무선 응용 프로토콜)

> 옛날 모바일 기기(피처폰)에서 인터넷을 사용하기 위한 기술

### WAP 구조

```
모바일 기기 ─── WTLS ───▶ WAP 게이트웨이 ─── TLS ───▶ 웹 서버
           (무선 구간)                         (유선 구간)
```

**문제점** : 게이트웨이 내부에서 잠깐 **평문(암호화 안 된 상태)** 으로 처리됨 → 게이트웨이가 보안 위협

### WTLS (Wireless TLS)

> SSL/TLS를 모바일 환경에 맞게 경량화한 프로토콜

| 기능 |
|------|
| 데이터 무결성 |
| 기밀성 (암호화) |
| 인증 (디지털 인증서) |
| DoS 방어 (재전송 메시지 탐지/제거) |

**구성**

```
HTTP
───────────────
Handshake | ChangeCipher | Alert   ← 상위 3개 프로토콜
───────────────
      WTLS                         ← 보안 계층
───────────────
     WDP                           ← 데이터 전송
```

### WAP2 (종단-대-종단 보안)

> 게이트웨이를 우회(Bypass)하여 모바일 → 서버 직접 TLS 연결

```
기존 WAP  : 폰 ─WTLS─▶ 게이트웨이(평문) ─TLS─▶ 서버  ← 게이트웨이에서 취약
WAP2      : 폰 ──────────────TLS──────────────▶ 서버  ← 종단간 암호화
```

## 11. 기기 인증 기술

### 인증 방식 보안 수준 비교

| 인증 방식 | 보안 수준 | 특징 |
|-----------|----------|------|
| ID/PW | 하 | 가장 기본, 부인방지 불가 |
| MAC 주소 인증 | 하 | MAC 스푸핑으로 우회 가능 |
| 암호 프로토콜 (802.1x 등) | 중 | 다양한 인증 방식 지원 |
| **PKI 인증서** | **최상** | **부인방지 가능**, 관리 복잡 |

### Challenge / Response 인증

> 문제를 내고 맞추면 인증 (OTP와 유사)

```
서버  → 랜덤 Challenge 전송
클라이언트 → 비밀키로 해시 계산 → 응답
서버  → 동일한 해시 값인지 확인
```

### 인증 수준 정리 ⭐⭐⭐

| Static WEP | Dynamic WEP | WPA v1 | WPA v2 |
|-----------|------------|--------|--------|
| RC4 / 24비트 IV | RC4 / 24비트 IV | RC4 + TKIP / 48비트 IV | **AES + CCMP / 48비트 IV** |
| 보안 하 | 보안 중/상 | 보안 상 | **보안 최상** |

## 12. RFID 보안

### RFID란?

> **Radio Frequency ID** - 무선 주파수로 태그를 식별하는 기술 (전자태그)
> 활용 : 물류, 주차장, 유통

| 종류 | 특징 |
|------|------|
| 수동형 (Passive) | 배터리 없음, 리더기 전파로 동작 |
| 능동형 (Active) | 자체 배터리 보유 |

### RFID 공격 유형

| 공격 | 설명 |
|------|------|
| 도청 | 리더기-태그 간 통신 도청 |
| 트래픽 분석 | 위치 추적 가능 |
| 위조 | 태그 데이터 조작 |
| DoS | 대량 질의로 리더기/태그 마비 |

### RFID 방어 기술 ⭐⭐

| 기법 | 설명 |
|------|------|
| Kill 명령어 | 태그 영구 비활성화 |
| Sleep/Wake | 안전한 곳에서만 활성화 |
| **Blocker 태그** ⭐ | 전용 IC 태그로 주변 태그 읽기 방해 |
| **Faraday Cage** ⭐ | 금속 박막으로 전파 차단 |
| **Jamming** ⭐ | 방해 신호로 불법 리더기 접근 차단 |

## 13. 모바일 보안 (iOS vs Android)

### 핵심 비교 ⭐⭐⭐

| 항목 | iOS | Android |
|------|-----|---------|
| 기반 OS | UNIX | Linux |
| 보안 통제권 | **애플** | **개발자/사용자** |
| 앱 실행 권한 | **관리자(root)** | **일반 사용자** |
| 앱 서명 주체 | 애플 CA | **개발자** |
| 샌드박스 | 엄격 (앱 간 통신 차단) | 상대적으로 자유로움 |
| 앱 정책 | **폐쇄적** | **개방적** |
| 앱 검수 | 애플이 사전 검수 | 등록 후 문제 발생 시 제거 |
| 보안 침해 | **탈옥 (Jailbreak)** | **루팅 (Rooting)** |

### 주요 보안 개념

**샌드박스** : 앱이 독립된 공간에서 실행 → 다른 앱/시스템에 영향 없음

**코드 서명** : 앱 배포 시 서명 → 설치 시 무결성 검증

**AndroidManifest.xml** ⭐

> 안드로이드 앱 명세서 파일

```xml
선언 내용:
- 앱의 패키지 이름
- 앱 구성 요소 및 실행 조건
- 필요한 권한 목록
- 필요한 하드웨어/소프트웨어 기능
```

## 14. BYOD 보안

> **Bring Your Own Device** - 개인 기기로 업무를 보는 환경

### 보안 기술 4가지 ⭐⭐

| 기술 | 적용 대상 | 설명 |
|------|-----------|------|
| **MDM** (Mobile Device Management) | 기기(Device) 전체 | 잠금, 원격 삭제, 암호화, 위치 추적 |
| **MAM** (Mobile Application Management) | 앱(Application)만 | 업무용 앱에만 보안 정책 적용 |
| **NAC** (Network Access Control) | 접속(Access) 제어 | 접속 전 보안 정책 준수 여부 검사 (ex. 백신 설치 요구) |
| **컨테이너화** | 데이터 분리 | 개인 데이터 ↔ 업무 데이터 분리 |

### 모바일 가상화 (Hypervisor)

> 하나의 기기에 OS 2개 운용

```
평상시   → 개인용 OS 사용
업무 시  → 업무용 OS로 전환

→ 개인 정보와 업무 정보를 OS 레벨에서 완전 분리
```

## ✅ 최종 핵심 암기 정리

### 암호화 방식 흐름

```
WEP (RC4, 취약)
  → WPA/TKIP (RC4 개선, 중간)
    → WPA2/CCMP (AES, 안전)
```

### 인증 수준

```
ID/PW < MAC 주소 < 암호 프로토콜 < PKI 인증서
```

### 자주 나오는 키워드 정리

| 키워드 | 연결 개념 |
|--------|----------|
| RC4 | WEP, WPA(TKIP) |
| AES | WPA2(CCMP) |
| IV 24비트 | WEP |
| IV 48비트 | TKIP, CCMP |
| ICV | WEP 무결성 (CRC-32, 암호화 아님) |
| MIC | TKIP 무결성 (암호화 맞음) |
| RADIUS | 802.1x 기업 인증 서버 |
| 4-way Handshake | WPA-PSK 키 교환 |
| Blocker 태그 | RFID 방어 |
| Faraday Cage | RFID 방어 |
| 폐쇄시스템 | SSID 숨김 운영 |
| Rogue AP | 가짜 무선 AP |
