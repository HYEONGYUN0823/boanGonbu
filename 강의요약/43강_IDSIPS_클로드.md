## 1. IDS 개요

**IDS (Intrusion Detection System)** = 침입 탐지 시스템

> 네트워크 또는 시스템에서 발생하는 의심스러운 행동을 **실시간으로 탐지**하여 관리자에게 경고 메시지를 전송하는 시스템

- **탐지(Detection)** 는 하지만 **차단(Block) 은 하지 않음**
- Passive + Reactive (수동적, 사후 대응)

```
[비유] IDS = 경보기
도둑이 들어오면 "삐뽀삐뽀!" 알려주지만, 직접 막지는 않음
```

### IDS 동작 단계 ⭐⭐⭐

| 단계 | 설명 |
|------|------|
| ① 감사 데이터 수집 | 로그, 네트워크 패킷 등 수집 |
| ② 데이터 가공/축약 | 수집 데이터 정제 |
| ③ 침입 분석 및 탐지 | 비정상 행위 탐지 or 오용 탐지 |
| ④ 보고 및 대응 | 관리자에게 알림 및 조치 |

## 2. IDS 탐지 방식: Misuse vs Anomaly

### 🔴 오용 탐지 (Misuse Detection) = 지식 기반 / 규칙 기반

> **"이미 알려진 해킹 패턴을 찾아내는 방식"**

- 기존 공격 패턴(Signature)을 DB에 저장해두고, 일치하면 침입으로 판단
- **전문가 시스템(추론 엔진)**, 상태전이 모델, 패턴 매칭 기법 사용
- 탐지 종류: 트로이목마, 백도어, 포트스캔 등 **알려진 공격**에 강함

```
[비유] 지명수배 사진을 들고 CCTV 비교하는 것
→ 아는 얼굴은 잘 잡음, 처음 보는 얼굴은 못 잡음
```

| 항목 | 내용 |
|------|------|
| 장점 | 정확도 높음, 오탐(FP) 적음 |
| 단점 | Zero-Day Attack 탐지 어려움, **미탐(FN) 증가** |
| 키워드 | Signature, 규칙, 패턴 DB |

---

### 🔵 비정상 탐지 (Anomaly Detection) = 행위 기반 / 통계 기반

> **"평소 행동과 다르면 수상하게 보는 방식"**

- 정상 행동 프로파일을 먼저 학습한 후, 벗어나면 침입으로 판단
- **통계 분석, 신경망 모델(AI)** 기반
- **Zero-Day Attack도 탐지 가능** (처음 보는 공격도 "이상함"으로 포착)

```
[비유] 평소엔 낮에만 로그인하는데, 새벽 3시 러시아에서 접속?
→ "어? 이상한데?" 하고 탐지
```

| 항목 | 내용 |
|------|------|
| 장점 | Zero-Day 탐지 가능, 신종 공격에 강함 |
| 단점 | **오탐(FP) 많음** (양치기 소년 문제) |
| 키워드 | 프로파일, 통계, AI, 신경망 |

---

### ✅ 비교 요약 ⭐⭐⭐

| 구분 | Misuse (오용 탐지) | Anomaly (비정상 탐지) |
|------|-------------------|----------------------|
| 기준 | 알려진 공격 패턴 | 정상 행동 기준 |
| 방식 | Signature / 규칙 | 통계 / AI |
| Zero-Day | ❌ 탐지 어려움 | ✅ 탐지 가능 |
| 단점 | 미탐(FN) 증가 | 오탐(FP) 증가 |
| 비유 | 아는 도둑만 잡는다 | 평소랑 다르면 의심 |

> 💡 **암기 팁**  
> - Misuse → "아는 도둑만 잡는다" → 신종 공격 못 잡음 → **미탐(FN)**  
> - Anomaly → "평소랑 다르면 의심" → 정상도 의심 → **오탐(FP)**

## 3. False Positive / False Negative

> ⭐⭐⭐ 시험 단골 개념

| 용어 | 설명 | 문제 |
|------|------|------|
| **False Positive (오탐)** | 정상인데 공격이라고 잘못 판단 | 양치기 소년, 가용성 저하 |
| **False Negative (미탐)** | 공격인데 정상이라고 놓침 | 실제 해킹 통과 |

```
오탐(FP): "정상 사용자인데 해커로 오해"
미탐(FN): "해커인데 정상인으로 오해"
```

- 탐지 범위를 **넓게** 잡으면 → FP 증가 (정상 사용자 피해)  
- 탐지 범위를 **좁게** 잡으면 → FN 증가 (실제 공격 놓침)

## 4. IDS 분류: HIDS vs NIDS

> ⭐⭐⭐ 데이터 수집원에 의한 분류

### 🖥️ HIDS (Host-based IDS)

> 호스트 내부의 **로그 파일, 감사 데이터**를 분석하는 IDS

```
[비유] VIP를 지키는 청원경찰 (한 사람만 전담)
```

| 항목 | 내용 |
|------|------|
| 장점 | **트로이목마, 백도어, 내부자 공격 탐지** 가능 |
| 단점 | 각 호스트마다 설치 필요, OS 종속적 |
| 수집 데이터 | 시스템 로그, 감사 파일 |

---

### 🌐 NIDS (Network-based IDS)

> 네트워크를 통해 전송되는 **패킷 정보**를 수집·분석하는 IDS

```
[비유] 음주단속 경찰 (도로 지나가는 차 전부 검사)
```

- **무차별 모드(Promiscuous Mode)** NIC에 설치 → 패킷 스니핑으로 수집
- 가장 널리 사용되는 IDS 방식

| 항목 | 내용 |
|------|------|
| 장점 | 초기 구축 비용 저렴, OS 독립적, 여러 시스템 동시 감시 |
| 단점 | **암호화된 패킷 분석 불가**, 스위칭 환경에서 비용 증가 |
| 수집 데이터 | 네트워크 패킷 |

---

### ✅ HIDS vs NIDS 비교

| 구분 | HIDS | NIDS |
|------|------|------|
| 감시 대상 | 단일 호스트 내부 | 네트워크 전체 트래픽 |
| 수집 데이터 | 시스템 로그 | 네트워크 패킷 |
| 내부자 공격 | ✅ 탐지 가능 | ❌ 어려움 |
| 암호화 패킷 | ✅ 분석 가능 | ❌ 분석 불가 |
| 비용 | 높음 (각 호스트 설치) | 낮음 |

---

## 5. NIDS 설치 위치

> ⭐ 한 대만 설치할 수 있다면?

```
인터넷 → [방화벽] → [NIDS] → 내부 서버
```

### ✅ 최적 위치: **방화벽 뒤쪽**

- 방화벽이 1차로 필터링 → IDS가 핵심 위협에만 집중 가능
- 방화벽 정책과 IDS 연동 용이
- **한 대만 설치 시 반드시 방화벽 뒤에 설치**

> ⚠️ HIDS는 특별한 위치 없이, **웹서버 등 중요한 시스템에 설치**

## 6. IDS 대응 방식 (수동/능동)

### 수동적 대응 (대부분의 IDS)

| 방법 | 설명 |
|------|------|
| SNMP Trap 전송 | 이벤트 발생 시 관리 서버에 자동 알림 전송 |
| 경고음 발생 | 관리자에게 청각적 알림 |
| 로그 저장 | 사건 기록 보관 |
| 관리자 화면 알림 | 깜빡이는 메시지 등 |

> 💡 **SNMP (Simple Network Management Protocol)**  
> 네트워크 장비(라우터, 스위치 등)의 상태를 모니터링하고 관리하는 프로토콜  
> **SNMP Trap** = 장비가 문제 발생 시 관리자에게 **먼저 알림을 보내는 기능**

### 능동적 대응 (IPS 방식)

| 방법 | 설명 |
|------|------|
| 패킷 차단 | 공격 패킷이 서버에 도달하지 못하게 차단 |
| ACL 수정 | 공격자 IP를 출입 통제 목록에서 차단 (`deny 1.2.3.4`) |
| 연결 강제 종료 | 공격 세션 즉시 끊기 |

> 💡 **ACL (Access Control List)**  
> 누가 어떤 자원에 접근할 수 있는지 정의하는 **출입 허용/차단 목록**  
> 공격 탐지 시 ACL에 공격자 IP를 추가하여 자동 차단

> ⚠️ 능동적 대응은 오탐(FP) 많을 경우 **정상 사용자까지 차단**될 위험 있음

## 7. IPS 개요 및 Firewall과 비교

**IPS (Intrusion Prevention System)** = 침입 방지 시스템

> IDS + 차단 기능 = **탐지하고 즉시 막는 시스템**

- Active + Proactive (능동적, 사전 대응)
- Zero-Day Attack에 대한 **실시간 침입 방지** 가능

```
[비유]
IDS = CCTV + 경보기     (보고 알림만)
IPS = 경찰 + 자동차단기  (보고 즉시 차단)
```

### ✅ Firewall / IDS / IPS 비교 ⭐⭐⭐

| 기능 | Firewall | IDS | IPS |
|------|----------|-----|-----|
| 패킷 차단 | ✅ | ❌ | ✅ |
| 패킷 내용 분석 | ❌ | ✅ | ✅ |
| 오용 탐지 | ❌ | ✅ | ✅ |
| 오용 차단 | ❌ | ❌ | ✅ |
| 이상 탐지 | ❌ | ✅ | ✅ |
| 이상 차단 | ❌ | ❌ | ✅ |

| 장비 | 역할 | 비유 |
|------|------|------|
| Firewall | 출입 통제 | 건물 입구 경비원 |
| IDS | 감시 + 경고 | CCTV + 경보기 |
| IPS | 감시 + 차단 | 경찰 + 자동문 잠금 |

## 8. 허니팟 (Honeypot)

> **"해커를 유인하기 위해 일부러 만들어둔 가짜 시스템"**

```
[비유] 인터넷 속 낚시터
해커가 중요한 시스템 대신 여기로 오게 유인함
```

### 목적

- 해커의 행동 패턴, 공격 기법, 목적 **분석**
- 중요 시스템으로부터 공격자 **유도**
- Zero-Day 공격 **사전 탐지** 가능

### 허니팟 요건 ⭐

- 해커에게 쉽게 노출되어야 함 (ex. `password.db`, 일급비밀 폴더)
- 실제 시스템처럼 구성요소를 갖추고 있어야 함 (단, 실제 데이터 아님)
- 접속하는 모든 사람의 정보를 관리자에게 전달해야 함

### 관련 개념

| 개념 | 설명 |
|------|------|
| **Honeynet** | 다수의 허니팟으로 구성된 네트워크 |
| **Enticement (유인)** | 합법적인 유인 행위 ✅ |
| **Entrapment (함정)** | 범죄를 유도하는 불법 행위 ❌ |

### 단점

- 법적 기준 미비
- 공격자를 더 자극할 수 있음

## 9. Snort 룰 분석

> ⭐⭐⭐ 실기 단골 / 대표적인 네트워크 기반 IDS 도구

### Snort란?

- **실시간 트래픽 분석, 패턴 매칭 기반** 침입탐지 도구
- 포트 스캔, CGI 공격, 오버플로우 등 다양한 공격 탐지 가능

---

### 룰 구조

```
action  protocol  src_ip  src_port  →  dst_ip  dst_port  (options)
```

#### 주요 Action

| Action | 설명 |
|--------|------|
| `alert` | 경고 발생 + 로그 저장 |
| `log` | 로그만 저장 |
| `pass` | 해당 패킷 무시 |

#### 주요 Options

| 옵션 | 설명 |
|------|------|
| `msg` | 경고 메시지 이름 |
| `content` | 패킷에서 찾을 **특정 문자열** |
| `nocase` | 대소문자 구분 안 함 |
| `offset` | 검색 시작 오프셋 |
| `depth` | 검사할 바이트 수 |
| `flags` | TCP 플래그 조건 |
| `sid` | 룰 고유 ID |

---

### 룰 예시 1: FTP root 로그인 탐지 ⭐

```snort
alert tcp any any -> 10.10.10.0/24 21 (msg:"FTP root user access"; content:"User root"; nocase; sid:100100;)
```

| 항목 | 내용 |
|------|------|
| `alert` | 경고 발생 |
| `tcp` | TCP 프로토콜 |
| `any any` | 출발지 IP/포트 무관 |
| `-> 10.10.10.0/24 21` | 목적지 21번 포트 (FTP) |
| `content:"User root"` | "User root" 문자열 포함 시 탐지 |
| `nocase` | 대소문자 무시 |

> FTP에서 root 계정으로 로그인 시도하는 패킷 탐지

---

### 룰 예시 2: NULL Scan 탐지 ⭐

```snort
alert tcp any any -> 10.10.10.0/24 any (msg:"NULL Scan Detect"; flags:!UAPRSF; sid:100100;)
```

| 항목 | 내용 |
|------|------|
| `flags:!UAPRSF` | URG, ACK, PSH, RST, SYN, FIN **모두 비활성화** |

> NULL Scan = TCP 플래그가 아무것도 없는 이상한 포트 스캔  
> → 플래그가 하나도 없는 패킷을 탐지

---

### 포트 범위 표현

| 표현 | 의미 |
|------|------|
| `1:1023` | 1 ~ 1023 포트 |
| `:1024` | 1024 이하 포트 |
| `1024:` | 1024 이상 포트 |
| `!192.168.1.0/24` | 해당 IP 대역 **제외** |

## 10. 기타 개념 정리

### 포트 스캔 (Port Scan)

> 해커가 열린 포트를 찾기 위해 다수의 포트에 순서대로 접속 시도하는 행위

```
[비유] 도둑이 아파트를 돌아다니며 "이 문 잠겼나?" 확인하는 것
```

- 짧은 시간에 다수 포트 접속 시도 → IDS 탐지 가능

### CGI 공격

> 웹서버의 CGI 프로그램 취약점을 이용한 공격

- 입력 값 검증이 부실한 CGI에 악성 명령 삽입
- IDS에서 `content` 옵션으로 특정 공격 문자열 탐지

### FDS (이상 금융거래 탐지 시스템)

> 전자금융거래에서 단말기 정보, 접속 정보, 거래 내용을 분석하여 이상 거래를 탐지·차단하는 시스템
