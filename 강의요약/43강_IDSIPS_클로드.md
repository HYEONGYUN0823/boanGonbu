# 🔍 침입탐지/방지시스템 (IDS / IPS) 완전 정리

> 정보보안기사 대비 강의 노트  
> ⭐ = 시험 출제 빈도

---

## 1. IDS 개요

### 정의

- **실시간**으로 침입을 탐지하여 관리자에게 경고 메시지를 전송하는 시스템
- 탐지만 할 뿐 직접 차단하지는 않음 → **Passive + Reactive**
- 허니팟 개념으로 점차 확장/변화 중 (Decoy 기반 IDS)

> **비유** 🚨  
> 경보기. 도둑이 들어오면 "삐뽀삐뽀!" 알려주지만 직접 막지는 못함.

### IDS 장점

- 내부 사용자의 오·남용 탐지 및 방어 가능

---

## 2. IDS vs IPS 핵심 차이 ⭐⭐⭐

| 구분 | IDS | IPS |
|---|---|---|
| 의미 | 침입 탐지 시스템 | 침입 방지 시스템 |
| 역할 | **탐지만** 수행 | 탐지 + **차단** 수행 |
| 방식 | Passive + Reactive | Active + Proactive |
| 특성 | 탐지적, 사후 조치 | 예방적, 사전 조치 |
| 비유 | 경보기 | 경찰 + 자동문 잠금 |
| 공격 차단 | ❌ | ✅ |

> **한 줄 비유**  
> IDS = "알려만 준다"  
> IPS = "알려주고 직접 막는다"

---

## 3. IDS 실행 단계 ⭐⭐⭐

```
1단계. 감사 데이터(Audit Data) 수집
         ↓
2단계. 데이터 가공 및 축약
         ↓
3단계. 침입 분석 및 탐지
   ├── 비정상 행위 탐지 (Anomaly)
   └── 오용 탐지 (Misuse)
         ↓
4단계. 보고 및 대응
```

---

## 4. 탐지 방법 분류 (가장 중요 ⭐⭐⭐)

> 선생님 강조:  
> **탐지방법** → 행위기반(Anomaly) / 지식기반(Misuse)  
> **데이터 수집원** → 호스트 로그파일(HIDS) / 네트워크 패킷(NIDS)

---

### ① Misuse Detection (오용 탐지 / 지식기반) ⭐⭐⭐

**한마디**: "이미 알려진 해킹 패턴을 DB에 저장해두고 비교하는 방식"

| 항목 | 설명 |
|---|---|
| 기준 | 알려진 공격 패턴 (Signature) |
| 방식 | 비정상 패턴을 DB에 등록 후 비교 |
| 키워드 | Signature, 규칙, 지식, Expert System, 상태전이 모델, 패턴 매칭 |
| Zero-Day | ❌ 탐지 어려움 |
| 대표 단점 | **F(-) 미탐 (False Negative) 증가** |

> **비유** 🕵️  
> 수배전단 비교. 얼굴 사진 아는 범인만 잡을 수 있음. 처음 보는 범인은 못 잡음.

**Misuse 탐지 시그니처 예시**

- root 사용자 이름으로 Telnet 로그인 시도
- 시스템 프로그램 복제 시도
- 패스워드 파일 복제 시도

**Misuse 종류**

- **Expert System (추론엔진)**: IF~THEN 규칙으로 침입 여부 판단
- **상태전이 모델**: 공격 상태를 미리 상태전이도로 표현하고 실시간으로 시스템 상태 변화를 추적
- **패턴 매칭**: 알려진 공격 시나리오를 DB에 저장 후 발생 사건과 비교

**Misuse 장단점**

| 장점 | 단점 |
|---|---|
| 알려진 공격 정확하게 탐지 | 새로운 공격(Zero-Day) 탐지 어려움 |
| 오탐(FP)이 상대적으로 적음 | 패턴 DB 지속 관리 필요 |
| 트로이목마·백도어 탐지 가능 | FN(미탐) 증가 |

---

### ② Anomaly Detection (비정상 탐지 / 행위기반) ⭐⭐⭐

**한마디**: "정상 행동을 먼저 학습해두고, 거기서 벗어나면 의심하는 방식"

| 항목 | 설명 |
|---|---|
| 기준 | 정상 행동 프로파일 |
| 방식 | 정상 데이터 수집 → 통계/AI로 이탈 감지 |
| 키워드 | 통계, 프로파일, 신경망, 인공지능 |
| Zero-Day | ✅ 탐지 가능 |
| 대표 단점 | **F(+) 오탐 (False Positive) 증가 (양치기소년)** |

> **비유** 👀  
> 평소랑 너무 다르면 수상하게 보는 것.  
> 항상 낮에만 접속하던 사람이 새벽 3시 러시아에서 접속하면 의심.

**프로파일 예시**

- "특정 호스트의 CPU 사용률이 80% 이상인 시간이 5분 이상 지속되지 않는다"
- "근무시간 이후 접속 시 합법적 로그인이라도 침입으로 간주"

**Anomaly 종류**

- **통계적 분석**: 평균·분산 기반으로 이탈 탐지
- **예측 가능한 패턴 생성**: 일정한 패턴 가설 기반
- **신경망 모델**: AI 적응 학습으로 이상 행동 탐지

**Anomaly 장단점**

| 장점 | 단점 |
|---|---|
| Zero-Day Attack 탐지 가능 | 오탐(FP) 증가 |
| DB 관리가 Misuse보다 용이 | 실시간 탐지 어려움 (시간 지연) |
| 침입 외 시스템 문제도 발견 가능 | 임계치 설정(기준 조정)이 어려움 |

---

### Misuse vs Anomaly 비교 ⭐⭐⭐

| 구분 | Misuse (오용) | Anomaly (비정상) |
|---|---|---|
| 기준 | 알려진 공격 패턴 | 정상 행동 기준 |
| 방식 | Signature / DB 비교 | 통계 / AI |
| Zero-Day | ❌ 어려움 | ✅ 가능 |
| 장점 | 정확함, 오탐 적음 | 신종 공격 탐지 가능 |
| 단점 | **미탐(FN) 증가** | **오탐(FP) 증가** |
| 키워드 | 시그니처, 규칙, 지식 | 통계, 프로파일, AI |

> ⚠️ **중요 포인트**  
> Misuse와 Anomaly는 **탐지 방식**이지 장비 종류가 아님!  
> IDS / IPS 둘 다 이 방식을 내부적으로 사용할 수 있음.

## 5. False Positive / False Negative ⭐⭐⭐

| 구분 | 뜻 | 예시 |
|---|---|---|
| **False Positive (오탐, FP)** | 정상인데 공격으로 잘못 판단 | 밤새 과제하다 트래픽 늘었는데 공격 경보 울림 |
| **False Negative (미탐, FN)** | 공격인데 정상으로 놓침 | 실제 해킹인데 "정상 사용자겠지" 하고 넘어감 |

```
탐지 범위 넓게 → FP 증가 (선의의 피해)
탐지 범위 좁게 → FN 증가 (공격 놓침)
```

**암기 꿀팁**

- Misuse → **FN(미탐) 증가** (모르는 공격 패턴 놓침)
- Anomaly → **FP(오탐) 증가** (정상도 이상하다고 잡음, 양치기소년)

> ⚠️ Misuse / Anomaly 둘 다 FP, FN 모두 발생 가능하지만  
> **대표적으로 Misuse는 미탐, Anomaly는 오탐이 많다**는 게 핵심!

## 6. 대응 방식에 따른 분류

| 방식 | 설명 |
|---|---|
| **수동적 대응** | 탐지 후 관리자에게 알림 (대부분의 IDS가 이 방식) |
| **능동적 대응** | 직접 차단, ACL 수정, 연결 끊기 (IPS 방식) |

> 능동적 대응 시 오탐이 많으면 시스템 가용성 저하 위험  
> → 그래서 IDS는 보통 수동적 대응을 선택

**수동적 대응 방법**

- SNMP Trap 전송
- 경고음 발생
- 로그 저장
- 관리자 화면 알림

## 7. 데이터 수집원에 의한 분류 ⭐⭐⭐

### ① NIDS (Network-based IDS)

- 네트워크를 통해 전송되는 **패킷을 감시**
- **무차별 모드(Promiscuous Mode)** 로 동작하는 NIC에 설치
- 패킷 스니핑으로 데이터 수집

> **비유** 🚔  
> 음주단속 경찰. 도로(네트워크)를 지나가는 차(패킷)를 검사함.

| 장점 | 단점 |
|---|---|
| 구축 비용 저렴 (몇 곳에만 설치) | 암호화된 패킷 분석 어려움 |
| 운영체제에 독립적 | 스위칭 환경에서 비용 증가 가능 |
| 가장 널리 사용됨 | 소규모 세그먼트에선 단일 호스트 모니터링으로 축소 |

---

### ② HIDS (Host-based IDS)

- 개별 호스트의 **로그 파일, 시스템 감사 자료** 분석
- 단일 호스트 또는 다중 호스트(Multi-host) 기반

> **비유** 🛡️  
> VIP 지키는 청원경찰. 특정 서버 한 대만 집중 감시.

| 장점 | 단점 |
|---|---|
| 트로이목마, 백도어, 내부자 공격 탐지 강함 | 각 호스트마다 설치 필요 |
| 세밀한 분석 가능 | 다양한 OS 지원 필요 (OS 종속적) |
| - | 관리 복잡, 상용·공개 제품 찾기 어려움 |

---

### HIDS vs NIDS 비교 ⭐⭐⭐

| 구분 | HIDS | NIDS |
|---|---|---|
| 감시 대상 | 개별 호스트 내부 로그 | 네트워크 패킷 |
| 장점 | 내부자·백도어·트로이목마 탐지 강함 | 구축 비용 저렴, OS 독립적 |
| 단점 | 호스트마다 설치 필요 | 암호화 패킷 분석 불가 |

> anomaly detection은 **정상적인 행동**이 무엇인지를 정의하고  
> signature detection은 **적절하지 않은 행동**이 무엇인지를 정의한다.

## 8. NIDS 최적 배치 위치 ⭐⭐

```
인터넷
  │
[방화벽]        ← 1차 차단
  │
[NIDS] ← ★ 한 대만 설치한다면 여기! (방화벽 뒤)
  │
[내부 서버]
```

**왜 방화벽 뒤?**

- 방화벽이 1차로 걸러준 뒤 IDS가 진짜 위협에 집중 가능
- 공격 방어 정책과 방화벽 연동이 가장 효과적
- 방화벽 앞은 너무 많은 공격 데이터로 정작 중요한 공격 대처 어려움

> HIDS는 특별한 위치 없이 **웹서버 등 중요 시스템**에 직접 설치

## 9. SNMP / ACL 수정 (IDS 대응 도구)

| 개념 | 설명 |
|---|---|
| **SNMP** | 네트워크 장비 관리용 프로토콜 (CPU, 트래픽, 장비 상태 모니터링) |
| **SNMP Trap** | 장비가 문제 발생 시 관리자에게 먼저 알림 보내는 것 (5분 대기조 인터럽트 느낌) |
| **ACL 수정** | 공격 탐지 시 공격자 IP를 자동으로 차단 목록에 추가 |

```
IDS 공격 탐지
  ├── SNMP Trap 전송 → 관리자 알림
  ├── 경고 로그 저장
  └── ACL 수정 → 공격자 IP 차단 (능동 대응)
```

## 10. 허니팟 (Honeypot) ⭐⭐

### 개념

- 해커를 유인하는 **미끼 시스템** (인터넷 속 낚시터)
- 실제 자료를 가진 척하면서 해커의 행동·공격기법·목적을 분석
- 허니넷(HoneyNet) = 허니팟 여러 개로 구성된 네트워크

### 허니팟 설계 목표

- 공격 성향이 있는 자를 중요한 시스템으로부터 다른 곳으로 끌어내도록 설계한 **유인(유도) 시스템**

### 허니팟의 요건

- 해커에게 쉽게 노출되어야 함 (ex: `password.db`, 일급비밀 폴더)
- 실제 시스템처럼 구성요소를 갖춰야 함 (단, 실제 데이터는 없음)
- 통과하는 모든 패킷을 감시
- 접속자 정보를 관리자에게 전달

### 허니팟 특징

- 고의적으로 취약점 포함 → **Zero-Day 공격 사전 탐지** 가능 (예방통제 기술)
- 보통 **DMZ** 또는 **스크린드 서브넷**에 위치

### Enticement vs Entrapment ⭐

| 구분 | 뜻 | 합법 여부 |
|---|---|---|
| **Enticement (유인)** | 자연스럽게 미끼로 유도 | ✅ 합법, 윤리적 |
| **Entrapment (함정)** | 범죄하도록 적극 유도 | ❌ 불법, 비윤리적 |

### 허니팟 단점

- 법적 기준 미비
- 공격자를 더 자극할 수 있음

## 11. FDS (이상금융거래탐지시스템) ⭐

- 전자금융거래에서 **단말기 정보 + 접속 정보 + 거래 내용**을 종합 분석
- 의심 거래 탐지 및 이상금융거래 차단
- 카드/은행에서 "이 거래 이상한데?" 하고 잡는 시스템

## 12. Snort ⭐⭐⭐

### 개념

- 대표적인 **오픈소스 NIDS 도구**
- 실시간 패킷 분석, 프로토콜 분석, 내용 검색/매칭, 룰 기반 탐지
- 탐지 가능한 공격: 포트 스캔, 오버플로우, CGI 공격, OS 확인 시도 등

---

### Snort 룰 구조 ⭐⭐⭐

```
action  protocol  src_ip  src_port  direction  dst_ip  dst_port  (options)
```

**예시**

```
alert  tcp  any  any  ->  192.168.159.0/24  80
```

| 필드 | 예시 값 | 의미 |
|---|---|---|
| action | `alert` | 경고 발생 + 로그 저장 |
| action | `log` | 패킷을 로그에만 저장 |
| protocol | `tcp` | TCP 패킷만 검사 |
| src_ip | `any` | 출발지 IP 무관 |
| src_port | `any` | 출발지 포트 무관 |
| direction | `->` | 단방향 (좌→우) |
| dst_ip | `192.168.0.0/24` | 목적지 네트워크 |
| dst_port | `80` | 목적지 포트 |

---

### IP 부정 연산자

```
!10.10.10.0/24  → 해당 네트워크 제외한 모든 IP
```

---

### 포트 범위 표기법 ⭐⭐⭐

| 표기 | 의미 |
|---|---|
| `1:1023` | 1번 ~ 1023번 포트 |
| `:1024` | 1024번 이하 포트 |
| `1024:` | 1024번 이상 포트 |

---

### 주요 룰 옵션 (바디)

| 옵션 | 설명 |
|---|---|
| `msg` | 이벤트 이름 (경고 메시지) |
| `content` | 패킷 안에서 찾을 문자열 |
| `nocase` | 대소문자 구분 안 함 |
| `offset` | 문자열 검색 시작 오프셋 |
| `depth` | offset부터 검사할 바이트 수 |
| `flags` | TCP 플래그 조건 |
| `sid` | 룰 고유 ID |

---

### 예시 1: FTP root 로그인 시도 탐지

```
alert tcp any any -> 10.10.10.0/24 21 (msg:"FTP root user access"; content:"User root"; nocase; sid:100100;)
```

| 부분 | 의미 |
|---|---|
| `tcp` | TCP 프로토콜 |
| `-> 10.10.10.0/24 21` | 목적지 21번 포트 (FTP) |
| `content:"User root"` | 패킷 안에 "User root" 문자열 탐지 |
| `nocase` | 대소문자 무시 |

> FTP에서 root 계정으로 로그인 시도 시 경고 발생

---

### 예시 2: NULL Scan 탐지

```
alert tcp any any -> 10.10.10.0/24 any (msg:"NULL Scan Detect"; flags:!UAPRSF; sid:100100;)
```

| 부분 | 의미 |
|---|---|
| `flags:!UAPRSF` | URG, ACK, PSH, RST, SYN, FIN 플래그가 **모두 설정되지 않은** 패킷 탐지 |

> TCP 플래그가 아무것도 없는 비정상 패킷 → NULL 포트 스캔 탐지

---

## 13. IPS 개요 ⭐⭐

### 정의

- **능동형 보안 솔루션**
- IDS는 탐지적·사후 조치, IPS는 **예방적·사전 조치**
- 탐지 즉시 패킷 차단, 연결 끊기, ACL 수정 등 수행

### IPS의 필요성

- **Zero-Day Attack** 실시간 대응
- 기존 보안 제품(방화벽, IDS)의 한계 보완
- 실시간 침입 방지

### IPS 위치

```
공격 트래픽 → [IPS] → 서버
```

IPS는 **중간에서 직접 막음** (인라인 배치)

> IDS는 트래픽 흐름 옆에서 감시만, IPS는 반드시 통과해야 함

---

### 바이러스 월 (Virus Wall)

- 메일/파일 공유로 전파되는 **악성코드(바이러스·웜 등)** 제거
- 방화벽과 유사한 개념이지만 **패킷 내용(악성코드 여부)** 에 집중

| 장비 | 역할 |
|---|---|
| Firewall | 출입 자체 통제 (IP, 포트 기준) |
| Virus Wall | 파일/메일 안의 악성코드 검사 |

---

## 14. Firewall / IDS / IPS 비교 ⭐⭐⭐

| 기능 | Firewall | IDS | IPS |
|---|---|---|---|
| 패킷 차단 | ✅ | ❌ | ✅ |
| 패킷 내용 분석 | ❌ | ✅ | ✅ |
| 오용 탐지 (Misuse) | ❌ | ✅ | ✅ |
| 오용 차단 | ❌ | ❌ | ✅ |
| 이상 탐지 (Anomaly) | ❌ | ✅ | ✅ |
| 이상 차단 | ❌ | ❌ | ✅ |

> **한 줄 비유**  
> Firewall = 출입문 경비원  
> IDS = CCTV + 경보기  
> IPS = 경비원 + 경찰 + 자동문 잠금

## 🔥 전체 핵심 한방 정리

| 개념 | 핵심 한줄 |
|---|---|
| IDS | 탐지만 함, 경보기, Passive |
| IPS | 탐지 + 차단, 경찰, Active |
| Misuse | 알려진 패턴 비교, **FN(미탐) 증가** |
| Anomaly | 정상 기준 이탈 탐지, **FP(오탐) 증가** |
| False Positive | 정상인데 공격으로 판단 (오탐, 양치기소년) |
| False Negative | 공격인데 정상으로 판단 (미탐) |
| NIDS | 네트워크 패킷 감시, Promiscuous Mode |
| HIDS | 호스트 로그 분석, 내부자·백도어 탐지 강함 |
| NIDS 위치 | 방화벽 뒤에 설치 |
| 허니팟 | 해커 유인 미끼 시스템 |
| 허니넷 | 허니팟 여러 개 네트워크 |
| Enticement | 합법적 유인 ✅ |
| Entrapment | 불법적 함정 ❌ |
| FDS | 이상 금융거래 탐지 시스템 |
| Snort | 룰 기반 오픈소스 NIDS 도구 |
| SNMP | 네트워크 장비 관리 프로토콜 |
| SNMP Trap | 문제 발생 시 자동 알림 전송 |
| ACL 수정 | 공격자 IP 차단 목록 추가 |
| 바이러스 월 | 악성코드 탐지·차단 보안 장비 |
| Zero-Day | Misuse ❌ / Anomaly ✅ / 허니팟 ✅ |
