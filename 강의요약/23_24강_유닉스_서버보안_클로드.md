## 1. UNIX의 특징

| 특징 | 설명 |
|------|------|
| **대화식 운영체제** | 명령 입력 → 즉시 응답 (배치식과 반대) |
| **멀티태스킹** | 여러 작업을 동시에 실행 가능 |
| **멀티유저** | 여러 사용자가 동시에 접속 가능 (각자 격리됨) |
| **호환성/이식성** | C언어로 작성 → 하드웨어에 독립적 (다른 CPU에도 컴파일만 하면 동작) |
| **계층적 파일시스템** | `/` (루트)를 기준으로 폴더가 나무처럼 뻗어나가는 구조 |
| **뛰어난 통신 기능** | TCP/IP 내장 + 파이프(`\|`)로 명령어 간 데이터 전달 |
| **다양한 유틸리티** | gcc, vim, grep, awk 등 강력한 도구 기본 내장 |

### 이식성이 좋은 이유
```
어셈블리어 → CPU마다 코드가 달라서 매번 새로 작성해야 함 (이식성 없음)
C언어      → 컴파일러만 바꾸면 어떤 CPU에서도 동일한 코드로 동작 (이식성 좋음)
```

## 2. UNIX의 구성 — 커널

```
사용자
  ↓
셸 (Shell)        ← 명령어를 커널이 이해하는 말로 번역
  ↓
커널 (Kernel)     ← 항상 메모리에 상주
  ↓
하드웨어 (CPU · 메모리 · 디스크)
```

### 커널이 하는 일
| 기능 | 설명 |
|------|------|
| 프로세스 스케줄링 | 여러 프로그램 중 누가 CPU를 쓸지 순서 결정 |
| 기억장치 관리 | 메모리를 각 프로그램에 나눠주고 회수 |
| 파일 관리 | 하드디스크에 파일 읽기/쓰기/삭제 담당 |
| 시스템 호출 인터페이스 | 프로그램이 커널에 요청하는 공식 창구 (커널 서브루틴 형태로 존재) |
| 입출력 서비스 | 키보드 입력, 화면 출력, 디스크 읽기 등 처리 |
| CPU 제어 | CPU에게 무엇을 실행할지 지시 |

> ⚠️ **중요**: 커널은 **C언어 + 어셈블리어**로 작성됨  
> - C언어 → 이식성 확보  
> - 어셈블리어 → 성능이 중요한 일부분에만 사용

### 시스템 호출 인터페이스
- 프로그램이 하드웨어에 **직접 접근 불가** → 반드시 시스템 호출 창구를 통해야 함
- 커널 **서브루틴 형태**로 존재 (`open()`, `read()`, `write()`, `fork()` 등)
- 비유: 은행 창구 — 금고(하드웨어)에 직접 못 들어가고, 창구(시스템 호출)를 통해 직원(커널)에게 요청

## 3. 셸 (Shell)

- **명령어 해석기** — 사용자가 입력한 명령어를 커널이 알아듣는 말로 번역
- 사용자마다 **다르게 설정 가능** → `/etc/passwd`의 **7번째 필드**에 명시됨

### /etc/passwd 구조
```
형윤 : x : 1001 : 1001 : : /home/형윤 : /bin/bash
 ①     ②    ③      ④    ⑤      ⑥           ⑦
사용자명 암호  UID    GID  코멘트  홈디렉터리   셸 경로
```

### 셸 종류
| 셸 | 특징 |
|----|------|
| `bash` | 가장 많이 사용 (Bourne Again Shell) |
| `sh` | 기본 본 셸 |
| `csh` | C스타일 문법 |
| `ksh` / `zsh` | 고급 기능 지원 |

## 4. UNIX 디렉터리 구조

```
/ (루트)
├── /etc        → 시스템 설정 파일 (passwd, fstab 등)
├── /dev        → 장치 특수 파일 (하드디스크=/dev/sda, 키보드=/dev/input/...)
├── /usr
│   ├── /usr/bin   → 일반 사용자도 실행 가능한 명령어 (ls, cp, grep)
│   └── /usr/sbin  → root(관리자)만 실행 가능한 명령어 (ifconfig, useradd)
├── /home       → 사용자별 개인 공간 (/home/형윤, /home/채현)
└── /tmp        → 임시 파일 저장 (sticky bit 적용)
```

> `/usr/sbin`의 `s` = **super user** 전용

### /tmp와 sticky bit
- 누구나 파일 **생성** 가능
- 단, **본인이 만든 파일만 삭제 가능** (남의 파일 삭제 불가)
- 공용 화장실인데 내 물건은 내가만 치울 수 있는 것

## 5. 파일시스템 구조

```
┌──────────┬──────────┬──────────────┬──────────────┐
│ 부트 블록 │ 슈퍼 블록 │ i-node 리스트 │  데이터 블록  │
└──────────┴──────────┴──────────────┴──────────────┘
```

| 영역 | 역할 |
|------|------|
| **부트 블록** | 부팅 정보, OS 시작 코드 |
| **슈퍼 블록** | 파일시스템 전체 현황판 (빈 i-node/블록 위치 포함) |
| **i-node 리스트** | 파일 속성 정보 (핵심!) |
| **데이터 블록** | 실제 파일 내용 |

### 슈퍼 블록
- 파일시스템의 **주차장 전광판** ("전체 500칸 중 빈 칸 127개")
- **빈 i-node 위치** + **빈 데이터 블록 위치** 관리 → 파일 생성 시 빠르게 할당
- 손상되면 파일시스템 전체 인식 불가 → 여러 곳에 **백업 복사본** 유지

## 6. i-node (중요)

- 파일의 **주민등록증** (속성 정보 + 데이터 블록 주소 저장)
- **120byte 고정 크기** 구조체
- **파일 이름은 i-node에 없음** → 이름은 디렉터리에 따로 저장

### i-node 구성 요소

| 필드 | 설명 |
|------|------|
| **i-node number** | 파일시스템 내 파일의 고유 ID (주민번호) |
| **파일 타입** | 일반파일 / 디렉터리 / 장치파일 등 |
| **접근 권한** | rwxrwxrwx |
| **link count** | 이 i-node를 참조하는 하드링크 개수 |
| **소유자 (UID)** | 파일 소유자 |
| **소유 그룹 (GID)** | 파일 소유 그룹 |
| **파일 크기** | byte 단위 |
| **MAC TIME** | 무결성 점검용 시간 정보 |
| **block index** | 데이터 블록 주소 (직접 12 + 간접 3 = 총 15개) |

### MAC TIME (무결성 점검)

| 구분 | 영문 | 변경 시점 |
|------|------|----------|
| **M** | Modification Time | 파일 **내용** 수정 시 |
| **A** | Access Time | 파일 **접근** 시 (read만 해도 변경) |
| **C** | Change Time | 파일 **속성** 변경 시 (소유자, 권한 변경 등 i-node 정보 변경) |

> 💡 침해사고 분석 시 MAC TIME 비교 → 언제 누가 파일을 건드렸는지 추적 가능

### 블록 인덱스 구조

```
i-node
├── 직접 블록 ×12  → 데이터 블록 (작은 파일)
├── 단일 간접 블록  → 포인터 블록 → 데이터 블록
├── 이중 간접 블록  → 포인터 → 포인터 → 데이터 블록
└── 삼중 간접 블록  → 포인터 → 포인터 → 포인터 → 데이터 블록
```

- 파일이 클수록 더 많은 단계를 거쳐 데이터 블록을 찾아감
- **색인(Index)** = 실제 내용이 어디 있는지 가리키는 주소록

### 하드링크
- 같은 i-node를 가리키는 **이름표를 하나 더 붙이는 것**
- 이름은 달라도 같은 데이터 블록 공유
- link count가 **0이 되어야 진짜 데이터 삭제**

## 7. 접근권한

### 기본 권한 값
| 권한 | 숫자 | 파일 | 디렉터리 |
|------|------|------|----------|
| `r` (읽기) | **4** | 파일 읽기/복사 | `ls`로 목록 조회 |
| `w` (쓰기) | **2** | 파일 수정/이동/삭제 | 파일 생성/삭제 |
| `x` (실행) | **1** | 파일 실행 | `cd` 명령 사용, 이동/복사 |

> ⚠️ 파일 삭제/이동 → **파일이 속한 디렉터리의 w 권한** 필요  
> (파일 목록을 수정하는 행위이기 때문)

### 권한 표기법
```
- rwx r-x r--
① ②   ③   ④

① 파일 타입 (-: 일반파일, d: 디렉터리, l: 링크)
② 소유자(u) 권한
③ 그룹(g) 권한
④ 기타(o) 권한
```

### 숫자 계산 예시
```
rwx = 4+2+1 = 7
rw- = 4+2+0 = 6
r-x = 4+0+1 = 5
r-- = 4+0+0 = 4
```

### 기본값 (Default)
| 대상 | 기본값 |
|------|--------|
| 일반 파일 | **666** |
| 디렉터리 | **777** |

> 실제 생성 시 umask 값을 빼서 최종 권한 결정

## 8. 파일 권한 관리 명령어

### chmod — 접근권한 변경
- 실행 가능: **소유자(o) 또는 root**

```bash
# 기호 방식
chmod u+x file       # 소유자에게 실행 권한 추가
chmod o-w file       # 기타 사용자 쓰기 권한 제거
chmod u=r file       # 소유자 권한을 읽기만으로 지정 (나머지 제거)
chmod a+r file       # 모든 사용자에게 읽기 권한 추가

# 숫자 방식
chmod 755 file       # 소유자:rwx(7) 그룹:r-x(5) 기타:r-x(5)
chmod 644 file       # 소유자:rw-(6) 그룹:r--(4) 기타:r--(4)
```

| 연산자 | 의미 | 예시 |
|--------|------|------|
| `+` | 권한 추가 | `chmod u+x` |
| `-` | 권한 제거 | `chmod o-w` |
| `=` | **해당 권한만 남기고 나머지 제거** | `chmod u=r` (r만 남고 wx 삭제) |

### chown / chgrp — 소유자 변경
- 실행 가능: **root만**

```bash
chown 형윤 file.txt          # 소유자를 형윤으로 변경
chgrp devteam file.txt       # 소유 그룹을 devteam으로 변경
chown 형윤:devteam file.txt  # 소유자와 그룹 동시 변경
```

| 명령어 | 변경 대상 | 실행 권한 |
|--------|----------|----------|
| `chmod` | 접근권한 (rwx) | 소유자 or root |
| `chown` | 소유자 (UID) | root만 |
| `chgrp` | 소유 그룹 (GID) | root만 |

## 9. 프로세스

### 첫 번째 프로세스 생성 순서
```
부팅
  ↓
PID 0 : swapper (OS가 직접 생성)
  ↓ fork/exec × 2
PID 1 : init      → 모든 프로세스의 단군할아버지
PID 2 : pagedaemon
```

### fork / exec
| 개념 | 설명 | 비유 |
|------|------|------|
| `fork()` | 부모 프로세스를 **복사**해서 자식 생성 | 붕어빵 틀로 찍기 |
| `exec()` | 자식 프로세스를 **새 프로그램으로 교체** | 틀 안에 새 팥 넣기 |

### init 프로세스의 역할
1. **런레벨에 따른 시스템 초기화**
2. **대리모 역할** — 고아 프로세스 입양

### 프로세스 종류

| 종류 | 발생 조건 | 처리 |
|------|----------|------|
| **일반 프로세스** | 정상 실행 중 | - |
| **고아 프로세스 (Orphan)** | 부모가 자식보다 먼저 종료 | init(PID 1)이 자동으로 대리모 |
| **좀비 프로세스 (Zombie)** | 자식이 죽었는데 부모가 확인 안 함 | 부모가 wait() 호출로 수거해야 완전 소멸 |

> ⚠️ 좀비 프로세스가 많이 쌓이면 PID 고갈 문제 발생

## 10. 런 레벨 (Run Level)

시스템의 운영 상태를 숫자로 표현

| 런레벨 | 의미 | 비고 |
|--------|------|------|
| **0** | 시스템 종료 (Power off) | |
| **1** | 단일 사용자 모드 | root만 접근, 복구용 |
| **2** | 멀티유저 (네트워크 없음) | |
| **3** | 멀티유저 + 네트워크 | 텍스트 모드 |
| **4** | 사용자 정의 | 거의 안 씀 |
| **5** | 멀티유저 + GUI | 리눅스 일반 사용 모드 |
| **6** | 재부팅 (Reboot) | |

> ⚠️ **유닉스 vs 리눅스 런레벨 5 차이**  
> - 유닉스: Power off 모드  
> - 리눅스: X윈도우 GUI 멀티유저 모드

```bash
init 3   # 런레벨 3으로 이동 (텍스트 모드)
init 6   # 재부팅
```

## 11. 시스템 관리 명령어

### shutdown
```bash
shutdown -h now    # 즉시 종료
shutdown -r now    # 즉시 재부팅
shutdown -h +10    # 10분 후 종료
```

### sync (하드디스크 동기화)
- OS는 속도를 위해 데이터를 **메모리에 임시 저장** 후 나중에 디스크에 기록
- `sync` = "메모리에 남은 데이터를 지금 당장 디스크에 전부 써라"
- 시스템 종료 전 **가장 중요한 작업**

## 12. 파일시스템 마운트

### mount
- 보조기억장치의 파일시스템을 특정 디렉터리에 **논리적으로 연결**
- 비유: USB를 꽂으면 디렉터리에 연결해주는 것

```bash
mount /dev/sdb1 /mnt/usb    # /dev/sdb1을 /mnt/usb에 연결
umount /mnt/usb              # 연결 해제
```

### 관련 파일
| 파일 | 역할 |
|------|------|
| `/etc/fstab` | 부팅 시 **자동 마운트 설정표** (미리 작성, 계획표) |
| `/etc/mtab` | **현재 마운트된 목록** (시스템이 자동 관리, 실시간 현황) |

> - mount 실행 → `/etc/mtab`에 추가 기록  
> - umount 실행 → `/etc/mtab`에서 삭제

## 13. cron (정기 스케줄 관리)

- 정해진 시간에 작업을 **자동 실행**해주는 데몬
- 비유: 예약 알람 ("매일 새벽 3시에 백업 실행해")

### crontab 형식
```
분   시   일   월   요일   실행명령어
0~59 0~23 1~31 1~12 0~6
                         (0=일요일, 6=토요일)
```

### 특수 기호
| 기호 | 의미 | 예시 |
|------|------|------|
| `*` | 전부 (모든 값) | `* * * * *` = 매분 |
| `-` | 범위 | `1-5` = 1,2,3,4,5 |
| `,` | 나열 | `1,3,5` = 1, 3, 5 |
| `/` | 간격 | `*/2` = 2마다 |

### 실전 예시
```bash
0  3  *  *  *  backup.sh       # 매일 새벽 3시 정각
30 9  *  *  1  update.sh       # 매주 월요일 오전 9시 30분
0  */2 * *  *  check.sh        # 2시간마다
0  9  1  *  *  report.sh       # 매월 1일 오전 9시
*  *  *  *  *  ping.sh         # 1분마다 (전부 *)
```

### 접근 제어
| 파일 | 역할 |
|------|------|
| `cron.allow` | cron **사용 가능**한 사용자 목록 |
| `cron.deny` | cron **사용 불가**한 사용자 목록 |

> allow 파일이 존재하면 → 거기 없는 사람은 전부 차단  
> root는 항상 다른 사용자의 crontab 편집 가능

## 핵심 암기 포인트 요약

| 항목 | 암기 내용 |
|------|----------|
| UNIX 작성 언어 | **C언어 + 어셈블리어** |
| 이식성이 좋은 이유 | C언어로 작성 → 컴파일러만 바꾸면 어디서든 동작 |
| 커널 상주 | **항상 메모리에 상주** |
| i-node 크기 | **120byte 고정** |
| i-node 블록 | 직접 **12개** + 간접 **3개** = 총 **15개** |
| i-node에 없는 것 | **파일 이름** (디렉터리에 저장) |
| MAC TIME | **M**=내용수정 / **A**=접근 / **C**=속성변경 |
| 파일 기본권한 | **666** |
| 디렉터리 기본권한 | **777** |
| chmod 실행권한 | **소유자 or root** |
| chown/chgrp 실행권한 | **root만** |
| init PID | **1** |
| 런레벨 0 | 종료 / 런레벨 6 = 재부팅 |
| /etc/passwd 7번째 필드 | **셸 경로** |
| crontab 형식 | **분 시 일 월 요일 명령어** |
