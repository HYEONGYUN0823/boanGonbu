# 현대 대칭키 암호
## 현대 블록 암호의 구성요서
1. 현대 블록 암호는 확산과 혼돈 [전치], [치환]요소로 설계
2. 현대 블록 암호는 치환으로 설계, 전치는 치환에 포함된다.
3. 혼돈(confusion) : 암호문과 키의 상관관계를 숨김
4. 확산(diffusion) : 평문의 통계적 성질을 암호문 전반에 퍼뜨려 숨김
5. 대칭키 암호에 의한 암호화 = 스크램블드 에그를 만드는 것과 비슷하다 (뒤죽박죽)
# P-박스
단순, 축소, 확장
## 축소 P-박스
1. 입력 비트 중 특정 비트는 [소실]되어 출력되지 않는다
2. 비트의 수를 줄이고자 할 때 사용한다.
## 확장 P-박스
1. 비트의 양을 증가시키고자 할 때 사용한다.
## 역함수의 존재성
* 단순 P-박스는 [역함수]가 존재한다.
* 축소 P-박스와 확장 P-박스는 역함수가 존재하지 않는다.
# S-박스

## S-box는 치환암호의 개념을 비트 단위로 구현한 것이다.
1. 치환 암호의 축소 모형으로 [생각]할 수 있다.
2. S-box는 역함수가 존재할 수도 있고, 존재하지 않을 수도 있다.
3. S-box는 입력 비트와 출력 비트의 [개수]가 동일하다.

## Substitution(치환)

* 입력 -> S-box -> 값 바뀜
* 값을 다른 값으로 바꿈.
* 목적 : 혼돈(Confusion)

* 목적 : 키를 암호문에 섞는다

# 합성 암호(Product Ciphers) *3
## 개요
합성 암호는 [치환], [전치] 그리고 그 밖의 구성요소를 결합한 [복합적]인 암호이다.
## 확산(Diffusion)과 혼돈(Confusion) *3
1. 확산(diffusion)의 개념은 암호문과 평문 사이의 [관계]를 숨기는 것이다.
2. 확산은 암호문에 대한 통계 테스트를 통하여 [평문]을 찾고자 하는 공격자를 [좌절]시킨다.
3. 혼돈(confusion)의 개념은 암호문과 키의 관계를 숨기는 것이다. 키의 단일 비트가 변하면 암호문의 거의 모든 [비트]가 변한다.
* Claude Shannon의 업적
1. 정보이론(Information Theory)의 창시다
2. 일회용 패드(One-Time Pad)의 안전성을 증명 / 절대적 안전성
3. 혼돈과 확산을 정의

2. [평문 블록의 길이], [키 K의 길이], [라운드의 수]
3. [복호화 과정]은 [암호화 과정]과 [동일]하다. SPN( 암호화 과정 ≠ 복호화 과정)
4. 복호화 과정의 입력은 암호문과 보조키 $$K_i$$이며 암호화 과정의 입력 순서와 반대가 된다. 즉, 복호화 과정의 첫 라운드에 $$K_n-_1$$, 마지막 라운드에 $$K_1$$의 보조키를 입력시킨다.

## Feistel 암호 특징
1. 페이스텔 네트워크는 많은 블록 암호 알고리즘에서 이용되고 있다. AES 최종 후보로 남은 5개 중에 3개의 알고리즘(MARS, RC6, Twofish)에서 사용되고 있다.(단, AES로 최종 선정된 [Rijndael](SPN구조사용)은 사용 안함)
## 대칭 블록 암호 구현 *5
1. 블록 크기(Block size) ex) 계란 2개 vs 계란 5개
2. 키 길이(Key size) : 가장 보편적인 키 길이는 128비트이다.
3. 라운드 수 (Number of rounds)
4. [서브키 생성] 알고리즘(Subkey generation algorithm) ex) DES : 56비트 -> 48비트(서브키)
5. [라운드 함수(Round function)

## SPN 구조
1. SP Network는 "여러 개의 함수를 중첩" Shannon의 이론에 근거 Sunbstitutin Cipher와 Permutation Cipher을 [중첩]
2. 입력을 여러 개의 소블록으로 나누고 각 소블록을 S-box로 입력하여 대치(substitution)시키고 S-box의 출력을 P-box로 전치(permutation)하는 과정을 반복
* 역함수 존재요소만 사용
* 절반으로 분할 불필요

SPN 특징:
* S-box (값 바꿈)
* P-box (자리 바꿈)
* 여러 번 반복
* 복호화 시 역연산 필요
* 반으로 나누지 않음

# 블록 암호에 대한 공격 *3 -> DES에 대한 공격
## 차분 분석(Differential)
1. 선택 평문공격법, 두개의 평문 블록들의 비트의 차이에 대하여 대응되는 암호문 블록들의 비트의 [차이]를 이용하여 사용된 암호키를 찾아내는 방법
2. 즉, 블록 암호를 보면 입력되는 평문이 한 비트라도 달라지면 **암호문**은 전혀 다른 비트 패턴으로 변화하게 된다. 그래서 암호문의 변화 형태를 조사하여 해독의 실마리를 얻는 해독 방법이다.
## [선형]분석(선형 해독법)
1. 마츠이(Matsui) [기지평문 공격] 비선형 구조를 적당히 선형화시켜 키를 찾는 방법이다.
2. 근사적 선형 관계식 찾았다면 기지평문공격에서 (Chosen Plaintext)공격을 수행할 수 있다.
* 선형 : $$y = ax + b$$
* 비선형 : $$y = ax^2 + b$$ (삼각함수, 로그함수)
* DES : 2^56 - > 2^47 해독가능
* AES : 차분, 선형 분석에 안전하도록 설계

* 차분 분석 : 살짝 바꿔보고 변화 추적
* 선형 분석 : 복잡한 걸 단순 규칙으로 근사

## 전수공격법(Exhaustive key search)
모든 경우에 대하여 조사, 가장 정확한 방법이지만 경우의 수가 많은 경우에는 실현 불가능한 방법(K비트면 $2^k$의 경우의 수 발생)

[빈도] 암호문에 대한 평문의 각 단어별 [빈도] 모든 통계적인 자료를 이용하여 해독하는 방법
## 수학적 분석
[통계적인 방법]을 포함하며 [수학적 이론]* 을 이용하여 [해독]하는 방법

# 동기식 스트림 암호 *3
## 개요
암호문을 복호화하여 평문을 찾을 때 키 스트림과 암호문 사이에 [동기]가 필요한 동기식 스트림 암호 시스템/ [키 스트림]과 [암호문]의 [독립성] 정보 유출의 가능성이 작다.
## One-Time Pad
1. one-time pad(Vernam 암호)
2. 무조건 안전(unconditionally secure), 이론적으로 해독 불가능(theoretically unbreakable)하다고 알려져 있다(공개키 : 계산적 안전성) -> 절대적 안전
3. 암호화 알고리즘과 복호화 알고리즘은 각각 배타적 논리합 연산을 사용한다. 배타적 논리합 연산의 성질에 의해 옴호화 알고리즘과 복호화 알고리즘은 서로 [역관계]이다.(역수관계아님) 이 암호에서 배타적 논리합 연산을 [한 번에 한 비트]씩 적용된다. ex) 역수관계(비대칭암호)
