# 인증
## [메시지] 인증
### 기본 개념
1. 정보보안 분야에서 중요한 문제 중에 하나인 메시지 인증은 전달되는 **메시지의 이상 유무를 확인**할 수 있는 기능으로 전송 중 발생할 수 있는 메시지 내용 [변경], 메시지 순서 [변경], 메시지 **[삭제] 여부를 확인**하는 기능이다.
2. 메시지 인증 방식으로는 **메시지 암호화 방식**(관용 암호 방식/공개키 암호방식(인증모드사용(송/개->송/공), MAC(Message Authentication Code)(가장 많이 사용)방식, 해시 함수를 이용하는 방식 등이 있다.

## 메시지 인증
암호를 사용하면 [소극적 ] 공격(도청)을 막을 수 있따. [적극적] 공격(데이터나 거래의 위조)을 방어하려면 다른 방법이 필요하다. 이런 공격에 대응하는 방법으로 **메시지 [인증]**이 있다.

## 개인 식별
개인 식별은 사용자 인증과 동일한 의미로 혼용하여 사용하고 있으나 **개인 식별은 사용자 신분 확인의 발전된 방식이다.** 개인 식별은 사용자 A가 사용자 B와 협조하여 사용자 A가 본인임을 확인할 수 있으나 제3자 **C가 A로 가장하거나** 사용자 **B가 다른 제3자 D에게 사용자 [A]로 가장할 수 없는 경우를 말한다.** 사용자 인증의 경우는 사용자 **B가 다른 제3자 D에게 사용자 A로 가장할 수 있다.** <- 이런걸 막고자 하는게 개인 식별

## 패스워드(Password)
### 개요
1. 가장 간단하면서 오랫동안 사용된 개체인증 방법은 패스워드-기반 인증인데, **패스워드란 주장자가 [알고 있는 것]에 해당된다.** **시스템에 접속하고자 할 때 사용된다.(로그인)**
2. 여기서 이런 인증 구조를 두 개의 집단으로 분리할 수 있다. 하나는 [고정된] 패스워드(fixed password)이고 다른 하나는 [일회용] 패스워드(one-time password)=(동적 패스워드)이다.

### 패스워드-기반 인증
* 가장 널리 사용되는 인증방법이면서 또한 가장 안전성이 떨어지는 메커니즘이다.

### 고정된 패스워드
1. 고정된 패스워드란 접속 시에 반복해서 사용되는 패스워드를 말한다.
2. 첫번째 방법
* 매우 초보적인 방법으로 **표나 파일**에 사용자 아이디를 정렬해서 저장해 놓는다. 이 **시스템 자원에 접속하기 위해서** 사용자는 자신의 사용자 아이디와 패스워드를 [평문 형태]로 시스템에 제공한다. -> 도청, passwd 훔치기가능
3. 두 번째 방법
* 보다 안전한 방법은 패스워드를 평문으로 저장하는 대신 **패스워드의 [해시] (일방향)를 저장**하는 것이다. 모든 사용자가 파일의 내용을 읽을 수 있지만 해시함수는 일방향이기 때문에 패스워드 값을 추측한다는 것은 거의 불가능한 일이다. -> 사전공격 가능
4. 세 번째 방법 *5
* 세 번째 방법은 패스워드 **솔팅(salting)**이라고 부른다. 패스워드로 사용할 문자열이 생성되면 솔트 라고 부르는 **[랜덤] 문자열을 패스워드에 이어 붙인다.** 그리고 솔팅된 **패스워드에 해시함수를 적용**한다.
* [UNIX] 운영체제에서는 **이런 방법의 변형된 형태**를 사용한다. /etc/shadow 파일의 2번째 필드 $id$salt$encrypted-password 해시함수종류

### salt의 기능
* **오프라인 사전 공격을 통한 공격을 획기적으로 [어렵게] 만들었다.**
* 둘 혹은 그 이상의 **시스템에서** 한 사람에 **의해 사용되는 같은 비밀번호를 발견하기는 거의 [불가능]하다.**

### 일회용 패스워드(one-time password) *3  (OTP (one-time pad) : 동기식 스트림 암호)
1. 일회용 패스워드(OTP, one-time password)는 **동적 패스워드**(dynamic password)라고 불린다.
2. **재전송 공격(replay attack) 방어**
3. 첫번째 방법
* 첫 번째 방법으로 **사용자와 시스템은 패스워드 [목록]에 대해 합의를 한다.** 목록상의 각 **패스워드는 오직 [한 번]만 사용하기로 한다.**
4. 두 번째 방법
* 두 번째 방법으로 사용자와 시스템은 패스워드를 **순차적으로 업데이트**하기로 합의한다. 다시 말해서, $P_i$를 이용해서 $P_{i+1}$를 생성한다

## 영지식 개인 식별 프로토콜
### 개요
1. 패스워드를 이용한 개인 식별과 시도-응답 방식의 개인 식별 프로토콜은 클라이언트가 서버에게 패스워드나 **비밀키와 같은 비밀 정보를 알고** 있다는 사실을 [직접] 또는 **간접으로 [입증]**하는 프로토콜이다.
2. 즉, 자신의 비밀 **[정보]를 서버에게 제공하지 않고** 자신의 신분을 증명하는 방식을 영지식 개인 식별 프로토콜**(zero-knowledge identification protocal)**이라 한다.
### 알고리즘
1. 영지식 개인 식별 프로토콜에서 [클라이언트]는 자신의 신분을 증명해야 하므로 [증명자] (prover)라 하고 [서버]는 클라이언트를 확인해야 하므로 [검증자] (verifier)라고 한다.

<준비과정>
순서 1. 적당한 집합이 정해진다.
<증명 과정>
순서 2. [증명자]는 정해진 집합 내에서 임의로 선정된 **난수에 대한 [증거] (witness)를 계산하여 [검증자]**에게 제시한다.
순서 3. **검증자는 증거를 이용하여 여러 개의 [질문]을 [증명자]에게 제시한다.**
순서 4. 증명자는 검증자가 제시하는 [모든 질문에 대답함]으로써 정당한 클라이언트임을 확인받는다. 이때 대답의 **내용은 증명자의 비밀 정보를 유추할 수 있는 어떤 정보도 포함되지 않아야 한다.**

### 영지식 증명
1. 완전성 : 진짜면 항상 성공
2. 건전성(정당성) : 가짜면 항상 실패
3. 영지식성 : 비밀은 절대 안샘

## i-PIN (Type 1)
### 개요
1. 인터넷상에서 **주민번호 대신에 [아이디]와 패스워드**를 이용하여 [본인 확인]을 하는 수단이다.
2. 아이핀 아이디와 패스워드를 이용하면 웹사이트에서 더 이상 **주민번호를 이용하지 않아도 회원가입 및 기타 [서비스] 이용이 가능하다.**
### i-PIN 발급기관**(본인확인기관)**
1. i-PIN을 발급해 주는 **기관을 [본인확인기관]**이라 하고, 현재 3개의 민간 본인확인기관과 공공i-PIN센터에서 i-PIN을 발급받을 수 있다. -> 당법: 주민등록번호수집 가능한 경우

## 소유 기반 인증(What you have) Type 2
### 개요
1. 사용 권한을 가지고 있는 문의 [열쇠], [운전면허증], [신분증], [암호키] 등의 다양한 수단이 있다. 즉, 우리가 실생활에서 사용하는 **다양한 [매체]** 를 사용자를 인증하는 방안으로 활용하는 것이 **소유에 기반을 둔 인증기법의 예**이다.
2. 소유에 기반을 둔 인증 방식은 다른 사람이 쉽게 도용할 수 있기 때문에 단독으로 쓰이지 않고, 일반적으로 **Something You Know**(Type 1)나 **Something You Are**(Type 3) 방식과 함께 쓰인다.

### 신분증
누구나 하나쯤 소유하고 있는 학생증, 주민등록증, 운전면허증 등의 신분증은 본인임을 확인하기 위해 얼굴을 대조하므로, **Something You Have와 Something You Are를 모두 이용하는 방법이다.** -> Type 2분류

3. 비용 측면에서 생체 인식방식보다 더 경제적 (중간정도)

## 메모리 카드(토큰)
1. 메모리 카드는 [저장]할 수 있지만, [프로세스] 데이터는 아니다. 대부분 흔하게 볼 수 있는 카드는 뒤에 마그네틱선이 있는 은행 카드이다.
2. 마그네틱선은 오직 간단한 보안 코드를 저장할 수 있으며 이 보안코드는 **값비싼 카드리더기를 통해서만 [읽을]** 수 있다.

## 스마트카드
### 개요
1. 스마트카드는 실질적으로 **정보를 [처리]** 할 수 있다는 점에서 메모리 카드 보다 발전된 기술이다.
2. 집적 회로가 내장된 신용카드 크기의 플라슽익 카드로서 접촉 카드 혹은 비접촉 카드가 있다. **스마트카드는 장비보호 기능이 있어야 한다.**
* 스마트카드 -> 리버스엔지니어링과 같은 부정조작에 대한 보안강화 필요 -> 부당변경 방지 필요

| | 저장 | 처리 |
|---|---|---|
| 메모리카드 | O | X |
| 스마트카드 | O | O |

### 스마트카드 인증
```
  단말          리더기
CA개인키로    CA공개키로
  서명          검증
```

## 일회용 패스워드(OTP,One-Time Password) *5
### 정의
OTP용 프로그램에서 **사용자 비밀번호**와 **일회용 비밀번호** 생성용 입력값을 입력하면 암호 알고리즘을 사용해서 **일회용 패스워드를 생성하는 [사용자 인증]** 방법이다.
### OTP 특징
1. **휴대폰을 통한 인증으로 사용자의 [편리성] 및 안전성을 확보한다.** => Type 2분류
2. OTP가 토큰을 발행하는 유형에는 [질의응답], [시간] 및 [이벤트 동기화], [S/Key] 방법이 있다.
* 질의응답 = 시도응답 = 비동기식

### OTP 생성 및 인증 방식
1. 질의응답 방식

|구분|설명|
|---|---|
|특징 및 절차|시도응답 방식|
|OTP 입력|은행에서 전달받은 질의 값(임의의 난수)|
|*장점|OTP 생성 매체와 인증서버 간 [동기화]가 필요 없음|
|*단점|*사용자가 질의 값을 직접 [입력]해야 하므로 사용이 [번거로움] *은행은 같은 질의 값이 생성되지 않도록 [인증 서버]관리 필요|

3. S/KEY 방식 (Leslie Lamport)
* [벨 통신] 연구소에서 개발한 OTP 생성 방식으로, [유닉스 계열] 운영체제에서 인증에 사용되고 있다. 그 생성 알고리즘은 다음과 같다.
  1. 클라이언트로부터 받은 비밀키를 첫 번째 값으로 사용하여, [해시 체인] 방식으로 **이전 결과값에 대한 [해시값]을 구하는 작업을 [n번] 반복한다.**
  2. 그렇게 생성된 **[n개]의 OTP를 서버에 [저장]** 한다.
* 사용자 마다 별도의 관련 소프트웨어를 설치해야 하고 **정해진 회수(n)마다 시스템을 재설정**(보통 1000부터 시작) 해야하는 부담이 존재한다.
