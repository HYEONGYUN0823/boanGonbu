# RSA 암호시스템 *5
## 개요
## 가) 기본 개념
1. 인수분해 문제(Prime factorization) 공개키 암호 알고리즘으로 암호화뿐만 아니라 [전자서명]의 용도로도 사용된다.
2. SSL 프로토콜을 갖빈 많은 웹 브라우저, [PGP](이메일) 그리고 공개키 암호시스템을 사용하는 정부시스템 등이 [RSA]를 사용한다.
## 키생성
1. **알고리즘** *5
* [와 q라고 하는 두 개의 서로 다른 (p ≠ q) 소수를 고른다.
* 두 수를 곱하여 N = pq을 찾는다
* φ(N) = (p - 1) (q - 1) 를 구한다
* φ(N) 보단 작고, φ(N)과 서로소인 정수 e를 찾는다.
* 확장된 유클리드 호제법을 이용하여 d X e 를 φ(N)로 나누었을 때 나머지가 1인 정수 d를 구한다 (de ≡ 1 (mod φ(N)))
2. 만약 (공개키 n과 e로부터)(비밀키 d)를 구할 수 있다면 RSA는 해독되게 된다.
3. 안전을 위해서 권장되는 각 소수 p와 q의 크기는 [512]비트이다. 이 크기는 10진수로 약 154자리수이다. 이런 소수를 사용하면 모듈러로 사용할 n은 [1024]비트로 십진수 309자리가 된다.
RSA 알고리즘
* 공개 : n, e
* 비밀 : p, q, d, φ(N)
```
                            키생성
p, q 선택        p와 q는 모두 [소수]이다.(p ≠ q)       (개인, 선택된 것)
N = p X q를 계산                                      (공개, 계산된 것)
φ(N) = (p - 1)(q - 1)를 계산
정수 e를 선택    gcd(φ(N), e) = 1; 1 < e < φ(N)        (공개, 선택된 것)
d를 계산         dc mod φ(N) = 1                       (개인, 계산된것)
공개키           PU = {e, n}
개인키           PR = {d, n}
```
```
                  암호화
평문 : P
암호문 : C = P^e mod n
                  복호화
암호문 : C
평문 : P =C^d mod n
```

## 선택 암호문 공격 (CCA)과 OAEP
1. 복호 오라클(Decdryption Oracle)이라 한다
2. 공격자는 자신이 만든 [위조] 암호문을 서버에 어려차례 노내고, 서버가 회신해 주는 오류 메시지나 타이밍을 해석해서 [키]나 [평문] 정보를 [조금]이라도 얻으려고 하는것이다. 이 경우 서버가 복호 오라클과 [유사한] 움직임을 하게 [되어버리는] 것이다.

## 개요 *5
1. 1985년에 도입된 타원곡선 암호(ECC)는 공개키 기반 암*복호화 방법을 혁신했다. ECC는 RSA나 고전적 디피-헬먼 같은 다른 대안들보다 훨씬 강력하고 효율적이다. (키가 256비트인 ECC는 키가 4096비트인 RSA보다 강하다.)
2. 이산대수 문제(DLP)에 의존하는 대부분의 암*복호화 응용들은 DLP의 타원곡선 버전인 [ECDLP(타원곡선 이산대수 문제)]에 기초해서도 잘 작동한다.
## 특징 *5
1. 메모리와 처리능력이 제한된 응용분야에 특히 효율적이다.
2. 타원곡선 암호는 유한체상에서 $E : y^2 = x^3 + ax + b$의 형태로 정의되는 타원곡선상의 점들 간의 [덧셈] 연산을 통해 키를 산출한다
# 타원곡선 그래프
## 개요
1. $y^2 = x^3 + ax + b$의 형태이다.(바이어슈트라스표준형)
2. 타원곡선의 모양은 a와 b의 값들로 결정된다. $[4a^3 + 27b^2 ≠ 0]$의 조건을 만족해야 한다.
## 두 점의 덧셈
1. P화 Q를 연결하는 선을 그린다. 그리고 그 선이 곡선과 만나는 교점을 찾고, 그 교점에서 x축에 대칭인 점을 찾는다. 그 점이 바로 [R=P+Q]이다.
## ECDLP 문제
1. 이산대수 문제 $x = g^y mod p$를 만족한느 수 [y]를 구하는 문제이다.
2. 타원곡선에 기초한 암호학에도 비슷한 문제가 있다. 어떤 두 점 P와 Q가 주어졌을 때 Q=kP를 만족하는 수 [k](개인키)를 구하는 문제이다. 이를 타원곡선 이산대수 문제, 줄여서 ECDLP라고 부른다.
# 대칭키 방식과 비대칭키 방식의 비교 *5

| 항목 | 대칭키 | 공개키 |
|---|---|---|
| 키의 상호관계 | 암호화키 = 복호화키 | 암호화키 ≠ 복호화키 |
| 안전한 키길이 | 128비트 이상 | 2048비트 이상 |
| 암호화키 | 비밀 | 공개 |
| 복호화키 | 비밀 | 비밀 |
| 비밀키 전송 | 필요 | 불필요 |
| 키 개수(중요) | [N(N-1)/2] (nC₂) | 2N (N+N) 개인키 개인키+공개키 |
| 암호화 속도 | 고속 | 저속 |
| 경제성 | 높다 | 낮다 |
| 제공 서비스 | 기밀성 | 기밀성, 부인방지, 인증 |
| 목적 | 데이터(파일) 암호화 | 대칭키 교환 |
| 전자서명 | 복잡 | 간단 |
| 단점 | 키 교환 원리가 없다 | 중간자 공격에 취약 |
| 해당 알고리즘 | DES, 3DES, AES, IDEA | RSA, ECC, DSA |

* (b) 공개키 암호는 [중간자 공격]에 약하다.
2. 하이브리드 암호 시스템은 이 중 (a) 의 문제를 해결하는 방법이다. (b)의 문제를 해결 하려면 공개키에 대한 [인증]이 필요하다.
# 하이브리드 암호 시스템 (hybrid cryptosystem)
## 개요
1. 하이브리드 암호 시스템은 대칭키 암호와 공개키 암호의 [장점]을 살릴 수 있도록 [조합]한 방법이다.
2. 일반적으로 서로 다른 2개의 방식을 [조합]하는 것을 하이브리드라고 한다.

대칭키 = 세션키 (한 번 통신할 때만 사용하는 대칭키)
## 하이브리드 암호 시스템
대개의 경우 키기가 큰 데이터는 대칭키 암호화 기법을 사용하고 공개 키 암호화 깁버은 통신 당사자 간 [대칭키]나 해시함수에 의해 축약된 작은 크기의 정보를 안전하게 전송하기 위해 주로 사용된다.

## 하이브리드 암호 시스템의 구체적인 예
1. 유명한 암호 소프트웨어인 [PGP]와 Web의 암호통신에서 사용되고 있는 [SSL/TLS]에서도 하이브리드 암호시스템을 사용하고 있다
2. 키설계 : 대칭키 쪽 보단 공개키(오래사용)쪽을 강하게 설계
