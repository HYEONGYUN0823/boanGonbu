# 악성 소프트웨어(악성 코드) -> (개발자 악성 소프트웨어(악성 코드) -> (개발자 ≠ 사용자)
## 개요
### 악성 소프트웨어의 기본 개념
1. [개발자]가 의도적으로 [사용자]가 원하지 않는 기능을 컴퓨터 프로그램에 포함시키거나 프로그램 자체의 오류로 인하여 사용자가 원하지 않는 시스템 충돌,프로그램 중단, 개인정보 수집, **네트워크 포트 개방 등의 결과가 발생**할 수 있다.
2. 특히 **[의도적]으로 컴퓨터의 보안속성을 침해할 목적으로 작성된 프로그램**을 **악성 프로그램이라 한다.**

### 악성 프로그램 특성 비교 *5
| 구분 | 컴퓨터 바이러스(무결성) | 트로이목마(기밀성) | 웜(가용성) |
|---|---|---|---|
| 자기 복제 | 있음 | [없음] | [매우 강함] |
| 형태 | 파일이나 부트섹터 등 **감염** 대상 필요 | 유틸리티로 위장하거나 유틸리티 안에 코드 형태로 삽입 | 독자적으로 존재 |
| 전파 경로 | 사용자가 암된 파일을 옮김 | 사용자가 내려받음 | 네트워크를 통해 스스로 전파 |
| 주요 증상 | 해당 컴퓨터의 시스템 및 파일 손상 | PC 성능 저하, 좀비 PC | 네트워크 성능 저하 |

### 악성 소프트웨어의 분류 *5
#### 독립형(자체실행)과 기생형
1. **전자의 경우에 기생적(parasitic)**이라고 하는데 사실은 프로그램 단편으로 다른 실제 응용프로그램이나 유틸리티나 시스템 프로그램 없이 독립적으로 존재할 수 없다. **바이러스, 논리폭탄 그리고 백도어**가 이런 예가 된다.
2. **후자에 속하는 악석 소프트웨어**는 자체적으로 구동될 수 있는 프로그램으로 운영체제에 의해 스케줄되어 구동될 수 있다. **웜과 좀비 프로그램이 여기에 속한다.**

#### 자기 복제 여부
1. **바이러스성 악성코드에는 웜과 바이러스**가 있으며, **비-바이러스성 악성코드에는 트로이목마와 백도어 등이 있다.**
2. **웜은 자기 복제를 한다는 측면에서 바이러스와 공통점을 갖는다.** 하지만 바이러스와 달리 숙주 파일 없이 자체 실행코드를 이용해 네트워크를 통하여 **자기 자신을 복제해 전파시키며, 다른 프로그램을 감염시키지 않는다.**
3. 그러나 **바이러스**는 정삭적인 프로그램이나 데이터를 파괴할 목적으로 시스템 부트, 메모리, 파일 영역 등에 기생하다가 **다른 프로그램을 감염시킨다.**

## 바이러스(Virus)
### 개요
1. **감염(infect)시키는 프로그램** -> 최신형태 : USB를 통한 감염에 주의
2. **컴퓨터 바이러스는 자신을 완벽하게 [복제]할 수 있는 비버을 가진 명령 코드를 가지고 있다. 전형적으로 바이러스는 컴퓨터상의 프로그램이나 실행 콘텐츠의 전송자 속에 내장된다.**

### 세대별 분류(안랩) *3
1. (중요) 제1세대 원시형 바이러스(Primitive Virus)
* 실력이 뛰어나지 않은 프로그래머가 만들어 **프로그램 구조가 [단순]하고 분석이 상대적으로 쉽다.**
* **코드의 변형이나 변화 없이 고정된 크기를 가지며(가변크기 아님)**, 주로 기억장소에 상주해서 부트 영역이나 파일을 감염시킨다.
2. 제2세대 암호화 바이러스(Encryption Virus)
* **프로그램의 일부 또는 대부분을 암호화시켜 저장한다.**
3. 제3세대 은폐형 바이러스(Stealth Virus)
* 깅거장소에 존재하면서 감염된 파일의 길이가 증가하지 않은 것처럼 보이게 하며, 백신 프로그램이 감염된 부분을 읽으려고 할 때 감염되기 전의 내용을 보여줘 바이러스가 없는 것처럼 백신 프로그램이나 사용자를 속인다.
4. 제4세대 갑옷형 바이러스(Armour Virus)
* **백신 프로그램 [제작자]에게 공격의 화살을 돌려 백신 프로그램**으로부터 숨기기보다는 여러 단계의 암호화와 다양한 기법을 동원하여 바이러스 분석을 어렵게 하고 **백신 프로그램 개발을 지연시킨다.**
5. **제5세대 매크로 바이러스**(Macro Virus)
* **매크로 기능**이 있는 MS사 오피스 제품군 이외에 비지오(Visio), 오토캐드 등 VBS(Visual Basic Script)를 지원하는 다양한 프로그램에서 활동하기 때문에 현재 등장하고 있는 바이러스 중에서 **가장 높은 비중을 차지**하고 있다.

#### 일시적 바이러스 vs 상주 바이러스
* 일시적(transient) 바이러스 : **호스트의 생명주기에 의존**, 프로그램이 실행될 때 동작하고 종료될 때 끝남.
* 상주형(resident) 바이러스 : **메모리에 상주**, 프로그램의 뒷 부분에 첨부되어 독립적으로 활성화*비활성화
#### 다형성 바이러스(polymorphic virus)
* 바이러스 스캐너가 검사하기 어렵게 자신의 모습을 스스로 바꾸는 컴퓨터 바이러스. **다형성 바이러스가 다른 모습으로 변신을 하게 되면 바이러스 스캐너는 새로운 바이러스라고 인식하게 된다.** -> 잡기 힘듬 (바이러스는 주로 misuse(오용)방법으로 잡음)

### 매크로 바이러스 *5
1. 매크로 바이러스가 특히 위협적인 데에는 다음과 같은 이유가 있기 때문이다.
* **매크로 바이러스는 플랫폼과 무관하게 사용된다.**
* 대부분의 바이러스는 **문서를 감염시키고 코드의 실행부분은 감염시키지 않는다.**
* 매크로 바이러스는 쉽게 퍼진다. 가장 보편적인 방법은 전자메일이다.
* 실행 파일(확장자가 COM이나 EXE)을 다룰 때보다 **주의를 덜 하기 떄문에 피해**가 더 크다.
2. **매크로 바이러스 형태의 또 다른 희생양은 바로 Adobe사의 PDF 문서이다. PDF 문서는 자바스크립트 같은 다양한형태의 스크립트를 포함할 수 있다.**

### 바이러스 방지책 -> never ending game
#### **안티** 바이러스 방법
1. **바이러스 위협에 대한 해결책은 [예방]이다.**
2. 예방을 하면 바이러스 공격의 송공률을 줄일 수 있지만, 일반적으로 이 목적을 달성한느 것은 불가능하다. 그 다음으로 종은 해결책은 당므을 수행할 능력을 갖추는 것이다.
* **탐지(Detection)** : 일단 감염되면 바이러스가 있는지 판단하고 **위치를 파악**한다.
* **식별(Identification)** : 일단 탐지되면 프로그램을 감염시킨 **특정 바이러스를 식별**한다. (어떤 바이러스인지?)
* **제거(Removal)** : 바이러스를 모든 감염된 시스템에서 **제거하여 더 이상 퍼지지 않도록 한다.**
#### 안티 바이러스 필터링 방법 (Antivirus Filtering Method)
1. signature scanning 방법 : Misuse(오용) -> 새로운 바이러스와 웜대처 힘듬
* **특정 바이러스만이 가진 유일한 형태의 signature를 찾아내는 방법이다.** 이런 signature는 실행 가능한 코드(executable code)의 경우에는 이진 스트링으로 표현되고, 스크립트 등의 언어로 된 바이러스에서는 명령어의 나열로 나타난다. 이 방법은 **현재까지 대부분의 안티바이러스 프로그램이 채택하고 있는 방법이다.**
2. bewhavioral virus scanning 방법 : Anomaly (비정상) - 일탈
* 이 방법은 signature scanning 방법에서 불가능했던 **새로운 바이러스와 웜에 대한 대처 능력을 가질 수 있다.** -> zero day attack 방어가능

#### 바이러스 예방
1. Windows Script Host, ActiveX, VBScript, JavaScript는 **비활성화**한다. (스크립트는 바이러스 포함 가능성 높음)
2. **정책과 [교육]이 가장 훌륭한 보호대책이다.**

#### 악성코드를 운반하는 [payload] (전달해서 무엇을 하는지?(목적))의 네 가지 분류
* 시스템 파괴(오염) : **논리폭탄, Stuxnet, 랜섬웨어**
* 공격 에이전트 : **좀비, 봇(bot)**
* 정보 [유출] : **키로거, 피싱, 스파이웨어**
* 잡입(스텔스) : **백도어, 루트킷**
