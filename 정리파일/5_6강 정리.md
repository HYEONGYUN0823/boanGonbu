# 현대 대칭키 암호
## 현대 블록 암호의 구성요서
1. 현대 블록 암호는 확산과 혼돈 [전치], [치환]요소로 설계
2. 현대 블록 암호는 치환으로 설계, 전치는 치환에 포함된다.
3. 혼돈(confusion) : 암호문과 키의 상관관계를 숨김
4. 확산(diffusion) : 평문의 통계적 성질을 암호문 전반에 퍼뜨려 숨김
5. 대칭키 암호에 의한 암호화 = 스크램블드 에그를 만드는 것과 비슷하다 (뒤죽박죽)

## 차분 분석(Differential)
1. 선택 평문공격법, 두개의 평문 블록들의 비트의 차이에 대하여 대응되는 암호문 블록들의 비트의 [차이]를 이용하여 사용된 암호키를 찾아내는 방법
2. 즉, 블록 암호를 보면 입력되는 평문이 한 비트라도 달라지면 **암호문**은 전혀 다른 비트 패턴으로 변화하게 된다. 그래서 암호문의 변화 형태를 조사하여 해독의 실마리를 얻는 해독 방법이다.
## [선형]분석(선형 해독법)
1. 마츠이(Matsui) [기지평문 공격] 비선형 구조를 적당히 선형화시켜 키를 찾는 방법이다.
2. 근사적 선형 관계식 찾았다면 기지평문공격에서 (Chosen Plaintext)공격을 수행할 수 있다.
* 선형 : $$y = ax + b$$
* 비선형 : $$y = ax^2 + b$$ (삼각함수, 로그함수)

* AES : 차분, 선형 분석에 안전하도록 설계

* 차분 분석 : 살짝 바꿔보고 변화 추적

# 암호(Cipher)
## 개요 *5
1. DES에서는 페이스텔 네트워크라는 기본 구조를 사용하고 있지만, Rijndael에서는 SPN
2. SubBytes( ), ShiftRows( ), 높은 [확산](diffusion)을 제공하기 위해 열 단위로 혼합(mixing)하는 [MixColumns( ), AddRoundKey( )
3. 암호화의 마지막 라운드에서는 MixColumns( )연산을 수행하지 않는다
4. AES는 알려진 블록 암호 알고리즘에 대한 모든 공격 방법으로부터 안전하도록 설계되었으며, 하드웨어나 소프트웨어 구현 시 속도나 코드 압축성 면에서 효율적이므로 스마트카드와 같은 응용에 적합하다는 장점이 있다.
## SPN 구조 *3
* 라운드 함수가 역변환이 되어야 한다는 제약, 더 많은 병렬성을 제공하기 때문
* 여러 개의 소블록으로 나누고 각 소블록을 [S-box]에 입력하여 치환시키고 S-box의 출력을 [P-box]로 전치하는 과정을 반복한다.
## Feistel과 SPN구조 알고리즘 *5

SPN은 S로 시작하는것들이 많음

* SPN : SAFER, SHARK, Square, CRYPTON, Rijndael, SAFER+, Serpent, PRESENT, (IDEA (Reistel 변형))

나머지는 Feistel

## SEED
1. 1999년 한국정보진흥원(현 한국인터넷진흥원)KISA에서 개발 / 인터넷, 전자상거래, 무선통신
2. 128비트 비밀키에서 생성된 [16개]의 [64비트] 라운드 키를 사용하여 총 [16회]의 라운드를 거쳐 [128비트]의 평문 블록을 [128비트 암호문] 블록으로 암호화 하여 출력하는 방식이다.(2009년 256비트 키를 사용하는 SEED-256으로 확정되어 보다 강력한 암호화 능력을 제공)
