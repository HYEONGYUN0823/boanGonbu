# 기본 보안용어 정의
## 자산 (A)
1. 자산 수유자(Data Owner)가 가치를 부여한 실체 (무형자산 포함)
## 취약점 (V)
1.암호화 안된거 생각하면 될듯
## 위협 (T)
1. [행동, 사건]
* [가로채기] -> [기밀성]
* [가로막음] -> [가용성]
* [변조] -> [무결성]
* [위조] -> [무결성, 인증]

위협은 크게 2가지 위협이 있음

[자연]에 의한 위협, [인간]에 의한 위협

[인간]에 의한 위협에는 2가지가 있음

[비의도]적 위협 -> 인간의 실수와 태만이 주된 원인 -> 대책 : 철저한 보안 정책수립, 사전점검(예방)

[의도]적 위협 -> 대책 : 기술적 대책
## 위험 (Risk)
1. [가능성], [영향] = 결과
2. 자산 X 위협 X 취약점 A T V -> V.A.T
## 노출
공격에 노출되어 있는 상태

한줄로 공격에 드라난 상태
## 대책/안전장치 
ex) 강력한 패스워드 관리, 교육

위험을 줄이기 위해 적용하는 방어수단 ex) 강력한 패스워드 정책, 방화벽 설치, 접근통제 설정, 백업 시스템 구축, 암호화 적용

한줄로 태책 = 위험을 줄이기 위한 조치
## 사회공학 (제일 약한 부분은 사람이다!)
1. 인간 상호 작용의 깊은 신뢰를 바탕 -> 정상적인 보안 절차를 깨트리기 위한 침입 수단
## 시점별 통제 (*5)
1. 에방통제 -> [사전]
2. 탐지통제 -> [탐지]
3. 교정통제 -> [감소]

# 집 비유로 한 번에 정리
상황 설정 내 집에 금고가 있음

1. 자산 (A) -> 보호해야 할것

* 집 안의 금고 속 현금

2. 위협 (T) -> 해칠 수 있는 존재

* 도둑

4. 취약점 (V) -> 약한 부분

* 창문이 안 잠겨 있음

* CCTV 없음

* 비밀번호 1234임

6. 노출 ->  위협이 취약점을 이용할 수 있는 상태

* 창문 열어둔 채로 여행 감

* 비밀번호가 문 옆에 있음

* 즉, 도둑이 오면 바로 들어올 수 있는 상태

8. 위험 -> 실제로 피해가 발생할 가능성

* 도둑이 금고 털 확률 높음

10. 대책/안전장치 -> 위험을 줄이기 위한 조치

* 창문 잠그기

* CCTV설치

* 경비 시스템

* 복잡한 비밀번호 설정

12. 사회공학 -> 사람을 속여서 문을 열게 만드는 공격

ex) 도둑이 "택배 왔습니다" 하고 내가 직접 문을 열음

이건 기술적 취약점이 아닌 사람의 심리를 이용한 공격

## 시점별 통제 (예방 / 탐지 / 교정)
이걸 집 기준으로 보면
1. 예방적 통제 -> 애초에 못 들어오게 함
* 문 잠그기
* 경비원 배치
* 복잡한 비밀번호
2. 탐지적 통제 -> 들어오면 알게 함
* CCTV
* 경보음
* 접근위반로그
3. 교정적 통제 -> 사고가 난 뒤 복구
* 보험금 지급
* 파손 수리
* 트렌젝션로그 (백업으로 데이터 복원) 백업,복구 (BCP/DRP)

전체 연결 그림

자산(금고)

 ↓

위협(도둑)

 ↓
 
취약점(문 안잠김)

 ↓
 
노출(열린 상태)

 ↓
 
위험(털릴 가능성)

 ↓
 
대책(잠금장치)

그리고

사회공학 = 도둑이 속여서 문 열게 함

예방 = 잠그기

탐지 = CCTV

교정 = 보험/복구
